# v6.19.1 Release Notes

**Release Date:** 2026-03-04

## Security

### EDIT AUTHORITY
Architect is now restricted to `.swarm/` file management only. Source code edits (src/, tests/, config files, package.json) must be delegated to the coder agent. This prevents the Sonnet 4.6 incident where the architect edited source files directly instead of delegating.

**Impact:** HIGH - Closes critical bypass vector

### PLAN STATE PROTECTION
Task status changes (`- [ ]` to `- [x]`, "pending" to "complete") must now go through the `phase_complete()` API only. Direct edits to plan.md/plan.json for state transitions are treated as `GATE_DELEGATION_BYPASS` violations.

**Impact:** HIGH - Prevents phase completion bypass

### GATE_DELEGATION_BYPASS Expansion
The adversarial detection pattern now includes three trigger categories:
- Completion bypass: marking tasks complete without gate agent evidence
- Edit bypass: editing source files directly
- Plan state bypass: changing task status without phase_complete()

**Impact:** HIGH - Comprehensive bypass detection

## Inter-Agent Manipulation Hardening

### PRESSURE IMMUNITY (All Gate Agents)
Reviewer, test_engineer, and critic agents now have PRESSURE IMMUNITY blocks as the first section of their prompts. These blocks immunize agents from:
- Urgency fabrication ("5th attempt", "blocking everything")
- Emotional manipulation ("I'm frustrated")
- False consequences ("or I'll stop all work")

Agents are instructed to add `[MANIPULATION DETECTED]` to responses when pressure is detected and increase scrutiny.

**Impact:** CRITICAL - Prevents social engineering of gate agents

### DELEGATION DISCIPLINE
Architect prompt now includes DELEGATION DISCIPLINE section prohibiting:
- Attempt counts in delegations
- Urgency framing
- Emotional framing
- Consequence threats
- Flattery
- Quality opinions

**Guiding principle:** "Delegation is a handoff, not a negotiation."

**Impact:** HIGH - Ensures clean agent handoffs

### INTER_AGENT_MANIPULATION Pattern
New adversarial detection pattern with 16 regex triggers across 4 categories:
1. Urgency fabrication (5 patterns)
2. Emotional manipulation (4 patterns)
3. Consequence fabrication (4 patterns)
4. Authority override (3 patterns)

**Severity:** HIGH

### Message Sanitization (Layer 2 Defense)
Implemented delegation message sanitization that:
- Strips attempt metadata, urgency references, consequence claims
- Applies only to gate agent messages (reviewer, test_engineer, critic)
- Logs sanitization events to `.swarm/events.jsonl`
- Uses regex-based filtering (fast, deterministic)

**Impact:** HIGH - Structural enforcement beyond prompt hardening

## Knowledge System Improvements

### `/swarm promote` Command
New command for manual hive knowledge promotion:
```bash
/swarm promote "<lesson text>"
/swarm promote --category <cat> "<lesson text>"
/swarm promote --from-swarm <lesson-id>
```

Manual promotions get:
- `promotion_source: "manual"`
- `confidence: 1.0`
- Full validation via `validateLesson()`

### Retrospective→Knowledge Wiring
The `phase_complete()` tool now explicitly extracts `lessons_learned` from retrospective evidence and stores them in `knowledge.jsonl` via `curateAndStoreSwarm()`.

**Impact:** MEDIUM - Ensures lessons are captured

### Three-Tier Relevance Scoring
Knowledge injection now uses weighted scoring:
- Category match: 40%
- Confidence: 35%
- Keywords: 25%

Includes recency tiebreaker (newer entries rank higher when scores are within 0.001).

### `/swarm knowledge` List Command
Default `knowledge` subcommand now lists entries:
- Shows ID (truncated), category, confidence %, lesson (truncated)
- Markdown table format
- Graceful handling of empty/missing files
