# OpenCode Swarm v6.33.0 Release Notes

## Summary

Version **6.33.0** delivers four feature areas: atomic evidence staging, trajectory logging for audit trails, destructive command protection, and test isolation hardening. The release includes bug fixes across linting, test assertions, and CI workflows, with new configuration options for atomic staging and guardrails.

---

## Features

### 1. Atomic staging in phase_complete

Phase evidence is now written to a temporary staging path first, then atomically moved to the final location on success. This prevents partial evidence writes on failure.

Configuration: `phase_complete.atomic` (default: true), `phase_complete.archive_on_complete` (default: true).

### 2. Trajectory logger

A new hook that logs all tool executions to `.swarm/evidence/trajectory-{phase}.jsonl`. Each entry captures agent, tool, duration, success/failure status, and token counts. Provides a complete audit trail for debugging and quality analysis.

### 3. Destructive command guard

New guardrails option `block_destructive_commands` (default: true) blocks dangerous shell commands from executing during agent sessions. Protected patterns include `rm -rf`, `DROP TABLE`, `git push --force`, and other destructive operations.

Configuration: `guardrails.block_destructive_commands` (default: true).

### 4. Test isolation

CI workflow split into isolated test groups to prevent vi.mock contamination between test files. Each test group runs independently with clean module state.

## Configuration

New optional config entries (all with backward-compatible defaults):

```jsonc
{
  "phase_complete": {
    "atomic": true,              // atomic evidence staging (default: true)
    "archive_on_complete": true  // archive evidence after phase completion (default: true)
  },
  "guardrails": {
    "block_destructive_commands": true  // block dangerous shell commands (default: true)
  }
}
```
