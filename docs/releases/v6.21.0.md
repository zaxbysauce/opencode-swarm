# OpenCode Swarm v6.21.0 Release Notes

## Summary

Version **6.21.0** introduced a per-task state machine architecture with explicit state transitions, scope declaration for task containment, and hard blocks that enforce gate policies at the infrastructure level.

---

## Per-Task State Machine

### Explicit state transitions

Each task now follows a well-defined state machine with explicit transitions: `pending` → `in_progress` → `completed` / `blocked` / `failed`. State transitions are logged and enforceable.

### State transition validation

Invalid state transitions are rejected with clear error messages, preventing corrupted task states and ensuring reliable workflow execution.

---

## Scope Declaration

### Task-level scope boundaries

Tasks now declare explicit file and directory scopes at the task level. Scope declarations enable precise containment — agents can only modify files within their declared boundaries.

```jsonc
{
  "tasks": {
    "implement-login": {
      "scope": {
        "files": ["src/auth/login.ts", "src/auth/session.ts"],
        "directories": ["src/auth/components"]
      }
    }
  }
}
```

### Scope validation at delegation

Scope declarations are validated at task delegation time, preventing scope creep and ensuring work stays within intended boundaries.

---

## Hard Blocks (Gate Enforcement)

### Infrastructure-level gate enforcement

Hard blocks replace soft advisories with enforceable gate checks at the infrastructure level. When a gate condition is not met, execution is blocked until the condition is satisfied.

### Block categories

| Block Type | Trigger | Resolution Required |
|------------|---------|-------------------|
| `scope_violation` | File access outside declared scope | Scope expansion or task split |
| `test_failure` | Required tests failing | Fix tests or update test requirements |
| `lint_failure` | Lint errors present | Fix lint errors |
| `gate_failure` | Quality gate criteria unmet | Address gate requirements |

---

## Migration Notes

- No breaking changes. Existing tasks without explicit scopes default to project-wide access.
- Hard blocks are opt-in via configuration; set `guardrails.enforce_hard_blocks: true` to enable.
