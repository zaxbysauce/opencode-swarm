# v6.59.0 (2026-04-09)

## Summary

This release completes the Explorer Role Hardening project and fixes 15 stale test failures. Explorer agent is now strictly factual with no judgment, verdict, or routing authority. All routing decisions are owned by Architect.

## What Changed

### Explorer Role Hardening (Phases 3-5)
- **Explorer Prompt** (`src/agents/explorer.ts`): Removed all judgmental language. `VERDICT` replaced with `COMPATIBILITY SIGNALS`, `MIGRATION_NEEDED` replaced with `MIGRATION_SURFACE`. Explorer now reports observable facts only.
- **Architect Prompt** (`src/agents/architect.ts`): Added `EXPLORER ROLE BOUNDARIES` section and `ROLE-BOUNDARY CHANGE VALIDATION` step. Architect owns all routing decisions.
- **Reviewer Prompt** (`src/agents/reviewer.ts`): Added `EXPLORER FINDINGS — VALIDATE BEFORE REPORTING` section. Reviewer must validate explorer findings.
- **Critic** (`src/agents/critic.ts`): Fixed stale comments referencing `createExplorerCuratorAgent` → `createCuratorAgent`.

### Documentation
- **Architecture Docs** (`docs/architecture.md`): Added Role Boundary Hardening (Phase 3-5) section with behavioral rules, before/after examples table, and test coverage note.

### New Test Files
- `src/agents/explorer-role-boundary.test.ts`: 25 adversarial prompt-contract tests for explorer role boundaries
- `src/agents/explorer-consumer-contract.test.ts`: 58 consumer regression tests for explorer output contract
- `tests/unit/agents/explorer-*.test.ts`: 6 additional test files for explorer sections and observations

### Test Fixes (15 stale test failures resolved)
- Updated explorer output format expectations to match new `COMPATIBILITY SIGNALS` / `MIGRATION_SURFACE` format
- Fixed word-boundary matching for coder tool substring checks (`patch` vs `suggest_patch`)
- Fixed bypass detection to exclude env var names (`SWARM_SKIP_SPEC_GATE`)
- Added `lint_spec` to `TOOL_DESCRIPTIONS` in constants.ts
- Updated agent count expectations (12 → 13) for `critic_oversight` agent
- Updated critic prompt export expectations (3 → 4) for `AUTONOMOUS_OVERSIGHT_PROMPT`
- Fixed regression sweep text expectations to match current prompt format

## Why

These changes address:
- **Role clarity**: Explorer was making routing decisions beyond its scope as a factual mapper
- **Test debt**: 15 test failures accumulated from prior agent/tool additions not reflected in test expectations
- **Documentation**: Role boundary hardening was not documented

## Migration Steps

None. This is an additive change with no breaking API or behavior changes.

## Known Caveats

Integration tests (Tier 3) have 88 pre-existing failures unrelated to this change. Security/adversarial tests (Tier 4) have 22 pre-existing failures. These failures existed before this release and are tracked separately.
