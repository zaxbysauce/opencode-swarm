# OpenCode Swarm v6.17.1 Release Notes

## Summary

Version 6.17.1 delivers **Knowledge System Hotfix** — six orphaned exports and missing command wirings introduced in v6.17.0 are resolved, completing the knowledge pipeline end-to-end.

- **`/swarm knowledge migrate` command**: `migrateContextToKnowledge()` was exported but never callable from the CLI — now fully routed and help-text documented
- **Retrieval outcome recording**: `updateRetrievalOutcome()` was exported but had zero callers — now called after each curation cycle to close the feedback loop
- **Dark matter → knowledge pipeline**: `darkMatterToKnowledgeEntries()` was exported but never persisted — `/swarm dark-matter` now appends findings to `.swarm/knowledge.jsonl`
- **Help text and architect prompt**: Both the knowledge command help block and the architect agent's SLASH COMMANDS list now list all three knowledge subcommands (quarantine, restore, migrate)

---

## Detailed Fixes

### 1. `/swarm knowledge migrate` Command Wiring

**Problem**: `migrateContextToKnowledge()` was implemented in `knowledge-migrator.ts` and exported, but `commands/knowledge.ts` had no handler for it and `commands/index.ts` had no routing case. The command was silently unreachable.

**Fix**: Added `handleKnowledgeMigrateCommand(directory, args)` to `src/commands/knowledge.ts` with full `MigrationResult` handling (skipped/migrated/partial outcomes). Wired the `migrate` routing case in `src/commands/index.ts` and added `migrate` to the help text `else` branch listing all subcommands.

---

### 2. Retrieval Outcome Recording (`updateRetrievalOutcome`)

**Problem**: `updateRetrievalOutcome()` in `knowledge-reader.ts` was designed to record whether injected knowledge entries led to successful phases — but it had zero callers. The feedback loop was broken: knowledge was injected, used, and never marked as producing good outcomes.

**Fix**: Added `await updateRetrievalOutcome(directory, \`Phase \${phaseNumber}\`, true)` in `src/hooks/knowledge-curator.ts` after the `curateAndStoreSwarm()` call. The import was added without creating a circular dependency (curator → reader is safe).

---

### 3. Dark Matter → Knowledge Pipeline

**Problem**: `darkMatterToKnowledgeEntries()` in `co-change-analyzer.ts` was designed to convert co-change findings into persistent knowledge entries — but `src/commands/dark-matter.ts` never called it. Running `/swarm dark-matter` would display findings once and discard them.

**Fix**: `src/commands/dark-matter.ts` now calls `darkMatterToKnowledgeEntries(pairs, projectName)` and appends each returned entry to `.swarm/knowledge.jsonl` via `appendKnowledge`. Uses `path.basename(path.resolve(directory))` to avoid `.` as project name. Guards on `entries.length > 0` before writing. Appends a summary line `[N dark matter finding(s) saved to .swarm/knowledge.jsonl]` to the output. Persistence failures are caught and warned without affecting the displayed output.

---

### 4. Help Text and Architect Prompt Documentation

**Problem**: The knowledge command help block (in the `else` branch of `src/commands/index.ts`) and the architect agent's SLASH COMMANDS list (in `src/agents/architect.ts`) both listed only `quarantine` and `restore`, omitting `migrate`.

**Fix**: Both locations now list all three subcommands: `knowledge quarantine`, `knowledge restore`, and `knowledge migrate`.

---

## Upgrade Notes

- **Backward compatible**: All existing behavior preserved. No breaking changes.
- **`/swarm knowledge migrate`**: Now callable. Run it once on any project with a mature `.swarm/context.md` to bootstrap the knowledge base from existing Lessons Learned, Patterns, and Decisions sections.
- **Retrieval outcomes**: No action needed — recording happens automatically during each curator hook cycle.
- **Dark matter persistence**: On next `/swarm dark-matter` run with findings, up to 10 entries will be appended to `.swarm/knowledge.jsonl` as `architecture` category candidates.
- **Idempotency note**: Running `/swarm dark-matter` twice will append duplicate entries. A deduplication pass at injection time (via existing near-duplicate detection) will filter them before they reach the architect.
