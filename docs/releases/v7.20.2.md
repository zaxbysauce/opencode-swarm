# v7.20.2

## What changed

- **phase_complete no longer emits internal diagnostics to the chat-visible console path** (`src/tools/phase-complete.ts`)
  - Replaced non-fatal `console.warn`/`console.info`/`console.error` calls with structured tool warnings or debug-gated logger calls
  - Preserved user-facing gate-skip feedback through the tool result `warnings` array

- **Regression coverage for chat-bar log leaks** (`tests/unit/tools/phase-complete.test.ts`)
  - Added a turbo-mode regression test that verifies phase completion returns a warning without writing to `console.warn`/`console.error`
  - Added a non-code drift-skip regression test that verifies the skip is returned in `warnings` without writing to `console.info`

## Why

Fixes issue #875, where internal phase-boundary diagnostics leaked into the OpenCode chat bar instead of staying in debug logging or structured tool output.

## Migration steps

None.

## Breaking changes

None.

## Known caveats

- Broader repository validation still exposes unrelated pre-existing failures in diagnostic-gating, integration, and adversarial suites; this patch only changes the `phase_complete` logging path.
