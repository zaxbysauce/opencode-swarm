# Release v6.83.0

## What Changed

### Council Critic Baseline Drift Enforcement
- **Tool access:** Added `get_approved_plan` to `AGENT_TOOL_MAP.critic` so the live council critic can retrieve the last approved plan baseline during review. Intentionally excluded from `critic_sounding_board` to preserve least-privilege divergence.
- **Prompt contract:** Added a mandatory `BASELINE COMPARISON` block to `PLAN_CRITIC_PROMPT` requiring the critic to call `get_approved_plan`, handle all four response states (`no_approved_snapshot`, `drift_detected: false/true/unknown`), report unauthorized mutations as CRITICAL, and evaluate spec-intent divergence beyond structural diffs.
- **Council dispatch:** Updated `buildCouncilWorkflow` Phase 1 critic dispatch to explicitly mandate approved-plan baseline comparison and spec-intent drift analysis.
- **Scope verification:** Inspected all 6 critic dispatch paths. Documented `critic_oversight` (full-auto autonomous oversight) as explicitly outside council/live plan-review scope, with tests verifying the separation.
- **Regression protection:** Verified existing `hallucination_guard` enforcement remains intact — zero code changes to `phase-complete.ts`, all 8 regression tests pass.

### Test Coverage
- Added 23 new tests across 3 test files:
  - `critic-registration.test.ts`: least-privilege divergence assertions
  - `critic.test.ts`: baseline comparison content, leak prevention (hallucination verifier isolation), oversight exclusion
  - `architect-council-prompt.test.ts`: council critic dispatch contract, council/oversight scope separation, non-regression byte-equivalence

## Why

GitHub issues #490 (critic missing `get_approved_plan` baseline comparison during council review) and #491 (hallucination guard already implemented — preserved as regression baseline). Fixes the gap where live council critics could not detect plan-level drift or spec-intent divergence because they lacked both the tool access and prompt instructions to compare against the approved baseline.

## Migration Steps

No migration required. This is a pure additive change:
- Existing plans without council mode enabled are unaffected.
- Existing `critic_drift_verifier` behavior is unchanged.
- No config or schema changes.

## Breaking Changes

None.

## Known Caveats

- Advisory items identified by the aggregate council review are tracked in #581 for future sprints (minor polish: `summary_only` guidance, snapshot guards, test coverage completeness).
