# v7.3.6 — Mutation testing reliability: timeout guard + Windows CRLF fix

## What changed

### Bug fix: `generate_mutants` no longer hangs when called in parallel

`generateMutants()` now wraps its two LLM network calls (`session.create` and
`session.prompt`) inside a 90-second `withTimeout` deadline. Previously, if the
calls were dispatched in parallel with other long-running tools, the OpenCode
plugin host could exhaust its wall-clock budget and abort the call before the JS
catch block could fire — leaving the tool stuck with "Tool execution aborted".

Now the function races the LLM work against a 90-second ceiling. If the deadline
fires, the existing outer catch returns `[]` (SKIP verdict) and logs a warning.
On the happy path (LLM responds in time) behaviour is unchanged.

A `_internals.timeoutMs` DI seam is exported so tests can override the ceiling to
50 ms and exercise the timeout path without waiting.

### Bug fix: `mutation_test` patches now apply correctly on Windows (CRLF/LF)

`executeMutation()` passes `--ignore-whitespace` to both `git apply` (apply) and
`git apply -R` (revert). This flag makes git ignore CR/LF differences in context
lines during patch matching.

**Root cause:** On Windows with `core.autocrlf=true` (the default), files in the
working tree use CRLF line endings. LLM-generated patches always use LF. Git's
context-line matching is byte-for-byte: a LF context line never matches a CRLF
line. Every patch returned `outcome: error`, giving a 0% kill rate. The same
issue can occur on any platform with CRLF-committed files.

`--ignore-whitespace` is a no-op on macOS and Linux (LF-native). Available in all
git ≥ 1.8 (released 2012).

The two argument builders are exported as `buildGitApplyArgs` and
`buildGitRevertArgs` for direct unit testing.

## Why

Both bugs were observed in a live agent session (screenshot):

1. `generate_mutants` was dispatched as one of three parallel tool calls. The two
   sibling calls completed; `generate_mutants` was aborted by the host.
2. `mutation_test` reported "0 killed, 0 survived (patch application failed)" on
   Windows — all 5 generated mutants hit `git apply` errors.

## Migration steps

None. The changes are additive and backwards-compatible:
- `withTimeout` wrapper is transparent on the happy path.
- `--ignore-whitespace` is a no-op on LF-native systems.

## Breaking changes

None.

## Known caveats

- The 90-second LLM deadline applies per `generate_mutants` call. For very slow
  models or large files, increase via `_internals.timeoutMs` if needed (for
  testing; production callers use the default).
- Windows CI is not exercised by this sandbox; the CRLF fix is validated by unit
  tests on `buildGitApplyArgs`/`buildGitRevertArgs` and by the known `git apply`
  semantics of `--ignore-whitespace`.
