# OpenCode Swarm v6.30.0 Release Notes

## Summary

Version **6.30.0** introduced the curator background analysis system, phase-level drift detection, knowledge injection, comprehensive agent prompt overhauls, secretscan path exclusion, session durability improvements, and turbo mode optimizations.

---

## Curator Background Analysis

### Async knowledge curation

The curator now runs as a background service that continuously analyzes completed work and extracts knowledge for future reference, without blocking active task execution.

### Curated knowledge recommendations

Automatically generated recommendations surface relevant past decisions and patterns when starting new tasks with similar contexts.

---

## Phase-Level Drift Detection

### Architecture drift monitoring

Detects when implementation diverges from planned architecture at phase boundaries, flagging inconsistencies for human review.

### Configuration drift detection

Monitors for unexpected configuration changes that may affect task behavior or quality gates.

---

## Knowledge Injection

### Contextual knowledge retrieval

Agents now receive relevant knowledge from the knowledge base at task start, reducing redundant reasoning and improving consistency with past decisions.

### Manual knowledge addition

New `/swarm knowledge-add` command for explicitly saving lessons and patterns:

```
/swarm knowledge-add "Always validate file paths before glob operations" --category debugging
```

---

## Agent Prompt Overhauls

### Coder prompt improvements

Enhanced coder agent prompts for better code quality awareness, clearer instruction following, and improved error messages.

### Reviewer prompt improvements

Updated reviewer prompts for more thorough code review with better false positive suppression.

### Docs agent prompt improvements

Revised documentation agent prompts for improved tone consistency and structure.

### SME agent prompt improvements

Updated SME agent prompts for clearer expert knowledge extraction.

### Explorer agent prompt improvements

Enhanced explorer prompts for more efficient codebase navigation.

### Test-engineer prompt improvements

Revised test generation prompts for better test coverage and edge case discovery.

---

## Secretscan Glob/Path Exclusion

### Fine-grained scan exclusion

Exclude specific paths or patterns from secrets scanning:

```jsonc
{
  "security": {
    "secretscan": {
      "exclude": [
        "**/*.test.ts",
        "docs/**",
        "node_modules/**"
      ]
    }
  }
}
```

---

## Session Durability

### Improved snapshot reliability

Enhanced session snapshot saving to prevent work loss on unexpected interruptions.

### Faster session recovery

Reduced time to restore session state from snapshots.

---

## Turbo Mode

### Performance optimizations

New turbo mode for faster iteration with reduced quality overhead:

```
/swarm turbo
```

Skips optional quality hooks for faster feedback during development.

---

## Migration Notes

- No breaking changes. Turbo mode and curator analysis are opt-in.
- Secretscan exclusion patterns require explicit configuration to activate.
- Knowledge injection is enabled by default for new sessions.
