# Release Notes — v6.35.0

**Minor Release | 2026-03-26**

---

## Overview

This release modernizes agent prompts with research-backed improvements, introduces a canonical path security module, and strengthens the evidence pipeline. No breaking changes require migration.

- **Agent prompts**: 6 research-backed additions across Coder, Reviewer, Test Engineer, Explorer, Architect, and Docs agents
- **Path security**: New canonical `src/utils/path-security.ts` replaces 9 duplicate implementations across tools
- **Evidence pipeline**: Gate evidence now recorded even when evidenceTaskId resolves to null
- **Model fallback**: `fallback_models` config field wired from dead code into runtime resolution
- **Migration**: 97 test files migrated from vitest to bun:test

---

## New Features

### Agent Prompt Modernization
Six research-backed prompt additions based on 2026 LLM research across all major agents:

- **Coder**: ANTI-HALLUCINATION PROTOCOL (grep-before-import) + TEST FRAMEWORK (bun:test instructions)
- **Reviewer**: SAST TRIAGE subsection (dismiss false positives from static analysis) + explicit DO list
- **Test Engineer**: bun:test migration instructions + MOCK ISOLATION RULES (vi.clearAllMocks in afterEach)
- **Explorer**: DOCUMENTATION DISCOVERY MODE (auto-scan project docs, write doc-manifest.json)
- **Architect**: DOCUMENTATION AWARENESS (check doc constraints before each phase) + VERIFICATION PROTOCOL (read modified files before running Stage B gates)
- **Docs agent**: RELEASE NOTES guidance + DOCUMENTATION RULES (no auto-generated CLAUDE.md/AGENTS.md)

### Path Security Module
New canonical `src/utils/path-security.ts` with deduplicated implementations:

- `containsPathTraversal` — detects path traversal sequences
- `containsControlChars` — blocks control characters including tab
- `validateDirectory` — directory safety validation

This replaces 9 duplicate implementations scattered across tools.

### Architect TASKID Protocol
Architect prompt now instructs inclusion of TASKID in delegation envelopes for reliable gate evidence recording.

### Model Fallback Runtime Wiring
The `fallback_models` config field (previously parsed but never consumed) is now wired into the runtime error detection path. When a transient model error (429, 503, timeout) is detected by guardrails, the system resolves the actual fallback model name from the user's config and includes it in the advisory message. Users can configure fallback models per agent:

```json
{
  "agents": {
    "coder": {
      "model": "primary-model",
      "fallback_models": ["fallback-a", "fallback-b"]
    }
  }
}
```

## Bug Fixes

- **Tab-char bypass in secretscan**: `containsControlChars` now blocks `\t` in addition to `\x00-\x1F`
- **Evidence pipeline gap**: Gate evidence now recorded even when evidenceTaskId resolves to null — added fallback scan of agent sessions + advisory messages
- **Critic model inheritance**: Critic sub-roles (sounding_board, drift_verifier) now check for explicit user config before falling back to the `critic` model. Users who set `critic_sounding_board.model` in their config get their choice; all others inherit from `critic`.
- **Model fallback warning**: One-time `console.warn` when model resolution falls back to DEFAULT_MODELS
- **Phase-complete async I/O**: Converted 4 `readFileSync` calls to async `readFile` in phase-complete.ts
- **No-op warning reset**: The no-op detection warning (fired after 15 tool calls with no file writes) now resets on each successful file write. Previously it fired only once per session, missing agents that looped after a productive write.
- **Biome CI**: Fixed all lint errors (8 no-explicit-any, 2 unused params, 2 use-template warnings)
