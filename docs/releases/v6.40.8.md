# v6.40.8

## Bug Fixes

- **Fix stale `coder_delegated` state permanently blocking the reviewer gate (Bug B)**: When sessions were rehydrated from disk, stale `coder_delegated` entries from prior sessions would permanently block coder re-delegation. The gate now detects stale state by checking whether the current session's delegation chains contain evidence of a recent coder delegation, and resets stale entries to `idle` instead of blocking.

- **Add `sessionRehydratedAt` timestamp for reliable rehydration-aware staleness detection**: The staleness check uses a `sessionRehydratedAt` timestamp (set during snapshot rehydration) to distinguish between delegation chain entries restored from a prior session vs entries created in the current session. This prevents false negatives where restored delegation chains would make stale state appear current.

- **Clear delegation chains on `/swarm reset-session`**: The `reset-session` command now clears `swarmState.delegationChains` alongside agent sessions, ensuring manual session resets fully clear all delegation state that could cause stale blocking.

- **Add recovery instruction to `REVIEWER_GATE_VIOLATION` error message**: The error message now includes guidance to run `/swarm reset-session` if the blocking state is from a prior session, giving users an immediate escape hatch.

### What Changed

- **`src/hooks/delegation-gate.ts`**: Added staleness detection logic in `toolBefore` handler before the reviewer gate block. Uses `sessionRehydratedAt` as the primary freshness threshold for rehydrated sessions, falling back to `lastPhaseCompleteTimestamp` for non-rehydrated sessions. Stale `coder_delegated` entries are reset to `idle` with a console warning. Updated `REVIEWER_GATE_VIOLATION` error message to include `/swarm reset-session` recovery instruction.

- **`src/state.ts`**: Added `sessionRehydratedAt: number` field to `AgentSessionState` interface (defaults to 0 for fresh sessions). Added migration safety check in `ensureAgentSession`.

- **`src/session/snapshot-reader.ts`**: Sets `sessionRehydratedAt = Date.now()` during session rehydration from snapshot. Added field to deserialization.

- **`src/session/snapshot-writer.ts`**: Added `sessionRehydratedAt` to serialization interface and serialization function.

- **`src/commands/reset-session.ts`**: Added `swarmState.delegationChains.clear()` call during session reset.

- **`tests/unit/hooks/delegation-gate-stale-state.test.ts`**: New test file with 11 tests covering: empty chains, no coder entries, stale timestamps, fresh legitimate blocking, error message content, multiple stale tasks, non-coder delegation passthrough, prefixed agent names, rehydrated session with restored chains, post-rehydration fresh delegation, and non-rehydrated fallback behavior.

### Why

Bug B was identified as a session-persistence issue where `taskWorkflowStates` entries survived across sessions via snapshot rehydration but the session context that would have resolved them (reviewer delegation) was lost. The previous gate check was session-blind — it could not distinguish between a legitimate in-session `coder_delegated` (should block) and a stale one from a dead session (should reset). This caused tasks to become permanently stuck.

### Migration

None required. The fix is backward-compatible — existing sessions will benefit from automatic stale detection on next startup. The `sessionRehydratedAt` field defaults to 0 for sessions that were persisted before this version.

### Breaking Changes

None.
