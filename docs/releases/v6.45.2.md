# v6.45.2

## What changed

### Per-agent file write authority fixes

- Prefixed agents now inherit their canonical authority rules when no exact override exists.
- Mixed-case authority config keys are normalized before lookup.
- Architect direct writes are now checked in the runtime `toolBefore` hook.
- An explicit empty `allowedPrefix: []` now denies all writes instead of bypassing the allowlist.
- README authority docs were updated to match the implemented behavior.

## Why

These fixes close correctness gaps in the per-agent file write authority system so prefixed swarm agents, runtime architect writes, and config overrides all behave consistently.

## Migration steps

None.

## Breaking changes

None.

## Known caveats

Custom agents without explicit authority rules are denied writes by default.
