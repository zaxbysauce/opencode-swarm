# v6.80.2 — Spiral Detection Fix

## What changed

Fixed a critical bug in spiral detection that caused 2–XX empty checkpoint commits with message `checkpoint: spiral-unknown-xxxx` to be created during normal operations before the actual code commit.

## Root causes fixed

1. **Cross-session buffer pollution** — The `recentToolCalls` buffer was a module-level global array shared across all sessions. Tool calls from any session contributed to every other session's spiral detection threshold, causing false-positive spirals with `taskId='unknown'`.

2. **No cooldown after detection** — Once a spiral was detected (5+ identical tool calls), the detection would re-fire on every subsequent tool call because the buffer was never cleared. This resulted in a new checkpoint commit for each additional tool call, producing 2–XX empty commits in a single operation.

3. **Uninformative taskId fallback** — When `currentTaskId` was `null` (before the first task status update), `taskId` fell back to the literal string `'unknown'`, producing commit labels like `spiral-unknown-1713456789123`.

4. **Unbounded memory growth** — The `lastSpiralTimestampBySession` map grew indefinitely as sessions accumulated, with no cleanup mechanism.

## How it's fixed

- **Per-session buffers** — Replaced the global `recentToolCalls` array with `recentToolCallsBySession: Map<sessionId, Array>`. Each session now maintains an independent call history, preventing cross-session false positives.

- **60-second cooldown + buffer clearing** — Added `SPIRAL_COOLDOWN_MS = 60_000` and `lastSpiralTimestampBySession: Map<sessionId, number>` to prevent re-triggering within the same session. When a spiral is detected, the session's buffer is immediately cleared, forcing a fresh detection window. Together, these changes limit spiral checkpoints to at most one per session per 60 seconds.

- **Session-based taskId fallback** — Changed from `'unknown'` to `session-${input.sessionID.slice(0, 12)}`, ensuring checkpoint labels are always session-identifiable and informative.

- **Memory eviction** — Added `MAX_TRACKED_SESSIONS = 500` with FIFO (insertion-order) eviction. When the timestamp map exceeds 500 entries, the oldest-inserted entry is deleted to prevent unbounded growth. The current session's own entry is skipped during eviction to prevent self-eviction immediately after detection.

- **Tests** — Added session isolation test (verifying that Session A's spiral doesn't affect Session B) and cooldown enforcement test (verifying that re-detection is blocked within 60 seconds). Deleted the stale `adversarial-detector-spiral.test.ts` file whose implementation no longer exists.

## Impact

Users will no longer see spurious empty checkpoint commits. Spiral detection (an intentional safety mechanism to alert when an agent is stuck in a retry loop) will still function correctly, but will now create at most one checkpoint commit per session per 60-second window.
