# v6.85.2

## Summary

- **Fixed**: `/swarm diagnose` command now correctly detects WASM grammar files when run via CLI binary (`bunx opencode-swarm run diagnose`). Previously reported all 20 grammar modules as missing due to incorrect bundle path resolution.
- **Fixed**: C# syntax checking no longer throws "Invalid language ID" errors. Grammar ID and WASM filename now use correct hyphens (not underscores).
- **Hardened**: Grammar path resolution in both `diagnose-service.ts` and `runtime.ts` now correctly handles all three bundle contexts (dev source, main plugin bundle, CLI binary bundle).

## Why

The CLI binary bundle (`dist/cli/index.js`) resolved `import.meta.url` to its own directory, causing the `checkGrammarWasmFiles()` function to look for grammars at the non-existent `dist/cli/lang/grammars/` instead of `dist/lang/grammars/`. Users running `bunx opencode-swarm run diagnose` would see all grammar files reported missing even though they were correctly installed.

The C# profile had inconsistent naming (`grammarId: 'c_sharp'` with underscore) which failed the grammar ID sanitizer and broke C# file parsing.

## Test coverage

- 6 new regression tests for `resolveGrammarDir` covering all bundle contexts and Windows paths
- 2 new assertions locking in corrected C# profile configuration
- All existing tests pass; no test regressions

## Known caveats

The `/cli` suffix detection (`endsWith('/cli')`) is broad but safe in practice — users are unlikely to deploy opencode-swarm in a path ending with `/cli`. The fix is defensive and also applied to `runtime.ts` to prevent similar issues if grammar loading is ever called from the CLI bundle in the future.
