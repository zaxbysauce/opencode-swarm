# Release Notes v6.60.1

## Summary

Bug fix release addressing 6 non-blocking gaps in the authority glob pattern guardrails system.

## What Changed

- **Cache key fix**: Fixed bug where `normalizePathWithCache` received `cwd` instead of resolved `dir`, causing incorrect cache keys when `cwd` was empty string
- **New export**: Added `clearGuardrailsCaches()` function for test isolation and runtime config reload support
- **Resilience**: Added try-catch wrapper to `getGlobMatcher` to gracefully handle malformed glob patterns (falls back to permissive matcher)
- **Case-insensitivity**: Parameterized `nocase` option in `getGlobMatcher` with platform-aware default for better CI testability
- **Tests**: Added test coverage for malformed glob pattern fallback behavior

## Why

Addressed follow-up items from post-merge review of PR #437 (authority glob pattern guardrails). None are breaking changes.

## Documentation Updates

- Fixed evaluation order in README (blockedPrefix now correctly shown before allowedPrefix)
- Added "Path Normalization and Symlinks" section documenting dangling symlink limitation and mitigation

## Known Caveats

- Malformed glob patterns in `blockedGlobs` will not block (fail-open behavior) - this is intentional to avoid crashes on bad patterns. Use `blockedExact` or `blockedGlobs` with well-formed patterns for security-critical blocking.
