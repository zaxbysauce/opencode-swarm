# OpenCode Swarm v6.85.0

**Released:** 2026-04-26

## What Changed

### New Features

- **Auto-generated project config reference** — On first plugin load, `.swarm/config.example.json` is automatically created in the project directory. This file contains all agent models with fallback chains, allowing users to easily see what models are available and copy entries to their `.opencode/opencode-swarm.json` for project-local overrides. (Resolves #594 point 3)

- **Web app example tutorial** — New comprehensive guide in `docs/examples/web-app.md` walks through building a React application with authentication using opencode-swarm. Covers the full autonomous workflow: brainstorm planning (all seven phases), TRANSITION checkpoint, and autonomous execution. (Resolves #594 point 4)

### Improvements

- **Clearer onboarding documentation**:
  - `docs/getting-started.md` Step 1: Fixed error messages to clearly direct users to install Bun first (not suggest npm as an equal alternative)
  - New Step 5.5: Explicit explanation of the TRANSITION checkpoint where the architect waits for human confirmation before proceeding with implementation
  - Added link to new web app example in Step 7 (Next Steps)
  - Enhanced `docs/index.md` with new Examples section linking to web-app tutorial

- **Installation verification** — CLI install output now explicitly lists `/swarm diagnose` as step 4, so new users know how to confirm the plugin is loaded and ready

- **Config schema warnings eliminated** — Generated `.swarm/config-example.json` now includes `fallback_models` for each agent, matching the installer template structure and suppressing schema validation warnings when users copy entries to their actual config

### Why

Users new to opencode-swarm reported confusion on four points (#594):
1. Which package manager to use (npm vs bun)
2. How to verify the plugin is running
3. Where to find and change model settings per project
4. How autonomous planning works and whether it truly runs without manual intervention

This release addresses all four through improved documentation, auto-discovery of config in the project directory, and a complete end-to-end example showing real autonomous workflow including the required TRANSITION confirmation.

### Migration

No breaking changes. Existing projects are unaffected. New installations will see `.swarm/config-example.json` appear in the project on first OpenCode session.

### Known Caveats

- `.swarm/config.example.json` is a reference file only — it is not loaded by the config system. Users must copy values to `.opencode/opencode-swarm.json` (project-local) or `~/.config/opencode/opencode-swarm.json` (global) to apply changes.
- The web app example assumes a basic React + TypeScript project. Behavior with other frameworks or project structures may differ.
