# v7.9.0

## What changed

### Command registration fixes
- Added `/swarm pr-review` and `/swarm issue` shortcuts to the command picker (were missing from `opencodeConfig.command` registration)
- Fixed `/swarm full-auto` template: dash separator replaced with space (`/swarm full-auto` → `/swarm full-auto`)

### Command-not-found UX overhaul
- Unknown commands now show a targeted error message with similarity-based suggestions instead of dumping the full help text
- Added empty-tokens guard: bare `/swarm` with no arguments still shows full help
- Added command name truncation (`MAX_DISPLAY=100`) to prevent long-input DoS in error messages
- Expanded `findSimilarCommands()` scoring: compound commands, token-level matching, dash-stripped comparison
- Added input length guard (`500` chars max) to prevent ReDoS in similarity scoring

### Error handling hardening
- **archive.ts**: `loadEvidence` wrapped in try/catch — corrupt evidence files no longer crash the archive command
- **benchmark.ts**: `loadEvidence` wrapped in try/catch — corrupt evidence files no longer crash benchmark cumulative mode
- **handoff.ts**: All file operations wrapped in try/catch with inline markdown fallback on failure; temp files cleaned up via `unlinkSync` if `renameSync` fails
- **dark-matter.ts**: `detectDarkMatter` wrapped in try/catch with user-friendly error message
- **simulate.ts**: `detectDarkMatter` wrapped in try/catch; report file write failures now logged via `warn()` instead of silently swallowed

### Error message consistency
- Standardized detectDarkMatter error wording across `simulate.ts` and `dark-matter.ts` (both now use "Error analyzing git history:")

### Observability improvements
- `archive.ts` and `benchmark.ts` now log diagnostic warnings (debug-gated via `OPENCODE_SWARM_DEBUG=1`) when skipping corrupt evidence files
- `simulate.ts` report write failures now logged via `warn()` (debug-gated)

## Why
The command system had three classes of issues discovered during a deep audit:
1. **Registration gaps**: Two commands (`pr-review`, `issue`) were registered in the command registry but had no shortcuts in the OpenCode command picker, making them undiscoverable in TUI/GUI
2. **Poor UX on unknown commands**: The fallback was to dump the entire help text (40+ commands), which is unhelpful and a potential DoS vector with long inputs
3. **Missing error handling**: Several command handlers would crash on corrupt evidence files, git errors, or file system failures instead of degrading gracefully

## Migration steps
None required. All changes are backward-compatible.

## Known caveats
- Diagnostic warnings for skipped evidence files are only visible when `OPENCODE_SWARM_DEBUG=1` is set
- Some test files from this work are placed in `src/commands/` rather than `tests/unit/commands/` (consistent with other command-level error handling tests in the project)
