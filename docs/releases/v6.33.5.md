# OpenCode Swarm v6.33.5 Release Notes

## Summary

Version **6.33.5** completes the session rehydration fix started in v6.33.4. While v6.33.4 refreshed stale timestamps on rehydrated InvocationWindows, it did not reset accumulated counters and the `hardLimitHit` flag — causing sessions to hit guardrail circuit breakers within seconds of a process restart.

---

## Bug Fixes

### Accumulated InvocationWindow counters and flags caused mid-session hangs after restart

Rehydrated InvocationWindows preserved their accumulated `toolCalls` count, `consecutiveErrors` count, `hardLimitHit` flag, and `recentToolCalls` buffer from the previous process. This caused:

- **`hardLimitHit = true`**: The circuit breaker check at guardrails line 737 immediately threw on every subsequent tool call, permanently blocking the agent with "🛑 CIRCUIT BREAKER: Agent blocked".
- **`toolCalls` near limit**: A coder at 395/400 tool calls would hit the `max_tool_calls` limit after just 5 more calls (~5-10 seconds of work).
- **`consecutiveErrors` near limit**: Could trigger the consecutive error circuit breaker almost immediately.
- **Stale `recentToolCalls`**: Old tool call history from hours ago caused false repetition detection.

**Fix:** `rehydrateState()` now fully resets all InvocationWindow counters and flags to their initial values on rehydration: `hardLimitHit = false`, `toolCalls = 0`, `consecutiveErrors = 0`, `recentToolCalls = []`, `warningIssued = false`, `warningReason = ''`. This is safe because a process restart means agents resume from scratch on the OpenCode side.

---

## Symptoms This Fixes

- Sessions hanging/getting stuck within 5-10 seconds of starting after a restart
- "🛑 CIRCUIT BREAKER: Agent blocked" errors appearing immediately after restart
- Agents hitting tool call or duration limits immediately despite being freshly started

---

## Migration Notes

- No config changes required
- Existing `.swarm/session/state.json` snapshots (v1 and v2) are fully compatible
- InvocationWindow counters are reset on first load — this is a safe operation
