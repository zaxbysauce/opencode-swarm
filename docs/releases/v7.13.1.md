# v7.13.1

## Fixed

- **Config override regression (issue #797)**: `opencode-swarm install` was writing full `DEFAULT_AGENT_CONFIGS` into the project-level config file (`.opencode/opencode-swarm.json`). Since the config loader deep-merges project config on top of global config with project config taking precedence, this silently overrode any custom models the user had set in their global `~/.config/opencode/opencode-swarm.json`. The project config is now initialized as minimal (`{ agents: {} }`) so the global config and schema defaults take effect as intended.

## Why

Users who configured custom models in their global config found those settings ignored when running `install` in a project. The project config is intended for project-specific overrides, not defaults.

## Migration steps

**For new installations:** No action required. Fresh `install` runs now write an empty project config and your global config takes effect automatically.

**For existing project configs already generated by a prior `install`:** The fix only applies to new installations — `writeProjectConfigIfMissing` skips existing files. If your `.opencode/opencode-swarm.json` was written by a prior install and contains the full default agent map, you must manually update it to use your global config:

Option 1 — Delete the project config and re-run install:
```bash
rm .opencode/opencode-swarm.json
opencode-swarm install
```

Option 2 — Replace the agents block with an empty object:
```json
{ "agents": {} }
```

After either option, your global `~/.config/opencode/opencode-swarm.json` models will take effect.
