# v7.7.0

## What changed

### feat(qa-gate): add final_council gate + derivePlanId utility + dead code removal

Adds runtime enforcement for the `final_council` gate — a holistic cross-cutting review that fires at project completion (last phase) via a new Gate 6 in `phase_complete`. Also extracts `derivePlanId` to a shared utility and removes dead sequential Stage B code.

**New tool: `write_final_council_evidence`**
Persists the final council verdict to `.swarm/evidence/final-council.json` with plan_id binding derived from the current plan. Normalizes verdict to lowercase. Fail-closed: missing or mismatched plan_id blocks phase completion.

**Gate 6 in `phase_complete`**
Runs only on the last phase of the plan. Checks for valid final council evidence with matching plan_id. Skipped when turbo mode is active. Error messages include actionable instructions.

**Shared `derivePlanId` utility**
Extracted to `src/plan/utils.ts` and re-exported from `src/plan/index.ts`. Replaces 6 private copies and 15 inline derivations across 14 files. Single canonical implementation prevents drift.

**Dead code removal**
Deleted 108 lines of unreachable sequential Stage B code from `delegation-gate.ts`. The parallel path has been the only active path since v7.0; the sequential branch was dead code.

**Pre-existing test fixes**
Fixed 3 gate test failures in `update-task-status.gates.test.ts` caused by PR #780's behavioral change: added minimal session state after `resetSwarmState()` so the gate doesn't hit the empty-sessions bypass.

## Why

Holistic council review at project completion was documented in the architect prompt but had no runtime enforcement — a project could advance without final council evidence. Gate 6 makes this check mandatory and fail-closed, with plan_id binding to prevent stale evidence from prior plans being reused.

The `derivePlanId` extraction eliminates 6 divergent private implementations across the codebase, preventing subtle hash inconsistencies. The dead sequential Stage B code removal reduces maintenance burden and eliminates confusion about which code path is active.

## Migration steps

None. The new gate is opt-in via the `final_council` flag in `set_qa_gates`. Existing projects without `final_council` enabled are unaffected.
