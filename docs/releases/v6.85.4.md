# Release 6.85.4

## What changed

Fixed architect agent prompt to properly exclude the designer agent when `ui_review` is disabled.

Previously, the architect's system prompt unconditionally listed the designer agent in six locations:
- The "Your agents" identity line
- The AGENTS section roster
- Rule 9 (UI/UX DESIGN GATE)
- A delegation example
- The execution pipeline step 5a
- A conditional note about UI design specs

However, the designer agent is only registered when `ui_review.enabled: true` is set in the plugin configuration. This mismatch caused architects to attempt delegation to `@designer`, which produced the runtime error: "designer is not a valid agent" (issue #645).

The fix mirrors the existing council feature pattern: `createArchitectAgent()` now accepts a `uiReview?: UIReviewConfig` parameter and strips all designer references from the rendered prompt when the feature is not enabled.

## Why

The designer agent is an opt-in feature. Users who do not enable it should not see it listed in the architect's capabilities, and the architect should never attempt to invoke it.

## Migration steps

No action required. The architect automatically detects whether `ui_review` is enabled in your plugin configuration and renders the correct prompt.

## Testing

Verified with 13 regression tests covering:
- Designer stripped when `ui_review` is not enabled (default)
- Designer stripped when `ui_review` is explicitly disabled
- Designer retained in prompt when `ui_review.enabled: true`
