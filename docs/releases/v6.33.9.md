# v6.33.9

## Bug fixes

### Session corruption from abandoned hook chains

The `toolAfter` hook chain was wrapped in a 30-second `Promise.race` timeout. When the timeout fired, the hook chain continued running in the background concurrently with the next tool call, corrupting shared session state (agent identity, delegation flags, evidence records). The timeout has been replaced with a direct `await` so hooks always run to completion before the next tool call begins.

### Guardrails skipped for returning Task tools

The Task handoff (which restores the orchestrator identity) was running *before* the hook chain, causing guardrails and the delegation gate to see `ORCHESTRATOR_NAME` instead of the subagent identity. Evidence was recorded against the wrong agent. The handoff now runs *after* the hook chain completes, so all hooks see the original subagent identity. The early-handoff workaround is no longer needed since `getEvidenceTaskId` was fixed to async in v6.33.2.

### Event loop blocking on Windows during evidence writes

`renameSync` in the evidence manager blocked the event loop when Windows virus scanners held file locks on the temp file. Both call sites now use `fs.promises.rename` (the containing functions were already async). Temp file cleanup in catch blocks is unchanged.
