# v7.2.0 Release Notes

## What Changed

### First-run UX and project config initialization

opencode-swarm now improves first-time setup by creating the expected project
configuration surface automatically.

- Auto-creates `.opencode/opencode-swarm.json` when opencode-swarm loads in a
  project directory that has no local config file.
- Writes the file atomically with exclusive create semantics, so concurrent
  plugin loads do not overwrite an existing config.
- Uses an empty JSON object (`{}`), which deep-merges as a no-op against any
  global config.
- Emits a one-line setup warning unless `quiet: true` is configured.
- Fixes `.swarm/config.example.json` creation for brand-new projects by ensuring
  `.swarm/` exists before writing the example file.

### Default architect and command discoverability

- Auto-selects the architect agent as primary with defensive fallback.
- Adds categorized `/swarm help [command]` output grouped by Core, Agent,
  Config, Diagnostics, and Utility commands.
- Adds deprecation aliases for legacy commands including `doctor`, `info`,
  `list-agents`, `health`, `check`, `clear`, `diagnosis`, `config-doctor`, and
  `evidence-summary`.
- Adds command alias infrastructure with `aliasOf` and `deprecated` registry
  fields, including module-load-time validation for circular references and
  missing targets.
- Generates `SKIP_ALIASES` dynamically from registry entries instead of
  maintaining a hardcoded list.

### Files Changed

- `src/config/project-init.ts` - new project config initialization helper.
- `src/index.ts` - calls project config initialization and fixes first-run
  `.swarm/` example creation.
- `src/commands/registry.ts` - command alias and deprecation metadata.
- `src/commands/help.ts` - categorized help output.
- `src/agents/architect.ts` - dynamic alias filtering in rendered command lists.
- `tests/unit/config/project-init.test.ts` - config initialization coverage.
- Command registry, help, and architect command-list tests covering aliases and
  deprecation behavior.

## Why

First-time users now get a discoverable project configuration file and clearer
command help without requiring a separate manual install step. Deprecated command
aliases continue to work while directing users toward canonical names.

## Migration Steps

None. Existing global-config-only setups continue to behave the same because the
auto-created project file is an empty JSON object. Deprecated aliases remain
backward-compatible and emit warnings.

## Breaking Changes

None.

## Known Caveats

- On read-only filesystems or containers without write access to the project
  root, the project config file will not be created. This is non-fatal and the
  plugin continues normally.
- The setup warning is suppressed when `quiet: true` is set.
- Schema defaults tests were updated to expect `default_agent: 'architect'` in
  parsed output.
