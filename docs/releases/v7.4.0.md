# v7.4.0 — Swarm Command Conflict Prevention

## What changed

**Three-layer defense against Claude Code command confusion inside swarm agent sessions:**

### Layer 1: Conflict Registry & CI Gate (Phase 1)

- **`src/commands/conflict-registry.ts`** — New pure data module defining a permanent machine-readable registry of 9 swarm commands that share names with Claude Code built-in slash commands (3 CRITICAL, 5 HIGH, 1 MEDIUM). Each entry includes severity, CC behavior, swarm behavior, and a disambiguation note.
- **`src/commands/conflict-registry.test.ts`** — CI gate with a ratchet mechanism. Adding a new CRITICAL command conflict without explicit acknowledgment causes CI to fail. 5 test groups covering completeness, disambiguation format, map integrity, and severity consistency.
- **`src/commands/registry.ts`** — `CommandEntry` type gains optional `clashesWithNativeCcCommand?: string` field, populated for all 9 conflicts. `buildHelpText()` renders ⚠️ conflict warnings for affected commands.
- **`src/config/constants.ts`** — New `CLAUDE_CODE_NATIVE_COMMANDS` frozen `ReadonlySet<string>` containing all known Claude Code built-in slash commands (105 unique entries), wrapped in a `freezeSet()` Proxy that blocks mutation at runtime.

### Layer 2: Prompt Hardening (Phase 2)

- **`src/agents/architect.ts`** — `## COMMAND NAMESPACE — CRITICAL` block injected before `IDENTITY`, listing all 8 CRITICAL+HIGH conflicts with NEVER/USE THIS language plus 3 banned commands (/clear, /compact, /memory).
- **`src/agents/coder.ts`** — `## COMMAND NAMESPACE` block listing 11 prohibited CC commands with DO NOT INVOKE language.
- **`src/agents/reviewer.ts`** and **`src/agents/test-engineer.ts`** — `## COMMAND NAMESPACE` blocks listing CRITICAL conflicts with PROHIBITED flags.
- **`.claude/skills/swarm/SKILL.md`** — Section renamed with disambiguation header; CRITICAL NAMING CONFLICTS table and COMMAND INVOCATION RULE added.
- **`.claude/skills/swarm-implement/SKILL.md`** — Command Namespace section with severity-accurate classifications.
- **`.claude/session/swarm-mode.md`** — Command Namespace section with 10-row mapping table.
- **`docs/commands.md`** — Full conflict table with severity icons and contributor instructions.
- **`README.md`** — Fixed broken link anchor to conflict registry section.

### Layer 3: Runtime Intercept Hook (Phase 3)

- **`src/hooks/cc-command-intercept.ts`** — New message pipeline hook that intercepts outgoing agent messages and:
  - **Hard-blocks** `/reset` and `/clear` via output mutation (replaces message content with structured advisory)
  - **Soft-corrects** `/plan` to `/swarm plan` with whitespace preservation
  - **Logs advisories** for HIGH severity commands
  - Skips code blocks, URLs, comments, `/swarm`-namespaced commands, and architect messages
  - Case-insensitive detection, idempotent via sentinel marker, <5ms processing target
- **`src/hooks/cc-command-intercept.test.ts`** — 23 unit tests covering all detection and intervention paths.
- **`tests/adversarial/cc-command-intercept.adversarial.test.ts`** — 22 adversarial tests across 8 attack vectors (case variation, whitespace, code fences, URLs, comments, multi-command, corrected-form bypass, case evasion).
- **`src/hooks/index.ts`**, **`src/index.ts`** — Hook exported and registered in `experimental.chat.messages.transform` pipeline.

## Why

Claude Code ships with 50+ built-in slash commands. AI models trained heavily on Claude Code carry those commands in their weights and will try to invoke them by bare name inside a swarm agent session — where the correct behavior is the fully-namespaced `/swarm <subcommand>` form. This can cause catastrophic context loss (`/reset` wipes conversation), enter unexpected modes (`/plan` blocks execution), or produce wrong output (`/status` shows CC version info instead of swarm state).

The three-layer approach provides defense in depth: the registry ensures new conflicts can't be added silently, prompt hardening prevents confusion at the context level, and the runtime hook catches invocations that slip through.

## Migration steps

No migration required. New features are purely additive:
- `clashesWithNativeCcCommand` is an optional field on `CommandEntry`
- `CLAUDE_CODE_NATIVE_COMMANDS` is a new exported constant
- The runtime hook is registered with default config (`{}`) and is inactive unless triggered
- Prompt blocks are additive text in existing template literals

## Known caveats

- CRITICAL `/checkpoint` is registered in the conflict data but receives no runtime hook intervention (neither blocked nor corrected). This is tracked in issue #740.
- FR-014 performance (<5ms) and idempotency tests are not yet implemented (tracked in issue #740).
- Adversarial tests pass 22/22; a broader security review is recommended before production use in high-risk environments.
