# v6.44.3

## What changed

### macOS /var→/private/var symlink resolution (67 test files)

On macOS, `os.tmpdir()` returns `/var/folders/...` but after `process.chdir(dir)`,
`process.cwd()` returns the symlink-resolved `/private/var/folders/...`. Any test
that stored the raw `mkdtempSync()` result and then compared it with `process.cwd()`
or passed it to a path-resolution function would fail with a path mismatch.

All test files that call `process.chdir(tempDir)` where `tempDir` comes from
`fs.mkdtempSync()` now use `fs.realpathSync(fs.mkdtempSync(...))` so both sides
resolve to the same canonical path.

Affected directories: `tests/unit/tools/`, `tests/unit/hooks/`,
`tests/unit/commands/`, `tests/unit/config/`, `tests/unit/evidence/`,
`tests/unit/quality/`.

### manager-write-marker test fix

Test 5 in `tests/unit/plan/manager-write-marker.adversarial.test.ts` made the
`.swarm/` directory read-only and expected the plan write to succeed. This is only
true when running as root (root ignores chmod restrictions). The test now:
- Skips entirely when `process.getuid?.() === 0` (running as root)
- Asserts `threw === true` for non-root, since a read-only `.swarm/` prevents
  `plan.json` from being written

### Windows: skip C:\\ path test that requires admin access

`save-plan-adversarial.test.ts` had a test that called `executeSavePlan` with
`working_directory: 'C:\\projects\\myworkspace'` and expected `success: true`.
On Linux/macOS this works because backslash is a valid filename character and
the path is created as a literal directory. On Windows CI, this resolves to the
real `C:\projects\` path which requires administrator write access not available
in the runner environment. The test is now skipped on `process.platform === 'win32'`.

## Why

These were pre-existing test failures that blocked removing `continue-on-error: true`
from the CI matrix. The goal is green CI on all three platforms (ubuntu, macOS,
Windows) without suppressing failures.

## Migration steps

None — test-only changes with no impact on the public API or runtime behaviour.

## Breaking changes

None.

## Known caveats

The `save-plan-adversarial` Windows-path test is skipped on Windows rather than
rewritten; the validation logic it tests (path traversal rejection, acceptance of
valid paths) is still covered by other tests that run on all platforms.
