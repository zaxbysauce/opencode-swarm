# OpenCode Swarm v6.20.0 Release Notes

## Summary

Version **6.20.0** introduced a major parallelism framework enabling concurrent task execution, AST-based diffing for code analysis, PR gate enforcement, checkpoint command extensions, agent output formatting, skill versioning, and context efficiency improvements.

---

## AST-Based Diffing

### Intelligent code change analysis

New AST (Abstract Syntax Tree) diffing capabilities replace naive line-based diffing for more accurate code change detection. This enables smarter incremental verification and more precise scope tracking during refactoring operations.

---

## Parallelism Framework

### Concurrent task execution

Introduced a task-level parallelism framework that allows multiple independent tasks to execute concurrently when configured. This significantly improves throughput for multi-task workflows.

### Task graph dependency resolution

Tasks now declare dependencies, and the framework ensures correct execution order while maximizing parallel opportunities where dependencies allow.

---

## PR Gate Enforcement

### Quality gate automation

PR gate checks are now enforced at the task level, blocking completion of tasks that fail required quality gates until violations are resolved.

---

## Checkpoint Extensions

### Enhanced session persistence

The checkpoint command now supports extended session state management, including branch state, working directory context, and agent progress snapshots for improved resumability.

---

## Agent Output Formatting

### Structured output improvements

Agent output now includes structured metadata for better integration with downstream tools and dashboards, including timing, token usage, and task correlation IDs.

---

## Skill Versioning

### Versioned skill registry

Skills now support semantic versioning, allowing teams to pin specific skill versions and manage upgrades independently from the core platform.

```jsonc
{
  "skills": {
    "coding": "1.2.0",
    "review": "^1.0.0"
  }
}
```

---

## Context Efficiency

### Optimized context management

Reduced token consumption through smarter context pruning, relevance-based filtering, and hierarchical summarization of older conversation segments.

---

## Migration Notes

- No breaking changes. All new features have backward-compatible defaults.
- AST diffing is enabled by default for new installations.
- Skill versioning requires updating skill references in configuration if using pinned versions.
