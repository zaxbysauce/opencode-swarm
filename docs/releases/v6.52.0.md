# v6.52.0 ‚Äî /swarm full-auto per-session toggle command

## What changed

### Fix: fullAutoMode is now a per-session toggle, not config-seeded

Full-Auto Mode was introduced in v6.51.0 with a design flaw: `fullAutoMode` was seeded from `swarmState.fullAutoEnabledInConfig` at session creation, meaning it required a config change and restart to activate. This release fixes that ‚Äî `fullAutoMode` now initializes to `false` on every new session, matching how `turboMode` works.

The `full_auto.enabled` config field is preserved and still controls the startup model-validation warning in `src/index.ts`. It no longer auto-activates sessions.

**Changed:** `src/state.ts` line 358 ‚Äî `fullAutoMode: false` (was `swarmState.fullAutoEnabledInConfig`)

### New: /swarm full-auto command

A new `/swarm full-auto [on|off]` command toggles Full-Auto Mode for the active session without requiring a config change or restart.

**Behavior:**
- `/swarm full-auto on` ‚Äî enables Full-Auto Mode for the session
- `/swarm full-auto off` ‚Äî disables Full-Auto Mode and resets interaction/deadlock counters (`fullAutoInteractionCount`, `fullAutoDeadlockCount`, `fullAutoLastQuestionHash`) for a clean re-enable cycle
- `/swarm full-auto` (no argument) ‚Äî toggles current state
- `swarm-full-auto` ‚Äî available in the OpenCode command picker with optional `[on|off]` argument

**New files:**
- `src/commands/full-auto.ts` ‚Äî `handleFullAutoCommand`, structurally equivalent to `turbo.ts` with counter-reset logic on disable
- `src/commands/full-auto.test.ts` ‚Äî 18 unit tests (toggle/on/off + 5 counter-reset-specific)
- `src/commands/full-auto-sessionid.test.ts` ‚Äî 7 empty/whitespace sessionID error path tests
- `src/commands/full-auto-registration.test.ts` ‚Äî 12 registration, routing, and help-text tests
- `src/commands/full-auto.regression.test.ts` ‚Äî 15 regression tests covering `hasActiveFullAuto()`, system-enhancer banner injection, and banner independence from turbo mode

**Updated files:**
- `src/commands/registry.ts` ‚Äî `'full-auto'` entry added after `turbo`
- `src/commands/index.ts` ‚Äî `handleFullAutoCommand` exported
- `src/index.ts` ‚Äî `swarm-full-auto` shortcut registered in OpenCode command picker
- `src/commands/shortcut-routing.test.ts` ‚Äî 2 shortcut routing tests added
- `README.md` ‚Äî `/swarm full-auto` row added to the slash commands table

## Why

Full-Auto Mode could only be enabled by editing the config file and restarting the session, making it impractical for ad-hoc use. The command parity with `/swarm turbo` was an explicit design goal that was not shipped in v6.51.0.

## Migration steps

No breaking changes. Sessions that previously relied on `full_auto.enabled: true` to auto-activate `fullAutoMode` will no longer see it auto-activated ‚Äî they must run `/swarm full-auto on` explicitly after starting a session. The startup model-validation warning is unaffected.

## Known caveats

None. The counter reset on disable is intentional: disabling full-auto and re-enabling starts a fresh interaction/deadlock cycle rather than continuing from where it left off.
