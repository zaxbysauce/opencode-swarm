# v6.57.0 (2026-04-08)

## Summary

This release implements the first three phases of the Structured Spec Enhancements plan, adding RFC 2119 keyword support, spec validation gates, spec staleness detection, and requirement-to-code coverage tracking.

## What Changed

### Architect Agent
- **Structured Spec Format** (`src/agents/architect.ts`, `src/commands/specify.ts`, `src/commands/clarify.ts`): Added SPECIFY and CLARIFY-SPEC modes that enforce RFC 2119 keywords (MUST/SHOULD/SHALL/MAY), FR-### requirement IDs, and Given/When/Then scenario format
- **Rule 6j. SPEC-GATE** (`src/agents/architect.ts`): save_plan now rejects plans without a .swarm/spec.md present
- **Rule 6k. SPEC-STALENESS GUARD** (`src/agents/architect.ts`): Architect must acknowledge spec drift via /swarm acknowledge-spec-drift before proceeding

### Spec Schema & Validation
- **Spec Schema** (`src/config/spec-schema.ts`): New Zod schemas for SwarmSpec, DeltaSpec, SpecRequirement, SpecScenario, SpecSection with FR-### pattern validation (/\bFR-\d{3}\b/)
- **validate_spec Tool** (`src/tools/validate-spec.ts`): Reads .swarm/spec.md, validates against schema, returns {valid, requirementCount, scenarioCount, errors, warnings, specMtime}

### Spec Staleness Guard
- **spec-hash.ts** (`src/utils/spec-hash.ts`): computeSpecHash() returns SHA-256 hex of spec.md; isSpecStale() compares current hash against plan.specHash
- **acknowledge-spec-drift Command** (`src/commands/acknowledge-spec-drift.ts`): /swarm acknowledge-spec-drift deletes spec-staleness.json and emits acknowledgment event
- **Events** (`src/types/events.ts`): Added SpecStaleDetectedEvent and SpecDriftAcknowledgedEvent types
- **Manager Integration** (`src/plan/manager.ts`): loadPlan now checks staleness and writes spec-staleness.json

### Requirement-to-Code Coverage
- **req_coverage Tool** (`src/tools/req-coverage.ts`): Extracts FR-### requirements, analyzes touched files, produces coverage report with covered/partial/missing categories
- **Critic Integration** (`src/agents/critic.ts`): Added STEP 3 REQUIREMENT COVERAGE to PHASE_DRIFT_VERIFIER_PROMPT
- **Evidence/Preflight** (`src/evidence/manager.ts`, `src/services/preflight-service.ts`): Preflight fails if spec.md exists but req-coverage report is missing
- **AGENT_TOOL_MAP** (`src/config/constants.ts`): Added validate_spec to architect, req_coverage to critic

### Bug Fixes
- **Architect Whitelist Tests** (`src/config/constants.architect-whitelist.test.ts`): Updated hardcoded tool counts to match current AGENT_TOOL_MAP state
- **Node Protocol** (`src/utils/spec-hash.ts`): Use node: protocol for builtin modules

## Why

These changes address:
- **Spec enforcement**: Plans should only be saved when a valid spec exists
- **Staleness detection**: Architects should know when spec diverges from plan
- **Requirement tracking**: Verify that FR requirements actually have corresponding code
- **Coverage visibility**: Missing requirements are flagged as CRITICAL blockers in drift verification

## Migration Steps

None. This is an additive change with no breaking API or behavior changes.
