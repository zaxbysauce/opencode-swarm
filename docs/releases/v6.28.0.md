# OpenCode Swarm v6.28.0 Release Notes

## Summary

Version **6.28.0** introduced the v7.0 beta testing pipeline, evidence guard hardening, and relaxed task granularity settings for improved workflow flexibility.

---

## v7.0 Beta Testing Pipeline

### Beta feature flags

New configuration system for opt-in beta features, enabling controlled rollout of v7.0 capabilities alongside stable v6 functionality.

```jsonc
{
  "beta": {
    "enabled": true,
    "features": ["new-curator", "turbo-mode"]
  }
}
```

### Beta testing feedback

Automated collection of beta feature usage telemetry for v7.0 readiness assessment.

---

## Evidence Guard Hardening

### Strengthened evidence validation

Evidence collection now includes cryptographic integrity checks to prevent tampering between collection and retrieval.

### Guardrail evidence enforcement

Evidence requirements are now enforced at phase boundaries, preventing incomplete evidence bundles from advancing to next phases.

---

## Relaxed Task Granularity

### Configurable task size thresholds

Task size thresholds are now configurable, allowing teams to define what constitutes a "small", "medium", or "large" task based on their workflow:

```jsonc
{
  "tasks": {
    "granularity": {
      "small": { "max_tools": 5, "max_files": 2 },
      "medium": { "max_tools": 15, "max_files": 10 }
    }
  }
}
```

### More flexible task decomposition

Relaxed default thresholds reduce unnecessary task fragmentation while maintaining quality gates.

---

## Migration Notes

- No breaking changes. Beta features are disabled by default.
- Task granularity settings are backward compatible — defaults match previous behavior.
- Evidence integrity checks are transparent to existing workflows.
