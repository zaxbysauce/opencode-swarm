# v6.58.0 (2026-04-08)

## Summary

This release implements the Structured Spec Enhancements feature, adding RFC 2119 keyword support, spec validation gates, spec staleness detection, requirement-to-code coverage tracking, and comprehensive QA fixes.

## What Changed

### Architect Agent
- **Structured Spec Format** (`src/agents/architect.ts`, `src/commands/specify.ts`, `src/commands/clarify.ts`): Added SPECIFY and CLARIFY-SPEC modes that enforce RFC 2119 keywords (MUST/SHOULD/SHALL/MAY), FR-### requirement IDs, and Given/When/Then scenario format
- **Rule 6j. SPEC-GATE** (`src/agents/architect.ts`): save_plan now rejects plans without a .swarm/spec.md present
- **Rule 6k. SPEC-STALENESS GUARD** (`src/agents/architect.ts`): Architect must acknowledge spec drift via /swarm acknowledge-spec-drift before proceeding

### Spec Schema & Validation
- **Spec Schema** (`src/config/spec-schema.ts`): New Zod schemas for SwarmSpec, DeltaSpec, SpecRequirement, SpecScenario, SpecSection with FR-### pattern validation (`/\bFR-\d{3}\b/`)
- **lint_spec Tool** (`src/tools/lint-spec.ts`): Reads .swarm/spec.md, lints for FR-IDs, obligations, and structure, returns {valid, requirementCount, scenarioCount, errors, warnings, specMtime}

### Spec Staleness Guard
- **spec-hash.ts** (`src/utils/spec-hash.ts`): computeSpecHash() returns SHA-256 hex of spec.md; isSpecStale() compares current hash against plan.specHash
- **acknowledge-spec-drift Command** (`src/commands/acknowledge-spec-drift.ts`): /swarm acknowledge-spec-drift deletes spec-staleness.json, updates plan.specHash to current spec hash, and emits acknowledgment event with previousHash/newHash audit trail
- **Events** (`src/types/events.ts`): Added SpecStaleDetectedEvent and SpecDriftAcknowledgedEvent types with previousHash/newHash fields
- **Manager Integration** (`src/plan/manager.ts`): loadPlan checks staleness, writes spec-staleness.json, and properly falls through with _specStale/_specStaleReason flags

### Requirement-to-Code Coverage
- **req_coverage Tool** (`src/tools/req-coverage.ts`): Extracts FR-### requirements, walks .swarm/evidence/ filtered by phase prefix, analyzes touched files, produces coverage report with binary covered/missing model
- **Critic Integration** (`src/agents/critic.ts`): Added STEP 3 REQUIREMENT COVERAGE to PHASE_DRIFT_VERIFIER_PROMPT
- **Evidence/Preflight** (`src/evidence/manager.ts`, `src/services/preflight-service.ts`): Preflight derives phase from loadPlan() and fails if spec.md exists but req-coverage report is missing
- **AGENT_TOOL_MAP** (`src/config/constants.ts`): Added lint_spec to architect whitelist, req_coverage to critic and critic_drift_verifier whitelists

### Evidence Wiring
- **write-drift-evidence** (`src/tools/write-drift-evidence.ts`): Added requirementCoverage optional field to tool schema, execute function, and evidence entry for persisting requirement coverage alongside drift verification results

### Bug Fixes
- **Architect Whitelist Tests** (`src/config/constants.architect-whitelist.test.ts`): Updated hardcoded tool counts to match current AGENT_TOOL_MAP state
- **Node Protocol** (`src/utils/spec-hash.ts`): Use node: protocol for builtin modules
- **Partial Coverage Model** (`src/tools/req-coverage.ts`, `src/agents/critic.ts`): Simplified from ternary (covered/partial/missing) to binary (covered/missing) model, eliminating ambiguous partial state

## Why

These changes address:
- **Spec enforcement**: Plans should only be saved when a valid spec exists
- **Staleness detection**: Architects should know when spec diverges from plan and can acknowledge drift to bless current state
- **Requirement tracking**: Verify that FR requirements actually have corresponding code
- **Coverage visibility**: Missing requirements are flagged as CRITICAL blockers in drift verification
- **Agent tool access**: critic_drift_verifier can now call req_coverage during phase drift verification

## Migration Steps

None. This is an additive change with no breaking API or behavior changes.

## Known Caveats

None.
