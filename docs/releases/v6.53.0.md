# v6.53.0 — /swarm full-auto biome compliance and integration test fix

## What changed

### Fix: integration test helper updated for v6.52.0 session initialization

`tests/integration/full-auto-mode.test.ts` — `startFullAutoSession()` was relying on the config-seeding behavior removed in v6.52.0. It set `swarmState.fullAutoEnabledInConfig = true` before `startAgentSession()`, expecting the session to auto-activate `fullAutoMode`. Since v6.52.0, `fullAutoMode` always initializes to `false`. The helper now sets `session.fullAutoMode = true` directly after session creation.

**Changed:** `tests/integration/full-auto-mode.test.ts:135-143` — `startFullAutoSession()` uses direct assignment instead of the removed config-seeding path.

### Fix: biome lint and format compliance across full-auto command files

Several full-auto files shipped in v6.52.0 had biome violations caught by subsequent CI runs:

- `src/commands/full-auto.ts` — multi-line ternary formatting
- `src/commands/index.ts` — `handleFullAutoCommand` export moved to correct alphabetical position
- `src/commands/full-auto-registration.test.ts` — import ordering (`handleFullAutoCommand` before `commandsIndex`); description assertion reformatted
- `src/commands/full-auto.regression.test.ts` — import ordering (`node:fs` before `node:os`); `biome-ignore lint/suspicious/noExplicitAny` suppressions added for `{} as any` hook constructor calls (mirrors `turbo.regression.test.ts` pattern)

## Why

The v6.52.0 biome violations were masked during development but surfaced on clean CI runs. The integration test helper regression was identified during independent PR review: removing config-seeding in v6.52.0 broke the helper without a corresponding test update.

## Migration steps

No changes to production code behavior. Test infrastructure and code style only.

## Known caveats

None.
