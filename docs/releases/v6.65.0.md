# v6.65.0

## What changed

### Repo Map / Code Graph (#475, closes #470)

A new structural-awareness layer for swarm agents. The graph is persisted at `.swarm/repo-graph.json` and queried via the new `repo_map` tool plus auto-injection into the coder and reviewer system prompts.

#### Phase 1 ŌĆö Graph core module
- Added `src/tools/repo-graph.ts` (~1485 lines) ŌĆö workspace dependency graph with node/edge model, safe load/save with atomic writes (tmp+rename with Windows retry), symlink boundary validation via `realpathSync`, cross-platform path normalization, module-level cache with mtime invalidation, and configurable file size/count limits
- Graph persisted at `.swarm/repo-graph.json` with schema versioning
- Import parsing supports: static imports, re-exports (`export { x } from './y'`, `export * from './y'`), dynamic `import()` expressions, `require()` calls
- Symbol extraction for TypeScript and Python exports

#### Phase 2 ŌĆö Hook wiring and automatic refresh
- Added `src/hooks/repo-graph-builder.ts` ŌĆö startup `init()` hook builds/refreshes graph when session starts; `toolAfter` hook incrementally updates graph when write tools modify `.ts`, `.tsx`, `.js`, `.jsx`, `.mjs`, `.cjs`, or `.py` files
- Dependency injection via `RepoGraphDeps` interface for test isolation (avoids `mock.module` global leakage)
- Wired into `src/index.ts` handler chain via `safeHook()` for non-blocking error handling

#### Phase 3 ŌĆö Security hardening
- Symlink boundary validation prevents workspace-escaping attacks (resolves symlinks before comparison)
- `containsPathTraversal` and `containsControlChars` validation on all inputs
- `validateSwarmPath` boundary checking for `.swarm/` paths
- Atomic temp+rename writes with `crypto.randomUUID()` temp filenames (prevents predictable tmpfile attacks)
- Windows rename retry (up to 3 attempts with 50ms delay) for EEXIST errors during handle release

#### Phase 4 ŌĆö Review council fixes
- Fixed `scanFile` call in `updateGraphForFiles` to pass raw file path
- Added workspace boundary validation in `toolAfter` hook
- `repo_graph` removed from tool registrations (no tool handler ŌĆö graph is maintained via hooks only)

#### Phase 5 ŌĆö Test suite stabilization
- DI pattern (`RepoGraphDeps`) eliminates `mock.module` global leakage
- Real temp directories for Windows compatibility (fake Unix paths fail boundary checks)
- `clearCache` calls for test isolation
- 208 tests across 11 test files (28 builder, 54 adversarial, 12 boundary, 18 wiring, 33 scan, 7 incremental, 3 atomic, 8 registration, 51+ main, plus Codex fix and adversarial files)

#### Post-merge Codex review fixes
- `validateWorkspace` now accepts absolute paths (previously rejected them, silently breaking the hook at runtime since `ctx.directory` is absolute)
- `resolveModuleSpecifier` now resolves extensionless imports (`.ts`, `.tsx`, `.js`, `.jsx`, `.mjs`, `.cjs`, `.py`, `.json`) ŌĆö previously produced dangling edges that didn't match graph node keys
- 60 additional verification + adversarial tests for the Codex fixes

#### Pre-existing lint fixes
- Fixed 17 files with biome lint warnings (`useLiteralKeys`, `useTemplate`)

## Why

The repo graph from #469 was wired but had critical runtime bugs: `validateWorkspace` rejected the absolute paths passed by the plugin runtime (`ctx.directory`), silently disabling the feature. Extensionless imports (`import { foo } from './utils'`) produced dangling graph edges. The review council (issue #474) identified security, architecture, and test quality concerns that needed addressing before the graph could be trusted for production use.

## Migration steps

No migration required. The graph is built automatically on session start and updated incrementally on file writes. Existing `.swarm/repo-graph.json` files from #469 are compatible.

## Breaking changes

None.

## Known caveats

- `repo-graph.ts` is a large module (~1485 lines) that could benefit from splitting into submodules
- Concurrent sessions updating the same workspace may experience lost graph updates (no inter-process locking)
- URL-encoded paths in tool arguments are not decoded before workspace boundary check
- Module-level graph cache can cause test pollution when boundary test files run in same process as other graph tests (27 failures in combined run, 0 failures individually)
- See issue #474 for post-hardening improvements identified by the review council
