# OpenCode Swarm v6.29.0 Release Notes

## Summary

Version **6.29.0** introduced self-correcting workflow hooks for automatic error recovery, context compaction for improved memory efficiency, and technical debt cleanup for long-term maintainability.

---

## Self-Correcting Workflow Hooks

### Automatic error recovery

New workflow hooks can now detect failure patterns and automatically trigger corrective actions without human intervention:

- Automatic retry with exponential backoff for transient failures
- Automatic scope correction when tasks drift outside declared boundaries
- Automatic test re-run for flaky test detection

### Hook composition

Hooks can be chained and composed, allowing complex recovery strategies:

```jsonc
{
  "workflow": {
    "hooks": [
      { "trigger": "test_failure", "action": "auto_retry", "max_attempts": 3 },
      { "trigger": "scope_drift", "action": "auto_correct" }
    ]
  }
}
```

---

## Context Compaction

### Intelligent context summarization

Context is now automatically compacted when approaching token limits. The compaction algorithm prioritizes recent relevant context while summarizing older interactions.

### Configurable compaction triggers

Set custom thresholds for when compaction activates:

```jsonc
{
  "context": {
    "compaction": {
      "threshold_percent": 80,
      "min_age_turns": 10
    }
  }
}
```

---

## Technical Debt Cleanup

### Deprecated API removal

Removed deprecated APIs from v6.18 and earlier. See migration guide in [CHANGELOG.md](https://github.com/opencode-ai/opencode-swarm/blob/main/CHANGELOG.md) for required updates.

### Internal consistency improvements

Refactored internal type definitions for improved type safety and reduced runtime errors.

### Dependency updates

Updated core dependencies to latest stable versions with security patches.

---

## Migration Notes

- **Breaking change**: Deprecated APIs removed. Review [CHANGELOG.md](https://github.com/opencode-ai/opencode-swarm/blob/main/CHANGELOG.md) for removed symbols.
- Workflow hooks require explicit configuration to activate — no behavior change unless configured.
- Context compaction is enabled by default with conservative thresholds.
