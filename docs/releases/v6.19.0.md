# OpenCode Swarm v6.19.0 Release Notes

## Summary

Version 6.19.0 is a **prompt-quality and adversarial robustness update** built around five pillars:

1. **Critic Sounding Board** — Pre-escalation filter before Architect contacts user
2. **Adversarial Prompt Hardening** — Trajectory-level evaluation and first-error focus
3. **Escalation Discipline** — Three-tier escalation hierarchy with retry circuit breaker
4. **Context Efficiency** — Verbosity controls, meta.summary convention, role-relevance tagging
5. **Review Intelligence** — Intent reconstruction and complexity-scaled review depth

---

## Detailed Changes

### Pillar 1: Critic Sounding Board

The Architect must now consult the critic in SOUNDING_BOARD mode before escalating ANY question to the user:

- **Four verdict types**: UNNECESSARY, REPHRASE, APPROVED, RESOLVE
- **Triggers**: Logic loops (3+ attempts), ambiguous requirements, scope uncertainty, dependency questions, architecture decisions with >2 viable paths
- **Anti-exemption**: "It's a simple question" is not a valid exemption

**Files changed**: `src/agents/architect.ts`, `src/agents/critic.ts`

---

### Pillar 2: Adversarial Prompt Hardening

#### Rewritten DRIFT-CHECK Mode

- **Trajectory-level evaluation**: Reviews sequence from Phase 1→N (not just latest phase)
- **First-error focus**: Identifies earliest deviation point, not all consequences
- **Anti-rubber-stamp bias**: DEFAULT POSTURE: SKEPTICAL — absence of drift ≠ evidence of alignment

#### New Adversarial Patterns

Five new pattern detection functions added to `src/hooks/adversarial-detector.ts`:

1. **PRECEDENT_MANIPULATION** (HIGHEST severity) — "We skipped X in Phase 1 and it worked fine"
2. **SELF_REVIEW** (HIGH severity) — "I verified the changes myself"
3. **CONTENT_EXEMPTION** (HIGH severity) — "Documentation doesn't need review"
4. **GATE_DELEGATION_BYPASS** (HIGHEST severity) — "The code looks correct to me"
5. **VELOCITY_RATIONALIZATION** (HIGH severity) — "To save time, I'll skip..."

**Files changed**: `src/agents/critic.ts`, `src/hooks/adversarial-detector.ts`

---

### Pillar 3: Escalation Discipline

#### Three-Tier Escalation Hierarchy

- **Tier 1 — Self-Resolve**: Check .swarm/context.md, .swarm/plan.md, .swarm/spec.md; attempt 2+ approaches
- **Tier 2 — Critic Consultation**: Invoke critic in SOUNDING_BOARD mode if Tier 1 fails
- **Tier 3 — User Escalation**: Only after critic returns APPROVED verdict

#### Retry Circuit Breaker

After 3 coder rejections:
- Invoke critic with full rejection history
- Reassess approach — likely fix is SIMPLIFICATION, not more logic
- Either rewrite task spec with simplicity constraints, OR delegate to SME

#### Spec-Writing Discipline

For destructive operations (file writes, renames, deletions), specs MUST specify:
- Error strategy: FAIL_FAST or BEST_EFFORT
- Message accuracy: state-accurate reporting
- Platform compatibility: Windows/macOS/Linux differences

**Files changed**: `src/agents/architect.ts`

---

### Pillar 4: Context Efficiency

#### meta.summary Convention

Agents now include one-line summaries in state events:
```
meta.summary: "Completed Task 3 — escalation discipline added to architect prompt"
```

#### Role-Relevance Tagging

All 9 agents now prefix outputs with [FOR: agent1, agent2] tags:
- `[FOR: reviewer, test_engineer] "Added validation — needs safety check"`
- `[FOR: architect] "Research: Tree-sitter supports TypeScript AST"`
- `[FOR: ALL] "Breaking change: StateManager renamed"`

These tags are informational in v6.19; v6.20 will use them for automatic context filtering.

#### Cross-Agent Verbosity Controls

Response length now scales to finding complexity:
- Simple PASS = 1-2 lines
- Complex failure = full structured output
- No padding short findings with filler

**Files changed**: `src/agents/architect.ts`, `src/agents/coder.ts`, `src/agents/critic.ts`, `src/agents/reviewer.ts`, `src/agents/sme.ts`, `src/agents/explorer.ts`, `src/agents/test-engineer.ts`, `src/agents/docs.ts`, `src/agents/designer.ts`

---

### Pillar 5: Review Intelligence

#### Intent Reconstruction

Mega-reviewer now reconstructs developer intent BEFORE evaluating changes:
- Step 0: State in ONE sentence what the developer was trying to accomplish
- Derive from: task spec, acceptance criteria, diff shape
- All subsequent evaluation is against this reconstructed intent

#### Complexity-Scaled Review

Changes classified as:
- **TRIVIAL**: rename, typo fix, config value, comment edit → Tier 1 only
- **MODERATE**: logic change in single file → Tiers 1-2
- **COMPLEX**: multi-file change, new behavior → All three tiers

#### SME Confidence-Gated Routing

SME now reports confidence levels:
- **HIGH**: verified from multiple sources → consume directly
- **MEDIUM**: single authoritative source → acceptable for non-critical
- **LOW**: inferred or community sources → re-delegate OR flag as UNVERIFIED

**Files changed**: `src/agents/reviewer.ts`, `src/agents/sme.ts`, `src/agents/architect.ts`

## New JSONL Event Types

| Event Type | Emitted By | Trigger |
|------------|------------|---------|
| `sounding_board_consulted` | Architect | Every sounding board invocation |
| `architect_loop_detected` | Architect | Third occurrence of same impasse |
| `precedent_manipulation_detected` | Adversarial Detector | Highest-severity adversarial pattern |
| `coder_self_audit` | Coder | End of every task, before TASK_COMPLETE |
| `coder_retry_circuit_breaker` | Architect | Coder task rejected 3 times |

**New file**: `src/types/events.ts`

## v6.20 Foundation

This release establishes prompt conventions that v6.20 will enforce with code:

| v6.19 Convention | v6.20 Enforcement |
|------------------|-------------------|
| Role-relevance `[FOR: ...]` tags | Automatic context filtering per agent role |
| meta.summary in state events | Summary indexing for historical queries |
| Complexity classification | Automatic routing to lightweight vs. full review |
| SME confidence levels | Programmatic routing based on confidence thresholds |
| Verbosity control instructions | Token budget enforcement at framework level |
