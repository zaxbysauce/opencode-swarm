# Release Notes — v6.41.1

**Patch Release | 2026-03-30**

---

## Overview

This release resolves 6 pre-existing test failures accumulated across v6.23–v6.40 development, plus 4 additional stale test files discovered during investigation. It also fixes a path authority enforcement bug where absolute file paths caused all coder write operations to be blocked, even when the target file was within allowed directories.

- **State machine tests**: Fixed stale assertions for `isValidTaskId()` guard behavior (null, undefined, empty, numeric taskIds)
- **Rehydration tests**: Fixed evidence setup that triggered wrong state machine path; fixed `schema_version` fixture mismatch
- **Version bump test**: Replaced brittle hardcoded version string with semver format validation
- **Reviewer gate test**: Fixed missing `workingDirectory` parameter in all `checkReviewerGate` calls
- **Critic prompt tests**: Rewrote ~60 assertions across 4 test files to match post-refactor `PHASE_DRIFT_VERIFIER_PROMPT` architecture
- **checkFileAuthority**: Absolute paths now correctly normalized to project-relative before prefix matching
- **declare_scope**: Absolute paths normalized at declaration time with warnings; out-of-project paths rejected
- **isInDeclaredScope**: Now resolves paths against the project directory instead of `process.cwd()`

Reported in #259 by @asakura42, root cause analysis by @LordVandal1976.

---

## Bug Fixes

### Defensive typeof guard in isValidTaskId (`src/state.ts`)

Added `typeof taskId !== 'string'` check before calling `.trim()`. Previously, passing a non-string value (e.g., a number via `as any`) at runtime would throw a `TypeError` instead of being gracefully rejected. This aligns with the function's existing defensive design for null/undefined inputs.

### State rehydration test fixture (`state-rehydrate.test.ts`)

- Fixed evidence setup that provided both `reviewer` and `test_engineer` gates when the test intended to verify the `tests_run` path (only `test_engineer` present). The `allPassed` short-circuit was returning `'complete'` instead.
- Fixed `schema_version: '1.0'` → `'1.0.0'` to match `PlanSchema`'s `z.literal('1.0.0')` requirement, which was causing all 17 rehydration tests to silently fail via Zod validation rejection.

### Version bump test (`version-bump.test.ts`)

Removed hardcoded `expect(pkg.version).toBe('6.22.19')` that had drifted across 19 releases. Replaced with semver format validation (`/^\d+\.\d+\.\d+$/`) that won't break on future releases.

### Reviewer gate test (`phase-complete-fix-adversarial.test.ts`)

All `checkReviewerGate()` calls now pass `tempDir` as the `workingDirectory` parameter. Previously, calls without this parameter caused `path.join(undefined, ...)` to throw a `TypeError`, which was caught as "corrupt evidence" and masked the actual test logic.

### Critic prompt test rewrites (4 files)

Rewrote all DRIFT-CHECK test assertions to target the new `PHASE_DRIFT_VERIFIER_PROMPT` architecture introduced in v6.40. Tests now use `createCriticAgent('test-model', undefined, undefined, 'phase_drift_verifier')` and verify the 4-axis rubric (File Change, Spec Alignment, Integrity, Drift Detection), per-task verdicts (VERIFIED/MISSING/DRIFTED), and phase verdict (APPROVED/NEEDS_REVISION).

### Path authority enforcement breaks on absolute paths (`src/hooks/guardrails.ts`)

`checkFileAuthority` used string prefix matching (`startsWith('src/')`) against relative prefixes, but the file path parameter could be an absolute path (e.g., `C:/Users/dev/project/src/file.ts` or `/home/dev/project/src/file.ts`). Since absolute paths never start with `src/`, every coder write was blocked with `"Path not in allowed list for coder"`.

**Fix**: The function now resolves the incoming path to absolute via `path.resolve(cwd, filePath)`, then converts back to project-relative via `path.relative(cwd, resolved)` before prefix comparison. This is fully backward-compatible — relative paths pass through unchanged.

### declare_scope silently accepts absolute paths (`src/tools/declare-scope.ts`)

`declare_scope` accepted and stored absolute paths without error or warning. The scope declaration appeared successful, but downstream `checkFileAuthority` calls would block every write — with no indication that path format was the problem.

**Fix**: Absolute paths are now normalized to project-relative at declaration time. A `warnings` array in the response notifies the caller of each normalization. Absolute paths that resolve outside the project directory are rejected with an explicit error message.

### isInDeclaredScope resolves against process.cwd() instead of project directory (`src/hooks/guardrails.ts`)

The post-execution scope validation function `isInDeclaredScope` used `path.resolve(filePath)` without a base directory, resolving against `process.cwd()` instead of the project's working directory. If `process.cwd()` differed from the project directory, scope containment checks could produce incorrect results.

**Fix**: `isInDeclaredScope` now accepts an optional `cwd` parameter, and the call site passes the project `directory` from the `createGuardrailsHooks` closure.

None. The `DeclareScopeResult` interface gained an optional `warnings?: string[]` field, which is additive only.
