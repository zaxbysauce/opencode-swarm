# v6.56.0

## What changed

### Issue #401 — Post-v6.48.0 Reliability Backlog (5 items)

1. **Configurable knowledge injection threshold**: Added `context_budget_threshold` to `KnowledgeConfig` interface and Zod schema, allowing users to configure the maximum context chars before the knowledge injector skips injection.

2. **Runtime capability snapshots for agents**: `getAgentConfigs()` now writes a per-agent tool snapshot to `.swarm/evidence/agent-tools-{sessionId}.json` (non-blocking). A new `Agent Tool Snapshots` informational check was added to the diagnose service.

3. **Granular failure classification in `createSwarmTool`**: The catch block now returns a `failure_class` field (`not_registered`, `not_whitelisted`, `binary_missing`, or `execution_error`) to help diagnose tool invocation failures.

4. **Pre-flight binary readiness advisory**: A new `getBinaryReadinessAdvisory()` function in `tool-doctor.ts` generates an advisory string for missing Class 3 binaries. This is injected into the architect's system prompt at session start (architect-only, non-fatal).

5. **snake_case normalization for knowledge tool names**: Renamed `knowledgeAdd`, `knowledgeRecall`, `knowledgeRemove` to `knowledge_add`, `knowledge_recall`, `knowledge_remove` across the entire codebase. Added a compile-time `AssertSnakeCase` type assertion to prevent future camelCase tool names.

### Issue #398 — Session-Level Environment Profiling

1. **Phase 1 — Detection and storage**: New `src/environment/profile.ts` module detects `HostOS`, `ShellFamily`, `ExecutionMode`, and `OperatingMode` once per session. Profile is stored in `swarmState.environmentProfiles` and a new `environment_detected` telemetry event is emitted.

2. **Phase 2 — Prompt injection**: New `src/environment/prompt-renderer.ts` renders environment-specific prompt blocks for `coder` and `test_engineer` agents. Injected via `system-enhancer.ts` with proper agent-name guards.

3. **Phase 3 — Tool context propagation**: `SwarmToolOptions` now documents that `EnvironmentProfile` is available to any tool via `getSessionEnvironment(ctx?.sessionID)`.

4. **Phase 4 — Barrel export**: New `src/environment/index.ts` provides clean re-exports for all environment types and functions.

## Why

- **#401**: These five items were deferred from the #395 audit. They harden tool diagnostics, improve runtime observability, and enforce naming consistency.
- **#398**: Windows, Docker, and WSL environments receive incorrect POSIX command suggestions. Session-level profiling enables environment-aware prompts and command policies.

## Migration steps

- **Tool name change**: If any external configuration or scripts reference `knowledgeAdd`, `knowledgeRecall`, or `knowledgeRemove`, update them to `knowledge_add`, `knowledge_recall`, `knowledge_remove`. The runtime validation in `constants.ts` will throw at startup if AGENT_TOOL_MAP contains unregistered names.
- **New config field**: `context_budget_threshold` is optional with no behavior change if omitted.

## Breaking changes

- Tool names `knowledgeAdd`, `knowledgeRecall`, `knowledgeRemove` have been renamed to `knowledge_add`, `knowledge_recall`, `knowledge_remove`. Any `tool_filter.overrides` in plugin config referencing the old names must be updated.

## Known caveats

- Agent tool snapshots are written at plugin init time with a timestamp-based ID (`init-<timestamp>`) since no session exists yet. Per-session snapshots require call sites that have a sessionId.
