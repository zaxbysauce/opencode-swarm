# Release Notes — v6.34.0

**Minor Release | 2026-03-26**

---

## Overview

This release introduces significant quality and reliability improvements to the Critic workflow, phase completion gates, and observability. No breaking changes require migration.

- **Agent count**: 9 → 11 (replaced 1 monolithic critic with 3 specialized critics)
- **New tools**: `completion-verify`, `telemetry` (structured event emitter)
- **Gates strengthened**: `phase_complete` now enforces deterministic verification + drift checking (bypassed in turbo mode)
- **Bug fixes**: 7+ reliability fixes across tests and runtime flows

---

## New Features

### Triple Critic Architecture
The single Critic agent is now three specialized critics with focused rubrics:

- **plan_critic** — Reviews task plans against 5-axis rubric (clarity, edge cases, security, etc.)
- **critic_sounding_board** — Conversational advisor for iterative design feedback
- **critic_drift_verifier** — Validates phase alignments against spec with 4-axis rubric

This separation improves review quality and allows parallel delegation.

### Deterministic Phase Completion Verify
New `completion-verify` tool enforces that plan task identifiers (functions, classes, config keys) are present in source files before marking a phase complete. This catches specification drift early.

- Parses identifiers from task descriptions (backticks, camelCase, PascalCase, config keys)
- Blocks phase completion if zero matches found
- Auto-bypasses in turbo mode

### Telemetry System
Structured, append-only telemetry emitter writes JSON lines to `.swarm/telemetry.jsonl`. Tracks:

- Session lifecycle, task status changes, gate evaluations
- Delegations, model fallbacks, circuit breaker trips
- Budget alerts, scope violations, turbo mode usage
- 15+ convenience methods integrated across runtime

Telemetry is non-throwing by design; write errors are caught and logged without disrupting workflow.

## Bug Fixes

- **Evidence gate robustness**: `checkReviewerGate` now uses `readTaskEvidenceRaw` to avoid malformed JSON crashes and handle missing evidence correctly (ENOENT → session state, corrupt → blocked with clear error).
- **Curator pipeline crash**: Fixed `driftResult.report.drift_score` optional chaining.
- **Test stability**: Fixed 7 pre-existing test failures (curator mock shape, events isolation, auto-checkpoint gate evidence).
- **Completion-verify identifier parsing**: Correctly skips (does not block) tasks with no parseable identifiers per specification.
