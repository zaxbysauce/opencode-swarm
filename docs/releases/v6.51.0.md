# v6.51.0 — Full-Auto Mode

## What changed

### Full-Auto Mode: autonomous oversight critic

Full-Auto Mode routes all architect-to-human interactions to an autonomous oversight critic instead of a human, enabling fully autonomous swarm operation with quality gates. When `full_auto.enabled: true` is set in the plugin config, the swarm can run without human intervention for extended periods.

**Core components:**

- **`full_auto` config field** (`src/config/schema.ts`): `enabled` (default false), `critic_model` (optional override), `max_interactions_per_phase` (5-200, default 50), `deadlock_threshold` (2-10, default 3), `escalation_mode` (`pause` or `terminate`)

- **`hasActiveFullAuto()`** (`src/state.ts`): Returns true when the session has `fullAutoMode: true`. Model validation is advisory-only — a warning is logged at startup if models match, but full-auto activation is not blocked.

- **`AUTONOMOUS_OVERSIGHT_PROMPT`** + **`createCriticAutonomousOversightAgent()`** (`src/agents/critic.ts`): A dedicated oversight prompt (89 lines) with a 10-rule CONSTITUTION, VERIFICATION PROTOCOL for plan/task/phase completion and question resolution, SELF-CRITIQUE, and ANTI-PATTERNS. The agent factory creates a read-only critic agent invoked dynamically and registered via the agent factory.

- **Intercept layer** (`src/hooks/full-auto-intercept.ts`): A hook factory that activates only when `hasActiveFullAuto()` returns true. Detects escalation patterns:
  - Phase completion: `Ready for Phase N+1?` / `Ready for Phase [N+1]?` / `Ready for Phase 2?`
  - Question at end of output: standalone `?`
  - Escalation keywords: `escalat`, `What would you like`, `Should I proceed`, `Do you want`
  - Filters mid-sentence question marks in code: `v1?`, `v1.2?`, `y?.z`, `API?`, `OK?`, `5?10`
  - Only fires when architect's turn ended without a tool call

- **Escalation backstop**: When interaction count reaches `max_interactions_per_phase` or deadlock count reaches `deadlock_threshold`, an escalation report is written to `.swarm/escalation-report.md`. In `pause` mode: event emitted + session paused. In `terminate` mode: `process.exit(1)`.

- **Audit trail** (`events.jsonl`): Every critic dispatch appends a JSONL entry with `type: 'auto_oversight'`, timestamp, interaction mode, architect output, critic verdict, and counts.

- **Full-auto banner** (`src/hooks/system-enhancer.ts`): Banner injected at runtime when full_auto is enabled. No placeholder in architect prompt — the banner is injected into the system prompt by the system-enhancer hook.

### Deadlock detection behavior

With `deadlock_threshold: 2`, **3 identical questions** are required to trigger escalation — not 2. First occurrence sets the question hash (count=0), second repeat increments to 1, third repeat increments to 2 and escalates.

## Why

The swarm was designed for human-in-the-loop operation. Full-Auto Mode addresses scenarios where extended autonomous operation is needed with preserved quality gates — the oversight critic reviews architect decisions without requiring human confirmation for every interaction.

## Migration steps

No breaking changes. The `full_auto` config field defaults to `enabled: false`, so existing configurations are unaffected. When enabling full_auto, set `critic_model` to a different model than the architect uses.

## Known caveats

- Model validation is advisory-only. If the critic and architect models match, a warning is logged at startup but full-auto activation proceeds normally.
