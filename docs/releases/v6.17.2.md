# OpenCode Swarm v6.17.2 Release Notes

## Summary

Version 6.17.2 delivers **CLI Plugin Command Runner** — a patch release fixing a discoverability gap where AI models operating outside an active OpenCode session had no standard way to invoke plugin commands.

- **`run` subcommand**: Added `bunx opencode-swarm run <command> [args]` CLI entry point that dispatches 22+ plugin commands by argv
- **Missing help text**: Fixed absent `knowledge migrate` entry in the HELP_TEXT array despite the handler being fully implemented since v6.17.1
- **Architect prompt fix**: Added CLI invocation note to prevent AI models from attempting internal path lookups that fail in user project directories

---

## Detailed Fixes

### 1. `run` Subcommand Wiring

**Problem**: `src/commands/` only exists in the plugin source tree — not in user project directories. AI models (e.g., Minimax M2.5) operating outside an active OpenCode session had no standard way to invoke plugin commands. Models attempting to call commands directly would fail with path resolution errors.

**Fix**: Added `export async function run(args: string[]): Promise<number>` in `src/cli/index.ts` that dispatches 22+ plugin commands by argv. Routes all existing `/swarm` commands: status, plan, agents, archive, history, config, config doctor, evidence, evidence summary, diagnose, preflight, sync-plan, benchmark, export, reset, retrieve, clarify, analyze, specify, dark-matter, knowledge migrate/quarantine/restore. Returns exit code 0 on success, 1 on unknown command or empty args. Wired into `main()` switch and `printHelp()` updated with `run` in Commands list and usage examples.

---

### 2. Missing `knowledge migrate` in Help Text

**Problem**: `knowledge migrate` was absent from HELP_TEXT array in `src/commands/index.ts` despite the handler being fully implemented since v6.17.1. Users running `/swarm help` would not see the command listed.

**Fix**: Added `- \`/swarm knowledge migrate\` — Migrate knowledge entries to the current format'` to the HELP_TEXT array in `src/commands/index.ts`.

---

### 3. Architect Prompt CLI Invocation Note

**Problem**: The architect agent's SLASH COMMANDS section in `src/agents/architect.ts` did not document how to invoke commands outside an OpenCode session. AI models would attempt to use `bun -e` or look for `src/commands/` in user project directories, which would fail.

**Fix**: Added a one-sentence note to the SLASH COMMANDS section: use `bunx opencode-swarm run <command>` outside OpenCode; do not use `bun -e` or look for `src/commands/`. Prevents models from attempting internal path lookups that fail in user project directories.

---

## Upgrade Notes

- **Backward compatible**: All existing behavior preserved. No breaking changes.
- **`run` subcommand**: AI models can now invoke plugin commands outside OpenCode sessions via `bunx opencode-swarm run status`, `bunx opencode-swarm run knowledge migrate`, etc.
- **No migration required**: This is a patch release with no data migration needed.
