# Release Notes — v6.35.2

**Patch Release | 2026-03-27**

---

## Overview

This patch release eliminates `critic_drift_verifier` as a separately registered agent. Drift verification is now handled by the existing `critic` agent (via `createCriticDriftVerifierAgent`) or by the curator's built-in drift check. This removes an unnecessary agent registration that could cause `ProviderModelNotFoundError` deadlocks on phase_complete.

- **Eliminated critic_drift_verifier agent**: Removed from agent index, delegations, and constants
- **Curator drift evidence**: Curator now writes `drift-verifier.json` directly during phase drift checks
- **Softer drift gate**: Missing drift evidence no longer hard-blocks phase completion when spec.md is absent

---

## Changes

- **critic.ts**: Added `createCriticDriftVerifierAgent()` factory function that returns an agent with name `'critic'` (not `'critic_drift_verifier'`) using the `PHASE_DRIFT_VERIFIER_PROMPT`. This is for internal use — not registered in index.ts.
- **agents/index.ts**: Removed the entire `critic_drift_verifier` registration block (5c). Added TODO comment noting `critic_sounding_board` has the same `ProviderModelNotFoundError` risk but is not on the phase_complete critical path.
- **delegation-gate.ts**: Removed `critic_drift_verifier` from `gateAgents` array. Removed `writeDriftVerifierEvidence` function.
- **curator-drift.ts**: Added drift-verifier.json evidence writing to `runCriticDriftCheck`. The curator now writes `.swarm/evidence/{phase}/drift-verifier.json` with the schema expected by phase_complete: `{ entries: [{ type: 'drift-verification', verdict: 'approved'|'rejected', summary: string, timestamp: string }] }`. Mapping: ALIGNED→approved, MINOR_DRIFT→approved with warning, MAJOR_DRIFT→rejected.
- **phase-complete.ts**: Softened the drift gate — when drift-verifier.json is missing AND spec.md does not exist, the gate degrades to an advisory warning instead of hard-blocking.
- **constants.ts**: Removed `critic_drift_verifier` from `QA_AGENTS`, `AGENT_TOOL_MAP`, and `DEFAULT_MODELS`.
- **agent-categories.ts**: Removed `critic_drift_verifier` entry from `AGENT_CATEGORY`.
- **architect.ts**: Updated architect prompt to reference critic (not critic_drift_verifier) for drift verification. Updated step 5.5 and 5.6 descriptions.

---

## Migration Notes

- The `critic_drift_verifier` agent name no longer exists in `QA_AGENTS`, `ALL_AGENT_NAMES`, `AGENT_TOOL_MAP`, `DEFAULT_MODELS`, or `AGENT_CATEGORY`.
- The `CriticRole` type still includes `'phase_drift_verifier'` — this is used internally by `createCriticAgent` and `createCriticDriftVerifierAgent`.
- `createCriticAgent` with `role='phase_drift_verifier'` still works and returns name `'critic_drift_verifier'` — this behavior is unchanged.
- The new `createCriticDriftVerifierAgent()` returns name `'critic'` and is used internally by the curator drift check.
- Drift evidence (drift-verifier.json) is now written by the curator's drift check instead of a separate agent delegation.
- Phase completion: When spec.md is absent, missing drift evidence only produces a warning, not a hard block.
