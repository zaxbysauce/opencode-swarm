## v7.21.1 — Guardrail enforcement: block architect rm-bypass via mv/move/ren

### What changed
- **`checkDestructiveCommand()` in guardrails.ts**: Added 7 new guardrail sections (16–22) that block shell commands targeting `.swarm/` paths:
  - POSIX `mv` — blocked when source or destination references `.swarm/`
  - Windows cmd `move`/`ren` — blocked for `.swarm\` paths
  - PowerShell `Move-Item`/`Rename-Item` (and aliases) — blocked for `.swarm/` paths
  - Non-recursive `rm` — blocked for `.swarm/` files (even without `-rf` flags)
  - `cp` + `rm` chains — secondary defense (the `rm` half is independently blocked by the non-recursive `rm` guard)
  - Archive tools with delete-source flags (`rsync --remove-source-files`, `tar --remove-files`, `zip -m`, `7z -sdel`) — blocked when targeting `.swarm/`
  - Git `clean -fd` / `worktree remove --force` — verified existing patterns already cover `.swarm/`
- **Architect prompt** (`src/agents/architect.ts`): Added explicit prohibition on using `mv`/`Move-Item`/`move`/`ren`/`cp+rm` chains as shell bypasses for blocked destructive commands (closes issue #859)

### Why
Issue #859 reported that the swarm architect bypasses `rm` blocks by using `mv` to relocate evidence files under `.swarm/`. The `checkDestructiveCommand()` function blocked `rm -rf` but did not block `mv`, `move`, `ren`, `Move-Item`, `Rename-Item`, non-recursive `rm`, or archive tools with delete-source flags — leaving `.swarm/` evidence files unprotected.

### Test coverage
- 48 new verification tests (`guardrails-swarm-path-guard.test.ts`)
- 64 new adversarial evasion tests (`guardrails-swarm-path-evasion.adversarial.test.ts`)
- 101 existing guardrail regression tests — all pass
- `scope-persistence.test.ts` — updated assertion for new prompt text

### Known caveats
- Cross-segment `cp && rm` chains (spanning `;` or newlines) are not caught at the guard level — each segment is evaluated independently. The `rm` half is independently blocked by Section 19.
- Shell wrapper bypasses (`sh -c 'mv .swarm/file'`) are documented as known bypasses in the adversarial test file.
- Quote-spliced command names (`m'v'`, `"mv"`) evade the guardrail regex.
- Path variations (`./swarm/` instead of `.swarm/`) are not normalized.
