# OpenCode Swarm v6.33.2 Release Notes

## Summary

Version **6.33.2** is a stabilization hotfix that widens loop detection coverage, fixes critical crash paths in delegation-gate and curator, adds two new documentation tools (`doc_scan`, `doc_extract`), introduces model fallback retry with bounded coder revisions, and adds structured error taxonomy for retrospectives.

---

## Critical Bug Fixes

### 1. Loop detection widened to catch extended loops

Loop detection guard in `guardrails.ts` was `=== 3`, which only caught loops hitting exactly 3 iterations. Changed to `>= 3 && < 5` to catch extended loops that start, partially recover, and re-enter the loop.

### 2. Delegation-gate session access fixed (CRIT-2)

Replaced `ensureAgentSession(sessionID).lastCoderDelegationTaskId` with direct `swarmState.agentSessions.get(sessionID)?.lastCoderDelegationTaskId`. The `ensureAgentSession` call could crash when the session was missing from `agentSessions`. Fixed by using optional chaining instead.

### 3. Delegation-gate async I/O fix (CRIT-3)

Converted `getEvidenceTaskId` from sync `readFileSync` to async `fs.promises.readFile` to prevent blocking the event loop during delegation-gate checks.

### 4. Curator atomic write (CRIT-4)

Replaced `Bun.write` with atomic temp-file + rename pattern in `writeCuratorSummary`. This prevents corrupted curator state if the process crashes mid-write.

### 5. Diagnostic noise reduction

Gated 5 `console.debug/warn` calls behind `process.env.DEBUG_SWARM` to reduce noise in production environments.

### 6. Typecheck errors fixed in turbo test files

Added missing default values for new `AgentSessionState` fields in 5 turbo test files.

### 7. ensureAgentSession migration guards added

Added missing migration guards for `coderRevisions` and `revisionLimitHit` fields. Previously, missing fields caused NaN propagation in bounded-revision advisory messages.

### 8. Snapshot schema version bump (v1 â†’ v2)

Schema version in snapshot state incremented from 1 to 2. This provides forward-compatible detection mechanism for future version mismatches. Old snapshots (v1) remain fully readable.

## New Features

### 1. doc_scan tool

Indexes project documentation with mtime-based caching. Generates a manifest at `.swarm/doc-manifest.json` tracking documentation files and their last-modified timestamps for efficient cache invalidation.

### 2. doc_extract tool

Extracts actionable constraints from relevant documentation using Jaccard bigram scoring. Matches document content against current task context to surface relevant guidelines and constraints.

### 3. Model fallback retry

Detects transient model errors (429, 503, timeout, overloaded) and injects retry advisory via `fallback_models` config. Supports up to 3 fallback models per agent:

```jsonc
{
  "agents": {
    "coder": {
      "fallback_models": ["gpt-4o-mini", "claude-3-haiku-20240307"]
    }
  }
}
```

### 4. Bounded coder revisions

New `max_coder_revisions` config limits coder retries per task with advisory injection. Prevents unbounded retry loops when the coder is making limited progress:

```jsonc
{
  "guardrails": {
    "max_coder_revisions": 3
  }
}
```

Default: **5** revisions per task. Range: 1-20.

### 5. Secretscan evidence gate

`pre-check-batch` now persists secretscan results to evidence. `check-gate-status` reports `BLOCKED` status for failed scans, ensuring security gates are enforced before merging.

### 6. Write-retro error taxonomy

Structured error classification with 5 categories for retrospective analysis:

| Category | Description |
|----------|-------------|
| `planning_error` | Incorrect plan or missing requirements |
| `interface_mismatch` | API/type mismatch between components |
| `logic_error` | Incorrect business logic implementation |
| `scope_creep` | Unplanned work added during execution |
| `gate_evasion` | Workaround or bypass of quality gates |

## Config Additions

### `fallback_models`

Up to 3 fallback model names for retry on transient errors.

```jsonc
{
  "agents": {
    "coder": {
      "fallback_models": ["model-name-1", "model-name-2"]
    }
  }
}
```

- Type: `string[]` (max 3)
- Required: No
- Default: `undefined`

### `max_coder_revisions`

Maximum coder retry attempts per task before injecting advisory.

```jsonc
{
  "guardrails": {
    "max_coder_revisions": 5
  }
}
```

- Type: `number` (integer, 1-20)
- Required: No
- Default: `5`
