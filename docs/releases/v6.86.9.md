# v6.86.9 — Plugin loader compatibility + cache eviction completeness

## What changed

### (a) Root cause: why 6.86.8 still didn't load

v6.86.8 fixed the `bun:sqlite` static import (Bug 1), but there was a second
unrelated root cause (Bug 2) that persisted across all plugin versions up to
and including 6.86.8.

OpenCode's `readV1Plugin` function requires `mod.default` to be an **object**
with at least one of `{ id, server, tui }` to succeed in detect mode. The
bundle's `default` export was a bare async function:

```js
export default async function OpenCodeSwarm(session) { ... }
```

When `readV1Plugin` receives a function it returns `undefined` and falls
through to `getLegacyPlugins`, which iterates `Object.values(mod)` and throws
`TypeError` on the `deferredWarnings` array re-export. OpenCode catches and
silently drops the plugin — the same "listed in sidebar, zero agents" symptom
as Bug 1, but a completely different cause.

This was independently verified against OpenCode's plugin source
(`packages/opencode/src/plugin/shared.ts` and `index.ts`).

### (b) The fix: v1 plugin export shape

Default export migrated to v1 object shape:

```js
export default { id: 'opencode-swarm', server: OpenCodeSwarm }
```

`readV1Plugin` now finds the object in detect mode and returns it immediately —
`getLegacyPlugins` is never called. The `addDeferredWarning`/`deferredWarnings`
re-exports removed from `src/index.ts` top level (they were the trigger for the
`TypeError`); internal consumers still import directly from
`./services/warning-buffer.js`.

A new regression test (`tests/unit/build/bundle-plugin-shape.test.ts`)
simulates both OpenCode loader paths (readV1Plugin detect mode AND
getLegacyPlugins walk) to catch this class of bug before release.

### (c) Cache eviction completeness: the missing canonical path

OpenCode v20+ caches plugins at `~/.cache/opencode/node_modules/opencode-swarm/`
([documented](https://opencode.ai/docs/plugins/)). The previous
`bunx opencode-swarm update` only cleared Layout 1
(`packages/opencode-swarm@latest`) and Layout 2 (legacy
`~/.config/opencode/node_modules/opencode-swarm`). Layout 3 — the one most users
on v20+ actually have — was **not** cleared, so even users who ran `update`
still loaded the old broken bundle.

All three layouts are now covered by `update` and `install`.
`isSafeCachePath` now has four-layer defense-in-depth:

1. Catastrophic-floor exclusion (blocks `..`, system paths)
2. Depth check (≥ 4 path segments required)
3. Recognized-leaf check (must end in `opencode-swarm`)
4. Canonical structure check (`opencode/{packages,node_modules}/opencode-swarm`)

### (d) Recovery: clear all three cache layouts

Remove the plugin from every cache location, then restart OpenCode. Or simply
run `bunx opencode-swarm update` which clears all three automatically.

**Windows 11 (PowerShell):**

```powershell
Remove-Item -Recurse -Force "$env:USERPROFILE\.cache\opencode\packages\opencode-swarm@latest" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "$env:USERPROFILE\.config\opencode\node_modules\opencode-swarm" -ErrorAction SilentlyContinue
Remove-Item -Recurse -Force "$env:USERPROFILE\.cache\opencode\node_modules\opencode-swarm" -ErrorAction SilentlyContinue
```

**macOS / Linux:**

```bash
rm -rf ~/.cache/opencode/packages/opencode-swarm@latest
rm -rf ~/.config/opencode/node_modules/opencode-swarm
rm -rf ~/.cache/opencode/node_modules/opencode-swarm
```

### (e) Diagnosing a failed load

If the plugin still fails to load after upgrading, run `/swarm diagnose` and
paste the output into the issue. The output now includes a **Plugin Caches**
section listing each cache path with status:

- ✅ present + version — plugin found at that location
- ⬜ absent — nothing cached at that location
- ⚠️ unreadable — permission denied or path is broken

Include the Plugin Caches section and the Version row in the issue comment.
Also include:

- OpenCode version (visible in the window title or from `opencode --version`)
- Which OS and how OpenCode was installed (desktop app, CLI, devcontainer,
  Codespaces)

## Bug fixes in this release

- **Fix:** plugin export shape — migrate default export to v1 object
  `{ id: 'opencode-swarm', server: OpenCodeSwarm }` (issue #675 second root
  cause)
- **Fix:** cache eviction — add canonical `~/.cache/opencode/node_modules/opencode-swarm/`
  path (Layout 3) so `update` clears the location v20+ users actually use
- **Security:** strengthen `isSafeCachePath` to 4-check defense-in-depth
- **Tests:** add `bundle-plugin-shape.test.ts` regression test simulating
  OpenCode's `readV1Plugin` + `getLegacyPlugins` contract

## Migration steps

- **No config changes required.**
- If the plugin currently fails to load: run `bunx opencode-swarm update`
  (once this version is published) and restart OpenCode.
- If you are on OpenCode v20+: verify your cache is clear by running
  `/swarm diagnose` and confirming the Plugin Caches section shows the
  expected layout.
