# v7.13.0 — Command UX Remediation and Phase Council Mutation Gap (closes #495)

## Summary

Closes [#495](https://github.com/zaxbysauce/opencode-swarm/issues/495)
("Test Quality Infrastructure umbrella"). Addresses deferred command UX
hazards around `/swarm plan` and `/swarm close`, adds a centralized
command-names registry, and ships FR-020 mutation_gap emission in the
phase council flow.

## What changed

### Canonical command renames

- `/swarm show-plan` is now the canonical command for viewing the execution plan (replaces `/swarm plan`)
- `/swarm finalize` is now the canonical command for closing a swarm project (replaces `/swarm close`)
- Both old names (`plan`, `close`) continue to work as deprecated aliases with backward-compatible handler dispatch
- The architect prompt, conflict registry, CLI templates, and help text all reference the new canonical names

### Centralized command-names registry

- New `src/commands/command-names.ts` exports `COMMAND_NAMES` (frozen array) and `COMMAND_NAME_SET` (frozen Set) derived from `COMMAND_REGISTRY`
- Provides a single source of truth for command name consumers, parity checks, and downstream validation
- Re-exported through `src/commands/index.ts` for convenient importing

### FR-020: Phase council mutation_gap emission

- `submit_phase_council_verdicts` now checks for mutation coverage gaps during phase council synthesis
- Emits a `mutation_gap` finding when `mutation-gate.json` is missing (HIGH), records `skip` (MEDIUM), `warn` (LOW), or `fail` (HIGH)
- Prevents duplicate emission if council member verdicts already include a `mutation_gap` category finding
- Persisted mutation-gap findings into `phase-council.json` evidence, keeping durable evidence aligned with the synthesis response

### Bug fixes

- Fixed `getPhaseMutationGapFinding` to correctly emit a HIGH severity finding for `verdict='fail'` (previously silently returned null)
- Added test coverage for malformed JSON in `mutation-gate.json` (catch block returns MEDIUM severity gracefully)
- Eliminated duplicate `phase-council.json` write: the tool now owns persistence exclusively with an unconditional write
