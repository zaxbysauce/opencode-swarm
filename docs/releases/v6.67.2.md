# v6.67.2

## What changed

### Equivalence detection is no longer dead code (CRITICAL)
The mutation testing equivalence detection feature was unreachable because the only production caller (`mutation_test` tool) never passed `sourceFiles` to `executeMutationSuite`. The tool now reads source file contents from disk and passes them through, activating equivalence checking that prevents false "killed" verdicts on functionally equivalent mutants.

### Missing command templates added (CRITICAL)
Two commands â€” `acknowledge-spec-drift` and `doctor tools` â€” had handlers in the command registry but lacked plugin config template entries in `src/index.ts`. Without templates, these commands were invisible to LLMs using the OpenCode plugin. Both templates have been added.

### Kill rate denominator consistency (HIGH)
The global `adjustedKillRate` metric used `total - equivalent` as its denominator, while the per-function `killRate` used `total - equivalent - skipped`. Both now use the same formula (`total - equivalent - skipped`) so mutation gate thresholds behave consistently.

### Test assertion made dynamic (HIGH)
The tool-names adversarial test had a hardcoded `expect(TOOL_NAMES.length).toBe(51)` that would break on every tool addition or removal. Replaced with a dynamic consistency check `expect(TOOL_NAMES.length).toBe(TOOL_NAME_SET.size)` that catches real bugs (duplicates, mismatches) without fragile numeric literals.

### test_command input validation (HIGH)
The `mutation_test` tool validated that `test_command` was a non-empty array but didn't check element types. An array like `['npx', undefined, 'vitest']` would pass validation and crash `spawnSync` at runtime. Added `test_command.every(c => typeof c === 'string')` validation.

### Additional fixes from Phases 1-3
- Registered 3 dead tools (diff_summary, test_impact, mutation_test) in plugin config
- Fixed shell injection vulnerability in mutation engine (execSync â†’ spawnSync)
- Fixed revert error recovery (no more abort on revert failure)
- Added `--` arg terminator to git spawnSync calls
- Fixed AST parse fallback in diff tool
- Wired markdown summary generation into diff pipeline
- Added command error handling (try-catch in command handler)
- Created registration smoke test for CI

## Why

A 5-member Work Complete Council (explorer, reviewer, critic, test_engineer, SME) audited all completed remediation work and found 2 CRITICAL and 3 HIGH issues that made the project unsafe to ship. This release fixes all of them.
