# v6.41.0

## Highlights

- **Dark Matter Detection Pipeline**: Automatic co-change analysis during DISCOVER mode to detect hidden file couplings
- **New `/swarm close` Command**: Idempotent close command that writes retrospectives, curates lessons, archives evidence, and clears state
- **write_drift_evidence Tool**: New tool for persisting drift verification results with gate-contract format
- **Session Improvements**: Reset-session now cleans session directory, endAgentSession wired at phase boundaries

## Features

### Dark Matter Detection Pipeline (Phase 2)

- **Automatic co-change analysis**: During DISCOVER mode, the system now automatically scans for files that frequently change together
- **Co-change knowledge entries**: Dark matter results are auto-generated as knowledge entries with category `architecture`
- **Architect guidance**: After declaring scope, architects are now instructed to consider co-change partners from dark-matter analysis
- **Silent git-less handling**: Repos without git history or fewer than 20 commits skip analysis silently

### `/swarm close` Command (Phase 4)

- **Idempotent close operation**: Safely closes a project at any state
- **Writes retrospectives**: Automatically generates retrospective evidence for in-progress phases
- **Knowledge curation**: Calls `curateAndStoreSwarm` to promote lessons to production knowledge
- **Sets closed status**: Marks all non-completed phases and tasks as `closed`
- **Archives evidence**: Calls `archiveEvidence` to compact old evidence bundles
- **Writes summary**: Generates `.swarm/close-summary.md` with project summary
- **State cleanup**: Calls `flushPendingSnapshot`, clears `agentSessions` and `delegationChains`

### write_drift_evidence Tool (Phase 5 HOTFIX)

- **Normalizes verdicts**: `APPROVED` → `approved`, `NEEDS_REVISION` → `rejected`
- **Gate-contract format**: Writes evidence to `.swarm/evidence/{phase}/drift-verifier.json`
- **Atomic writes**: Uses temp file + rename pattern to prevent corruption
- **Path validation**: Uses `validateSwarmPath` to prevent path traversal

## Bug Fixes

### Session Reset (Phase 3)

- **`/swarm reset-session` now cleans session directory**: After deleting `state.json`, the command now cleans all files in `.swarm/session/` except `state.json`

### Phase Completion (Phase 3)

- **`endAgentSession` wired at phase boundaries**: Sessions that are no longer active after a phase transition now have `endAgentSession` called properly

### Command Help Text (Phase 3)

- **8 missing commands added to `/swarm` help**: turbo, write-retro, reset-session, simulate, promote, checkpoint, config-doctor, evidence-summary
