## Detailed Changes

### Phase 1: Bug Fixes & Atomic Writes

- Fixed `knowledge` command edge cases that caused intermittent failures
- Implemented atomic write for `context.md` using temp+rename pattern to prevent corruption
- Fixed `guardrails` hook null reference when `ctx.directory` is undefined
- Migrated 17 tools from `process.cwd()` to `ctx.directory` for multi-project correctness
- Background plan manager wired and initialized
- Fixed stale SME cache invalidation logic in `context.md`

---

### Phase 2: Command Discoverability

- Registered all 21 `/swarm` commands in the command registry
- Implemented command handler routing for all 21 `/swarm` commands
- Removed stale comments from `index.ts` command handling
- Added `stop()` cleanup for background workers on plugin teardown
- Hardened `sast_scan` and co-change analyzer tools
- Added `createSwarmTool()` factory for consistent tool wrappers (`src/tools/create-tool.ts`)

---

### Phase 3: New Commands: simulate & rollback

- **`/swarm simulate`** (`src/commands/simulate.ts`): analyzes hidden coupling between files using dark matter detection; previews potential impact of changes based on git history co-change patterns
- **`/swarm rollback`** (`src/commands/rollback.ts`): restores from a named checkpoint by copying files directly from the checkpoint directory to .swarm/
- Both commands registered in `src/commands/index.ts`
- 81 tests for simulate + rollback

---

### Phase 4: Health Checks & Steering Acknowledgment

- Atomic writes in `save_plan` tool (temp+rename)
- 5 new health checks added to `/swarm diagnose`: config schema validity, background worker status, events stream integrity, checkpoint manifest consistency, steering directive staleness
- **`/swarm steering-consumed` hook** (`src/hooks/steering-consumed.ts`): automatically acknowledges unconsumed steering directives after each tool call, preventing silent directive accumulation

---

### Phase 5: Agent Intelligence Hooks

- **Co-change suggester hook** (`src/hooks/co-change-suggester.ts`): after each tool call touching a file, suggests related files that historically change together (from git history); advisory warnings only
- **Dark matter detector hook** (`src/hooks/dark-matter-detector.ts`): detects "dark matter" gaps â€” code paths mentioned in plan but not yet touched by any file writes; advisory warnings only
- 131 tests for Phase 5 hooks

---

### Phase 6: Session Persistence

- **`src/session/snapshot-writer.ts`**: serializes `swarmState` to `.swarm/session/state.json` after each tool call atomically (temp+rename). Handles `Map`/`Set` â†’ plain object/array conversion.
- **`src/session/snapshot-reader.ts`**: on plugin init, reads `.swarm/session/state.json` and rehydrates all agent sessions, tool aggregates, delegation chains, and active agent mappings. Includes JSON.parse reviver blocking `__proto__`/`constructor` prototype pollution.
- Wired into `src/index.ts`: `loadSnapshot()` at startup, `createSnapshotWriterHook()` writes after every tool call
- Result: closing your editor and reopening preserves full swarm state across sessions
- 122 session persistence tests (adversarial coverage: prototype pollution, path traversal, concurrent writes, oversized payloads)
