# OpenCode Swarm v6.29.1 Release Notes

## Summary

Version **6.29.1** delivers targeted bug fixes discovered during post-release QA of v6.29.0. Three behavioral gaps are resolved: advisory hook findings now reach the LLM architect instead of being silently discarded to stderr, and compaction metrics are live in `/swarm status` rather than hardcoded to zero.

---

## Bug Fixes

### 1. Advisory hooks now inject into LLM context (critical)

**Affected hooks:** `slop-detector`, `incremental-verify`, `compaction-service`, context-pressure warning

**Problem:** All four advisory paths in `src/index.ts` were wired to `console.warn(...)`, meaning their findings were silently logged to stderr and never seen by the architect LLM. SLOP CHECK findings, POST-CODER typecheck results, context compaction tier instructions, and the 50% context pressure warning were all invisible to the model.

**Fix:** Messages are now queued in a `pendingAdvisoryMessages: string[]` field on `AgentSessionState` and drained into the architect's system message via the existing `messagesTransform` hook in `guardrails.ts`. The injection follows the same proven pattern as the loop detection warning. Non-architect sessions are unaffected.

**Files changed:**
- `src/state.ts` — added `pendingAdvisoryMessages?: string[]` with initialization and migration guard
- `src/index.ts` — rewired all four advisory callbacks to push to the queue
- `src/hooks/guardrails.ts` — added drain block in `messagesTransform` (between loop warning and self-coding warning injection)
- `tests/unit/hooks/advisory-injection.test.ts` *(new)* — 5 tests covering the drain round-trip

### 2. Compaction metrics wired into `/swarm status` (medium)

**Affected:** `compactionCount` and `lastSnapshotAt` fields in `StatusData`

**Problem:** Both fields were hardcoded to `0` and `null` in all `getStatusData()` return paths. The compaction service tracked tier triggers internally but never exposed them, so `/swarm status` always showed no compaction activity regardless of actual runtime behavior.

**Fix:** Module-level state promotion in `compaction-service.ts` allows a `getCompactionMetrics()` getter to return live counts and the ISO timestamp of the last snapshot write. `status-service.ts` now consumes this getter.

**Files changed:**
- `src/services/compaction-service.ts` — promoted `state` to module-level, added `lastSnapshotAt` field, added `getCompactionMetrics()` and `resetCompactionState()` exports
- `src/services/compaction-service.test.ts` — calls `resetCompactionState()` in `beforeEach` to prevent module-level hysteresis leakage between tests
- `src/services/status-service.ts` — imports and uses `getCompactionMetrics()` in all three return paths
- `tests/unit/services/compaction-metrics.test.ts` *(new)* — 3 tests covering initial state, post-trigger state, and status-service integration

---

## Upgrade Notes

- **Backward compatible** — no breaking changes, no API surface changes
- **Advisory messages** — starting from this version, slop-detector, incremental-verify, compaction-service, and context-pressure findings appear inside `[ADVISORIES]...[/ADVISORIES]` blocks in the architect's system message
- **`/swarm status`** — `compactionCount` and `lastSnapshotAt` now reflect real compaction activity; values reset to 0/null on plugin reload (in-memory only, not persisted)
