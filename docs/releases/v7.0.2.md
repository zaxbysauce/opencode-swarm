# v7.0.2

## What changed

### General Council Mode: role-framed dispatch + architect synthesis

- **Removed `council_member` and `council_moderator` agent types**: Eliminated dedicated agent registrations that caused model resolution to always fall back to `DEFAULT_MODELS.council_member` because no user config ever contained a `council_member` entry.
- **Introduced `council_generalist`, `council_skeptic`, `council_domain_expert`**: Three role-framed agents dispatched via existing reviewer/critic/sme factories, each pulling their model from the user's `agents.reviewer.model`, `agents.critic.model`, and `agents.sme.model` overrides respectively.
- **Architect owns web research**: Architect now runs 1–3 `web_search` calls pre-dispatch and compiles a RESEARCH CONTEXT block shared with all three agents, replacing the prior pattern where each council member independently searched. Centralizes research, reduces redundant API calls, and ensures all agents reason from the same evidence.
- **Architect owns synthesis**: Synthesis rules (LEAD WITH CONSENSUS, ACKNOWLEDGE DISAGREEMENT HONESTLY, CITE STRONGEST SOURCES, BE CONCISE) moved from a dedicated moderator agent prompt to inline architect instructions, eliminating a delegated LLM round-trip.
- **Deprecation warning for `council.general.moderatorModel`**: Setting this field at runtime now surfaces a deferred deprecation warning. The `moderator` boolean field is intentionally not checked — its schema default of `true` would have triggered the warning for every council user.
- **Schema backward compatibility**: Deprecated fields (`moderator`, `moderatorModel`, `members`, `presets`) are retained on the strict schema for backward compatibility but ignored at runtime.

## Why

- The original `council_member` agent type caused a silent model resolution bug: `getModel('council_member')` always fell back to `DEFAULT_MODELS.council_member` because users never have a `council_member` entry in their swarm config. All three council agents now resolve through existing named config paths (`reviewer`, `critic`, `sme`).
- The separate moderator delegation added a full LLM round-trip for synthesis work the architect can do inline.
- Independent per-member web searches produced redundant API calls and inconsistent evidence across agents; centralizing research through the architect eliminates both problems.

## Migration steps

No action required for most users. If you have custom config:

- `agents.reviewer.model`, `agents.critic.model`, `agents.sme.model` now also control the models used by the General Council's generalist, skeptic, and domain expert agents respectively.
- If you previously set `council.general.moderatorModel`, you will now see a deprecation warning. Remove that field — the architect synthesizes directly and the field has no effect.
- `council.general.members` and `council.general.presets` are retained on the schema but ignored at runtime. Remove them from your config to silence any future warnings.

## Additional fixes

- **Deterministic lesson ordering in `/swarm close`**: `retroDirs` is now sorted alphabetically before iteration, so the lesson list passed to `curateAndStoreSwarm` is stable across filesystems with non-deterministic `readdir` order.
- **`close-finalizer` test mock completeness**: Added `resetToRemoteBranch` to the branch mock in the finalizer test suite, fixing a `SyntaxError: Export named 'resetToRemoteBranch' not found` that surfaced after the function was added to `src/git/branch.ts` in the prior release.
- **`/swarm pr-review` bare-number resolution with proxy remotes**: Extended `parseGitRemoteUrl` with a generic path-segment fallback so bare PR numbers resolve correctly when the git remote is a proxy or GitHub Enterprise URL (not just `github.com` HTTPS/SSH).
- **CI double-run eliminated**: Added `concurrency` group to the CI workflow so push and pull_request events for the same branch share one slot, preventing the transient dist-check failure that occurred when both events triggered simultaneously.

## Known caveats

- The architect's synthesis pass does not perform new web searches — it reasons only from the RESEARCH CONTEXT compiled pre-dispatch. For rapidly changing topics, the architect's pre-search pass (1–3 queries) may not capture all relevant recent developments.
