# OpenCode Swarm v6.32.0 Release Notes

## Summary

Version **6.32.0** introduced truncation expansion for longer contexts, compaction hints for smarter summarization, prompt hardening for improved output quality, and a comprehensive regression sweep of critical paths.

---

## Truncation Expansion

### Extended context windows

Increased default truncation limits to support longer context windows for complex multi-file tasks:

```jsonc
{
  "context": {
    "max_tokens": 200000,
    "truncation_strategy": "preserve_recent"
  }
}
```

### Smart truncation strategies

New truncation strategies control which parts of context are preserved when limits are reached:
- `preserve_recent`: Keeps most recent context, truncates older content
- `preserve_relevant`: Keeps context most relevant to current task
- `preserve_milestones`: Keeps key decision points and architecture decisions

---

## Compaction Hints

### Content-aware summarization

Compaction now respects user-defined hints about what content is worth preserving:

```jsonc
{
  "context": {
    "compaction_hints": {
      "preserve": ["architecture_decisions", "api_contracts", "error_patterns"],
      "summarize": ["debugging_steps", "exploration_attempts", "failed_approaches"]
    }
  }
}
```

### Hint-based prioritization

Context that matches preservation hints is prioritized for retention during compaction.

---

## Prompt Hardening

### Improved instruction clarity

Enhanced prompt engineering for all agents to reduce ambiguity and improve output consistency:
- More explicit success criteria
- Clearer error reporting requirements
- Better handling of edge cases

### Quality threshold enforcement

Prompts now include explicit quality thresholds that trigger automatic review loops when not met.

---

## Regression Sweep

### Critical path validation

Comprehensive review and testing of high-risk code paths identified through complexity hotspots analysis.

### Edge case coverage

Added test coverage for previously untested edge cases discovered during the regression sweep.

### Bug fixes

Resolved issues discovered during the regression sweep, focusing on:
- Session state corruption scenarios
- Delegation boundary violations
- Context loss during phase transitions

---

## Migration Notes

- No breaking changes. New truncation and compaction settings have conservative defaults.
- Prompt hardening is transparent to existing workflows.
- Existing sessions continue with their previous context settings.
