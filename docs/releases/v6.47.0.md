# v6.47.0

## What changed

### `/swarm close` aligned with full session close-out workflow

The `/swarm close` command now performs the complete set of cleanup steps that were
previously done manually at the end of a session. Five specific gaps were closed:

**Plan-free session support**
Sessions that never created a `plan.json` (PR reviews, investigations, ad-hoc analysis)
previously caused `/swarm close` to exit with an error. The command now detects a missing
plan as a plan-free session and runs all cleanup steps anyway. Malformed `plan.json`
(file exists but is invalid JSON) still returns an error.

**`context.md` reset**
After closing, the command writes a clean `context.md` to `.swarm/` noting the session
is done and no active plan exists. Previously, `context.md` accumulated stale agent
activity tables from prior sessions that could confuse new sessions reading the file.

**Config-backup cleanup**
The config-doctor writes `config-backup-{timestamp}.json` files to `.swarm/` and never
removes them. The close command now deletes all `config-backup-*.json` files on close.
Per-file failures are non-blocking.

**Optional git branch pruning (`--prune-branches`)**
Running `/swarm close --prune-branches` deletes local branches whose remote tracking
ref is marked as `gone` (merged/deleted upstream). Uses `execFileSync` with array
arguments to prevent shell injection. Safe delete only (`-d`, never `-D`). Branches
that are unmerged or checked out in a worktree are skipped with a warning rather than
force-deleted. Non-git directories are handled gracefully.

**Lesson injection via `.swarm/close-lessons.md`**
If the architect writes explicit session lessons to `.swarm/close-lessons.md` before
running `/swarm close`, the command reads them and passes them to `curateAndStoreSwarm`
for dedup, validation, and storage in the knowledge base. The file is deleted only after
confirmed curation success — if curation throws, the file is preserved so lessons are
not lost. Comment lines (`#`) and blank lines are filtered out. The file-based protocol
avoids the multi-word parsing limitation of flag-based arg passing.

## Why

The manual session close-out process included steps that `/swarm close` did not automate:
resetting context, removing stale backup files, pruning merged branches, and injecting
explicit lessons into the knowledge base. These gaps meant the command was only useful
for plan-based sessions and left cleanup work to be done manually each time.

## Migration steps

None for existing users — all new behaviours are additive or triggered only by the new
`--prune-branches` flag or the optional `close-lessons.md` file.

## Breaking changes

One test expectation updated: the test asserting that a missing `plan.json` returns
`"Failed to read plan.json"` was updated to assert a successful plan-free close instead.
No public API changes.

## Known caveats

- `--prune-branches` requires `git` in `PATH` and must be run from inside the git
  repository root. Non-git directories fail silently.
- Lessons in `close-lessons.md` go through the same dedup and validation pipeline as
  all other knowledge entries — lessons that are too similar to existing entries will be
  skipped rather than stored.
