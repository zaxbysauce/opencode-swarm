# OpenCode Swarm v6.29.2 Release Notes

## Summary

Version **6.29.2** delivers language portability improvements and a critical planning enhancement. The incremental-verify and slop-detector hooks now work across 5 languages (TypeScript, Go, Rust, Python, C#), the spawn helper is portable across Bun/Node.js, and the Architect now performs a CODEBASE REALITY CHECK before planning to prevent stale assumptions.

---

## Features

### 1. Multi-Language Incremental Verify

The incremental-verify hook now detects and runs typecheck/build commands for 5 languages in priority order:

| Language | Detection Marker | Command |
|----------|-----------------|---------|
| TypeScript | `package.json` with `typecheck`/`type-check` script | `bun run typecheck` |
| TypeScript | `package.json` with TS deps/tsconfig | `npx tsc --noEmit` |
| Go | `go.mod` present | `go vet ./...` |
| Rust | `Cargo.toml` present | `cargo check` |
| C# | `*.csproj` or `*.sln` in project root | `dotnet build --no-restore` |
| Python | `pyproject.toml`, `requirements.txt`, or `setup.py` | **No default** — emits advisory |

**Python behavior:** Python emits an advisory message (injected into architect context) instead of running a checker:
```
POST-CODER CHECK SKIPPED: python project detected but no default checker available. 
Set incremental_verify.command in .swarm/config.json to enable.
```

The advisory is deduplicated — only emitted once per session per language. To suppress, set `incremental_verify.command` in your config.

**Array command support:** The `command` config now accepts both `string` and `string[]`. The array form bypasses shell splitting, enabling commands with special arguments:

```json
{
  "incremental_verify": {
    "command": ["python", "-m", "mypy", "--config-file", "mypy.ini"]
  }
}
```

### 2. spawnAsync Portability

New `src/hooks/spawn-helper.ts` provides a `spawnAsync(command, cwd, timeoutMs)` utility wrapping `node:child_process.spawn`. Returns `{ exitCode, stdout, stderr }` or `null` on timeout/error. Replaces Bun-specific `Bun.spawn` calls in incremental-verify and slop-detector for Node.js compatibility.

### 3. Slop-Detector Hardening

Several improvements to the dead-export and abstraction-bloat heuristics:

- **Dead-export guard:** Now only runs on JS/TS projects (requires `package.json`). Was silently running on all projects.
- **File scan:** Now scans `.ts`, `.tsx`, `.js`, `.jsx` recursively, excluding `node_modules` and `.git`. Was hardcoded to `src/**/*.ts`.
- **Comment strip:** Regex now covers `#` (Python, Ruby, Shell, YAML) and `--` (SQL, Lua, Haskell) in addition to `//` and `/* */`.
- **Abstraction bloat:** Regex now covers `struct` (Go/Rust/C) and `impl` (Rust) in addition to `class`.

### 4. CODEBASE REALITY CHECK (Architect)

**Important behavioral change.** Before any spec generation, plan creation, or plan ingestion, the Architect now dispatches the Explorer in targeted scoped chunks to verify the current state of every referenced item.

**When it fires:**
- MODE: SPECIFY — step 1b, before explorer dispatch
- MODE: PLAN — after SPEC GATE, before save_plan
- EXTERNAL PLAN IMPORT PATH — step 1 (new), before existing steps

**What it produces:** A CODEBASE REALITY REPORT listing every referenced file/function/class with one of four statuses:
- `NOT STARTED` — Item referenced in plan but not implemented
- `PARTIALLY DONE` — Item exists but incomplete
- `ALREADY COMPLETE` — Item fully implemented
- `ASSUMPTION INCORRECT` — Plan assumed something that's no longer true

**Greenfield exemption:** Skipped if work is purely greenfield with no existing codebase references.

**Why this matters:** Prevents planning against stale assumptions — e.g., assuming a function still exists when it was already fixed in a prior release.

## Bug Fixes

### Evidence Schema phase_number Minimum

The `phase_number` field in evidence-schema now has a minimum of `1` instead of `0`. Phase 0 was never valid in the swarm model.

---

## Upgrade Notes

- **Incremental verify** — Now runs on Go, Rust, C# projects automatically. Python projects receive an advisory instead of silently skipping.
- **Config change** — `incremental_verify.command` now accepts an array for commands with special arguments.
- **Architect workflow** — CODEBASE REALITY CHECK adds a verification step before planning. Expect slightly longer initial planning time on brownfield projects.
- **Slop-detector** — Now guarded to JS/TS only. May emit new advisories on projects with dead exports or abstraction bloat.
