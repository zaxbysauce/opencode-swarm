# v6.26.1 — Gate Recovery Fix for Pure-Verification Tasks

## Overview

Patch fix for `update_task_status(completed)` incorrectly blocking lower-quality models
when reviewer and test_engineer were delegated without a prior coder delegation (pure-verification
tasks, code-organization tasks, scaffold tasks).

## Bug Fixes

### `recoverTaskStateFromDelegations` — unscoped fallback scan

The recovery function previously only scanned delegation chains from sessions where
`currentTaskId` or `lastCoderDelegationTaskId` matched the target task ID. For tasks with
no coder delegation (code-organization, scaffold, pure-verification), neither field was ever
set, so the recovery found zero matching sessions and returned early — leaving the gate blocked
even though reviewer and test_engineer had genuinely run.

**Fix:** Added a Pass 2 unscoped fallback that scans ALL delegation chains when the task-scoped
scan finds nothing. Safety constraint: only counts reviewer/test_engineer delegations that appear
*after* the last coder delegation in each chain (prevents attributing a prior task's gates to
the current task).

### `checkReviewerGate` — delegation-chain direct check

Added a final fallback before returning `blocked: true`. Scans delegation chains directly using
the same two-pass logic. Returns `blocked: false` only if BOTH reviewer AND test_engineer are
confirmed in the chains.

### Test isolation fix

Two gate tests were calling `checkReviewerGate` without a `workingDirectory`, causing them to
read the project's `.swarm/evidence/` directory (which now contains evidence files from prior
phases). Fixed by passing `tmpDir` to isolate tests from project state.

## Testing

- 4 new targeted tests covering the unscoped fallback behavior
- All 15 gate tests passing (70/70 across both test files)
- Verified: gate still blocks when delegation chains are genuinely empty
- Verified: reviewer/test_engineer before the last coder are correctly ignored
