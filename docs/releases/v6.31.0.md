# OpenCode Swarm v6.31.0 Release Notes

## Summary

Version **6.31.0** delivers five feature areas: process.cwd() tech debt elimination, curator pipeline wiring, watchdog enforcement hooks, self-correcting workflow hardening, and a tool-based knowledge memory system. Across 68 files and 5,800 new lines, the release adds 6 new architect-callable tools, 4 new runtime hooks, and 141 new tests with no breaking changes.

---

## Features

### 1. process.cwd() tech debt cleanup

All 14 source files that previously called `process.cwd()` now use the plugin-injected `directory` parameter as their primary source. `process.cwd()` remains only as a documented final-resort fallback in 3 intentional locations (`createSwarmTool`, CLI entrypoint, standalone functions).

Five tools (diff, gitingest, imports, retrieve-summary, secretscan) were migrated from raw `tool()` to the `createSwarmTool` wrapper for consistent directory injection.

**Files changed:** 14 source files across `src/tools/`, `src/hooks/`, `src/background/`, `src/context/`, `src/knowledge/`, `src/quality/`

### 2. Curator pipeline wiring

The curator analysis system is now directly accessible to the architect through a new `curator_analyze` tool. Previously, curator results were computed but silently discarded. Now:

- **`curator_analyze` tool** — architect can explicitly trigger phase analysis and apply knowledge recommendations
- **Phase-complete advisory injection** — after each phase, a structured advisory prompts the architect to review curator findings and apply recommendations
- **Drift detection advisory** — when plan-vs-spec drift is detected, a visible advisory is injected into the architect's context

### 3. Watchdog pattern

Three enforcement hooks form a coordinated watchdog system:

- **`scope_guard`** (tool.execute.before) — blocks out-of-scope file writes by non-architect agents. Paths are sanitized against log injection (control chars, ANSI escape codes). Runs WITHOUT safeHook to preserve blocking throw behavior.
- **`delegation_ledger`** (tool.execute.after) — maintains an in-memory per-session ledger tracking every sub-agent tool call. Injects a compact summary when the architect session resumes.
- **Loop detector escalation** — at the existing threshold of 3 consecutive identical delegations, a structured escalation message now includes the loop pattern, suggested alternatives, and an accomplishment summary.

Configuration: `config.watchdog.scope_guard` (default: true), `config.watchdog.delegation_ledger` (default: true), `config.watchdog.skip_in_turbo` (default: false — scope violations are never skippable).

### 4. Self-correcting workflow hardening

- **`self_review` advisory hook** — after `update_task_status` moves a task to `in_progress`, an advisory reminds the architect to delegate to the critic for self-review focus (broken conditionals, off-by-one errors, scope creep). Skips in turbo mode via `session.turboMode`. Configuration: `config.self_review.enabled` (default: true), `config.self_review.skip_in_turbo` (default: true).
- **`checkStaleImports` heuristic** — 5th slop-detector check that flags files with 2+ unused import identifiers using regex-based analysis. Handles named, default, and namespace imports. Skips re-exports and type-only imports. Configuration: `config.slop_detector.importHygieneThreshold` (default: 2).

### 5. Tool-based knowledge memory system

Three new tools give any agent direct access to the persistent knowledge base:

- **`knowledge_recall`** — search knowledge entries using Jaccard bigram similarity with status-based boosting. Supports tier filtering (swarm/hive/all), configurable top-N (default 5).
- **`knowledge_add`** — store a new lesson with validation (15–280 chars, 9 valid categories, optional tags, capped at 20). Auto-sets `status: candidate`, `confidence: 0.5`, `auto_generated: true`.
- **`knowledge_remove`** — delete a knowledge entry by ID. Idempotent — double-deletion returns "entry not found" without error. Uses file locking via `rewriteKnowledge`.
- **Compaction reminder** — when context window compaction occurs, a reminder of the three knowledge tools is injected to survive context resets.

## Configuration

New optional config entries (all with backward-compatible defaults):

```jsonc
{
  "watchdog": {
    "scope_guard": true,        // block out-of-scope writes (default: true)
    "delegation_ledger": true,  // track agent tool calls (default: true)
    "skip_in_turbo": false      // scope violations never skippable (default: false)
  },
  "self_review": {
    "enabled": true,            // self-review advisory (default: true)
    "skip_in_turbo": true       // skip in turbo mode (default: true)
  },
  "slop_detector": {
    "importHygieneThreshold": 2 // stale import flag threshold (default: 2)
  }
}
```

- 2 stale schema test expected values in `tests/unit/config/schema.test.ts` for `adversarial_testing` and `knowledge` config defaults.
