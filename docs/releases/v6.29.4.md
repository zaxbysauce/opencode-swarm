# OpenCode Swarm v6.29.4 Release Notes

## Summary

Version **6.29.4** addresses two structural QA gate failures surfaced by the RAGAPPv2 retrospective: cross-task test regressions invisible to per-task testing (15 regressions found), and redundant dedicated test tasks consuming execution budget without value (15/16 redundant). Additionally, the Curator data pipeline outputs are now visible to the architect, and full-suite test execution is conditionally unlocked behind an explicit opt-in flag.

---

## Features

### 1. Regression Sweep Gate

Added step 5l-bis to the architect QA pipeline. After test_engineer-verification passes, the architect runs `test_runner` with `scope: "graph"` on all source files changed by the coder. This traces imports to discover test files beyond the task's own tests that may be affected.

Three outcomes:
- **SKIP** — no additional tests found beyond what test_engineer already ran
- **PASS** — additional tests found and all pass
- **FAIL** — regression detected, coder retry with structured feedback

The architect calls test_runner directly for this step (not via test_engineer, whose EXECUTION BOUNDARY restricts it to its own test files).

### 2. Test Task Deduplication Guidance

Added guidance block to architect MODE: PLAN explaining that the QA gate already writes and runs tests on every implementation task. Separate "write tests for X" plan tasks are redundant with the gate and waste execution budget.

Research reference: arXiv:2602.07900 found that large shifts in test-writing volume yielded only 0–2.6% resolution change while consuming 20–49% more tokens.

Three exceptions where dedicated test tasks remain appropriate:
- Pure test infrastructure (fixtures, helpers, mock factories, CI config)
- Cross-module integration tests spanning multiple implementation tasks
- Explicit user-requested coverage pass

### 3. Conditional scope:"all" Unlock

Replaced the hard block on `scope: "all"` in test_runner with an `allow_full_suite: true` opt-in flag. The root cause of the original restriction is opencode's SSE pipeline instability with large payloads (anomalyco/opencode #17977, #15645, #17908), not a general prohibition.

- `scope: "all"` without `allow_full_suite` → error with clear explanation
- `scope: "all"` with `allow_full_suite: true` → passes through
- `scope: "convention"` and `"graph"` → unaffected

### 4. Curator Wiring Fix

Connected the Curator data pipeline outputs to the architect's context injection path:

- **Briefing persistence**: `phase-monitor.ts` now captures the `runCuratorInit()` briefing and writes it to `.swarm/curator-briefing.md` (overwritten each session init, so stale data is not a concern)
- **Briefing injection**: `knowledge-injector.ts` reads `curator-briefing.md` and injects it as a `<curator_briefing>` tag in architect context (truncated to 500 chars)
- **Compliance surfacing**: `phase-complete.ts` now surfaces curator compliance observations (missing reviewer, missing retro, etc.) in the return value via the existing warnings array when `suppress_warnings: false`
- **Drift guard fix**: Restructured `knowledge-injector.ts` so drift reports and curator briefing inject even when no knowledge entries exist. Previously, the `entries.length === 0` early return prevented drift injection from executing.

## Bug Fixes

### Knowledge Injector Drift Guard

Fixed a control flow bug where `knowledge-injector.ts` line 170 (`if (entries.length === 0) return;`) exited before the drift injection block (lines 229–241) could execute. When no knowledge entries existed but a drift report was present, the drift report was silently dropped. The drift injection block was moved before the early return guard.

### Knowledge Overwrite Bug

Fixed a secondary bug where the knowledge section assignment (`cachedInjectionText = knowledgeSection`) overwrote any drift report or curator briefing that had already been injected. Changed to append knowledge after existing cached injection text, preserving the priority order: run memory > drift > curator briefing > knowledge > rejected patterns.

## Upgrade Notes

- **Regression sweep** is automatic — no config changes needed. It runs after every test_engineer-verification step.
- **Test task deduplication** is prompt-level guidance only — no config changes. The architect will no longer create redundant test tasks.
- **scope:"all" gated access** — the architect can now use `allow_full_suite: true` for deliberate full-suite runs. Default behavior (rejection) is unchanged.
- **Curator briefing** — if curator is enabled, architects will now see the curator's session-start briefing in their context. No config changes needed.
- **Curator compliance** — compliance observations are surfaced in phase_complete output only when `suppress_warnings: false`. Default behavior (suppressed) is unchanged.
