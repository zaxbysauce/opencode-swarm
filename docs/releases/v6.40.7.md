# v6.40.7

## Bug Fixes

- **Fix tree-sitter WASM ABI mismatch causing `LinkError: _emscripten_memcpy_js`**: The build script was copying `tree-sitter.wasm` (the core runtime) from `@vscode/tree-sitter-wasm` v0.3.0, but the JavaScript runtime is `web-tree-sitter` v0.25.10. These are different builds with incompatible Emscripten ABIs, causing a `LinkError` at WASM instantiation time.

### What Changed

- **`scripts/copy-grammars.ts`**: Changed the source of `tree-sitter.wasm` from `node_modules/@vscode/tree-sitter-wasm/wasm/tree-sitter.wasm` to `node_modules/web-tree-sitter/tree-sitter.wasm`. Language grammar WASMs (e.g., `tree-sitter-javascript.wasm`) still come from `@vscode/tree-sitter-wasm` since they are ABI-compatible across versions.

- **`src/lang/grammars/tree-sitter.wasm`**: Updated to match `web-tree-sitter` v0.25.10 (205,488 bytes, previously 206,218 bytes from `@vscode/tree-sitter-wasm`).

### Why

The `@vscode/tree-sitter-wasm` package bundles its own `tree-sitter.wasm` built for VS Code's internal tree-sitter version, which uses a different Emscripten build configuration than `web-tree-sitter`. When v6.40.6's `locateFile` callback redirected `web-tree-sitter` to load this incompatible WASM binary, it triggered `LinkError: import function env:_emscripten_memcpy_js must be callable` at runtime.

### Migration

None required.

### Breaking Changes

None.
