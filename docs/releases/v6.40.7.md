# v6.40.7

## Bug Fixes

- **Fix tree-sitter WASM ABI mismatch causing `LinkError: _emscripten_memcpy_js`**: The build script was copying `tree-sitter.wasm` (the core runtime) from `@vscode/tree-sitter-wasm` v0.3.0, but the JavaScript runtime is `web-tree-sitter` v0.25.10. These are different builds with incompatible Emscripten ABIs, causing a `LinkError` at WASM instantiation time.

- **Register 4 shipped-but-unregistered grammar WASMs**: `tsx`, `powershell`, `ini`, and `regex` grammars were copied to `dist/lang/grammars/` by the build script but missing from `LANGUAGE_WASM_MAP`. They worked via the fallback path but were invisible to `getSupportedLanguages()` and `bun run doctor`.

- **Add core `tree-sitter.wasm` to diagnose grammar check**: The `checkGrammarWasmFiles()` health check validated language grammar files but not the core runtime WASM — the very file that caused the original v6.40.6 ENOENT bug. Now checked explicitly.

- **Remove phantom `tree-sitter-html.wasm` and `tree-sitter-json.wasm` from diagnose grammar check**: These files don't exist in `@vscode/tree-sitter-wasm`, aren't in `LANGUAGE_WASM_MAP`, and were never shipped — causing `bun run doctor` to always report 2 missing grammars.

- **Reset `treeSitterInitialized` in `clearParserCache()`**: Previously, clearing the parser cache did not reset the tree-sitter initialization flag, meaning subsequent `initTreeSitter()` calls would skip re-initialization. This could cause issues in tests switching between dev/bundle contexts.

- **Use `import.meta.url` for script-relative paths in `copy-grammars.ts`**: All path resolution now uses `dirname(fileURLToPath(import.meta.url))` instead of `process.cwd()`, making the script work correctly when invoked from a different working directory (e.g., monorepo orchestrators).

### What Changed

- **`scripts/copy-grammars.ts`**: Changed the source of `tree-sitter.wasm` from `@vscode/tree-sitter-wasm` to `web-tree-sitter`. Replaced all `process.cwd()` path resolution with `PROJECT_ROOT` derived from `import.meta.url`.

- **`src/lang/grammars/tree-sitter.wasm`**: Updated to match `web-tree-sitter` v0.25.10 (205,488 bytes, previously 206,218 bytes from `@vscode/tree-sitter-wasm`).

- **`src/lang/runtime.ts`**: Added `tsx`, `powershell`, `ini`, and `regex` to `LANGUAGE_WASM_MAP`. `clearParserCache()` now resets `treeSitterInitialized = false`.

- **`src/services/diagnose-service.ts`**: Added `tree-sitter.wasm` core runtime check. Added `tsx`, `powershell`, `ini`, `regex` grammar entries. Removed phantom `html` and `json` entries. Changed hardcoded count to dynamic `grammarFiles.length`.

### Why

The `@vscode/tree-sitter-wasm` package bundles its own `tree-sitter.wasm` built for VS Code's internal tree-sitter version, which uses a different Emscripten build configuration than `web-tree-sitter`. When v6.40.6's `locateFile` callback redirected `web-tree-sitter` to load this incompatible WASM binary, it triggered `LinkError: import function env:_emscripten_memcpy_js must be callable` at runtime.

### Migration

None required. The 4 newly registered grammars were already loadable via the fallback path; registration makes them visible to `getSupportedLanguages()`.

### Breaking Changes

None.
