# OpenCode Swarm v6.16.0 Release Notes

## Summary

Version 6.16.0 introduces **Multi-Language Support** — a profile-driven abstraction that makes every tool in the QA pipeline language-aware across 11 languages and 3 tiers. Previously tools like `lint`, `test_runner`, `build_check`, `pkg_audit`, and `sast_scan` had hardcoded detection for only 3–5 languages. They now resolve the correct binary, framework, and rules for any supported language automatically, with graceful degradation when optional tools are absent.

- **11 languages, 3 tiers** — full pipeline coverage for TS/JS, Python, Rust, Go; strong coverage for Java, Kotlin, C#, C/C++, Swift; basic coverage for Dart and Ruby
- **Profile-driven tools** — all tool detection now reads from `LanguageRegistry` profiles instead of hardcoded if-branches
- **Semgrep SAST** — auto-mode dispatch for languages without native rulesets; soft skip when binary absent
- **Language-aware prompt injection** — coder and reviewer agents receive language-specific constraints and checklists
- **317+ new tests** — full unit, integration, and adversarial coverage for all new paths

## What's New

### Language Profile Registry (`src/lang/profiles.ts`, `src/lang/detector.ts`)

A new `LanguageRegistry` singleton holds `LanguageProfile` objects for all 11 languages. Each profile captures the language's tier, file extensions, build commands, test frameworks, lint tools, audit command, SAST configuration, and language-specific coder/reviewer prompt snippets.

Two new exported functions:
- `detectProjectLanguages(dir)` — async, inspects root directory files (`package.json`, `Cargo.toml`, `go.mod`, `pubspec.yaml`, `build.gradle.kts`, etc.) and returns all active language profiles
- `getProfileForFile(filePath)` — sync, resolves the language profile for a given source file by extension

### New Tree-sitter Grammars

Kotlin, Swift, and Dart WASM grammars are now vendored in `src/lang/grammars/` and wired into `LANGUAGE_WASM_MAP` in `src/lang/runtime.ts`. The `syntax_check` tool now parses `.kt`, `.swift`, and `.dart` files natively.

### Profile-Driven Build Detection

`discoverBuildCommandsFromProfiles()` in `src/build/discovery.ts` reads active language profiles and returns the highest-priority available build binary per language. Existing ECOSYSTEMS-based detection is preserved as a fallback.

### Profile-Driven Test Framework Detection (16 frameworks)

`src/tools/test-runner.ts` gained 9 new detector functions:

| Language | Frameworks |
|----------|-----------|
| Go | `go test` |
| Java | Maven (`mvn`), Gradle (`gradle`) |
| Kotlin | `gradle test` |
| C# / .NET | `dotnet test` |
| C / C++ | CMake / ctest |
| Swift | `swift test` |
| Dart | `dart test` |
| Ruby | RSpec, minitest |

### Profile-Driven Lint Detection (10 additional linters)

`detectAdditionalLinter()` in `src/tools/lint.ts` checks for and dispatches to: `golangci-lint`, Checkstyle, `ktlint`, `dotnet-format`, `cppcheck`, `swiftlint`, `dart analyze`, RuboCop, `scalafmt`, `buf`. All follow the same soft-skip pattern when the binary is absent.

### Package Audit Expansion (7 auditors)

`src/tools/pkg-audit.ts` now audits 7 ecosystems, all normalized to a unified `AuditResult` format:

| Ecosystem | Tool |
|-----------|------|
| Node.js | `npm audit` |
| Python | `pip-audit` |
| Rust | `cargo audit` |
| Go | `govulncheck` |
| C# / .NET | `dotnet list package --vulnerable` |
| Ruby | `bundle-audit` |
| Dart | `dart pub outdated` |

### Semgrep SAST Integration

`src/tools/sast-scan.ts` now dispatches to Semgrep for languages without native rulesets using `semgrep --config auto --lang <lang> --json`. Semgrep is never a hard dependency — if the binary is absent, a soft warning is emitted and the gate proceeds.

### Language-Aware Prompt Injection

Coder and reviewer agents now receive language-specific instructions injected into their system prompts:
- **Coder**: `coderConstraints` from the detected language profile (e.g. "Use idiomatic Go error returns, never panic in library code")
- **Reviewer**: `reviewerChecklist` items as markdown checkboxes (e.g. "[ ] No use of `unsafe` without documented justification")

Detection uses `getProfileForFile()` on the task's target file path. Injection is additive and silently skipped when no language is detected.

### Graceful Degradation

Every new tool path follows the same contract: if the required binary is not on PATH, emit a `⚠️ [tool] not found — skipping` warning and return a soft-skip result. No new hard gate failures were introduced.

## Upgrade Notes

No breaking changes. All new detection is additive — existing hardcoded detection paths remain as fallbacks. Tools that previously only supported TS/JS/Python/Rust now silently skip unsupported languages if optional binaries are absent.

Optional tools to install for full coverage:
- **Go**: `go install golang.org/x/vuln/cmd/govulncheck@latest`, `go install github.com/golangci/golangci-lint/cmd/golangci-lint@latest`
- **Semgrep**: `pip install semgrep`
- **Ruby**: `gem install bundler-audit rubocop`
- **Dart**: included with the Flutter/Dart SDK
- **Kotlin**: `brew install ktlint` or download from GitHub releases
- **Swift**: `brew install swiftlint`
- **C/C++**: `brew install cppcheck`
