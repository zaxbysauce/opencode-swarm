# v6.53.5

## What changed

### Full-auto command discoverability and config guard
- The TUI shortcut for `/swarm full-auto` now uses a dashed template (`/swarm-full-auto`) so OpenCode's shortcut routing correctly dispatches it to the plugin command handler instead of treating it as an unknown subcommand.
- A config guard prevents activating full-auto mode at the session level when `full_auto.enabled` is not set in the plugin config. Additionally, session snapshot restore now reconciles `fullAutoMode` against the current config — if a restored session had full-auto enabled but the config no longer allows it, the flag is cleared on rehydration.

### Full-auto autonomous continuation at phase boundaries
- After the critic returns APPROVED or ANSWER verdicts, the intercept hook now injects explicit `[FULL-AUTO CONTINUATION]` user-role messages that instruct the architect to proceed. Previously, only a passive verdict was injected, causing the architect to stall waiting for user input.
- Escalation pattern detection broadened from 5 to 14 patterns to catch more question/phase-completion signals.

### `/swarm close` full session finalization
- Rewritten as a 4-stage pipeline: finalize (retros + knowledge curation) -> archive (timestamped bundle) -> clean (state removal) -> align (safe git fast-forward).
- The clean stage uses per-file archive tracking: only active-state files that were successfully copied to the archive are deleted. This prevents data loss in partial-archive scenarios where some files copy but others fail.
- Safe git alignment includes detached HEAD detection, dirty worktree checks, and fast-forward-only merge policy.

### `/swarm handoff` continuation prompt
- `formatContinuationPrompt()` rewritten to emit a structured, copy-paste-ready markdown block with phase, current/next task, pending QA blockers, recent decisions, and numbered resume steps.
- The prompt is written atomically to `.swarm/handoff-prompt.md` and included in the command output as a fenced code block.

### Skill/command namespace collision fix
- `.claude/skills/swarm/SKILL.md` now lists known plugin subcommands so that models using the skill system (rather than the plugin hook) correctly identify `/swarm close`, `/swarm handoff`, etc. as plugin commands rather than task descriptions.
- Includes a sync comment referencing `src/commands/registry.ts` to keep the list current.

## Why
- Full-auto was unusable in TUI due to shortcut routing mismatch and could enter a broken half-enabled state.
- Full-auto stalled at every phase boundary, defeating autonomous operation.
- `/swarm close` left orphan state files and didn't archive evidence or align git.
- `/swarm handoff` produced vague continuation guidance that was insufficient for session resumption.
- External models (GPT 5.4 mini, Kimi K2.5) couldn't find `/swarm close` because the skill intercepted `/swarm` before the plugin hook.

## Migration steps
None required. All changes are backward-compatible.

## Breaking changes
None.

## Known caveats
- The SKILL.md subcommand list is a static allowlist that must be kept in sync with `COMMAND_REGISTRY` in `src/commands/registry.ts` when new subcommands are added.
- The dashed template fix only applies to the `full-auto` shortcut; other shortcuts continue to use the spaced template, which works correctly via the plugin's `command.execute.before` hook.
