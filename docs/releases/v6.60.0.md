# v6.60.0

## Summary
- CI pipeline test trigger to evaluate test suite execution
- Fix CI bun version mismatch causing dist-check failures
- Fix macOS unit failures caused by symlink path mismatch in guardrails authority tests

## What changed
- Added CI test trigger file to initiate evaluation of CI pipeline
- Fixed Biome formatting on 6 source/test files that caused quality gate failure
- Updated CI bun version from 1.3.8 to 1.3.11 across all 7 job definitions in
  `.github/workflows/ci.yml` to match the local development environment used to
  build and commit `dist/`. The version mismatch caused `dist-check` to fail on
  every CI run.
- Fixed 4 `beforeEach` blocks in `tests/unit/hooks/guardrails-authority.test.ts`
  that create temporary directories via `mkdtemp` and then call into
  `checkFileAuthority` / `createGuardrailsHooks`. On macOS, `/tmp` is a symlink to
  `/private/tmp`. `guardrails.ts` calls `fs.realpathSync` internally to resolve
  symlinks for security normalization, which resolves `/tmp/xxx` to
  `/private/tmp/xxx`. Without the fix, `path.relative('/tmp/xxx',
  '/private/tmp/xxx/.swarm/plan.md')` produces a broken traversal path that
  bypasses authority checks and causes test failures. Fixed by wrapping each
  `mkdtemp` call with `await fs.realpath(...)` so `tempDir` holds the resolved
  canonical path before any authority checks run.
- Converted all batched hook test CI steps to per-file isolation to prevent
  `--smol` module cache poisoning across test files (confirmed 82 spurious
  failures when system-enhancer files are batched in one `bun` invocation).
- Added 9 previously uncovered hook test files to CI: `destructive-command-guard`,
  `full-auto-intercept` (3 files), `review-receipt`, `system-enhancer-coder-context`,
  `system-enhancer-test-constraints`, `telemetry-guardrails-wiring`, and
  `trajectory-logger`.

## Why
The committed `dist/` was built with bun 1.3.11 (local dev environment) but CI
was pinned to bun 1.3.8. Bun's bundler produces version-specific output, so
`git diff --exit-code dist/` always detected drift after rebuilding with 1.3.8.

The macOS failures were caused by the `/tmp` → `/private/tmp` symlink: four test
describe blocks in `guardrails-authority.test.ts` create real files under a
`mkdtemp` path and then `chdir` into it, but `guardrails.ts` resolves the file
paths through `realpathSync` for security normalization. The mismatch between the
unresolved `tempDir` (`/tmp/xxx`) and the realpath-resolved canonical path
(`/private/tmp/xxx`) caused `path.relative()` to generate invalid traversal paths.
