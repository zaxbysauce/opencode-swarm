# v6.84.6 (2026-04-25)

## What changed

- **Fixed ProviderModelNotFound for default agent models** — Replaced `opencode/trinity-large-preview-free` (no longer consistently available in free tier) with `opencode/big-pickle` across all agent defaults.
- **Added fallback protection** — Added `opencode/gpt-5-nano` as `fallback_models` for all agents with potentially-unavailable primary models. When a primary model is unavailable, the guardrails fallback chain has a recovery path.
- **Implemented multi-level fallback chains** — Expanded fallback models from single-level to 2-level chains using only models confirmed to be consistently available in free tier (`big-pickle` and `gpt-5-nano`). This ensures users have a deeper recovery path if their primary model becomes unavailable.
  - Primary and reviewer agents: `fallback_models: ['opencode/gpt-5-nano', 'opencode/big-pickle']` (cycles back to big-pickle on deep exhaustion)
  - Lightweight agents (test_engineer, curator, critic secondary): `fallback_models: ['opencode/big-pickle']` (single fallback to primary that's often available)
  - This maximizes availability in constrained environments while respecting the schema limit of 3 fallbacks per agent.

## Why

Users would encounter `ProviderModelNotFound` when default agent models tried to initialize with `opencode/trinity-large-preview-free`, which was removed from the OpenCode Zen free model roster. Single-level fallback (`gpt-5-nano` alone) leaves users stranded if that model also becomes temporarily unavailable. Multi-level fallback ensures better resilience.

## Migration steps

- **New installations:** No action required; fresh installs use the corrected defaults automatically with multi-level fallbacks.
- **Existing installations with an `opencode-swarm.json`:** The installer does not overwrite existing config files. If you explicitly pinned `opencode/trinity-large-preview-free` (or another removed/unavailable model), update that setting to a currently supported model such as `opencode/big-pickle`.
- **To upgrade existing configs with multi-level fallbacks:** Add secondary fallbacks to agents:
  - For agents using `big-pickle` as primary: add `fallback_models: ['opencode/gpt-5-nano', 'opencode/big-pickle']`
  - For agents using `gpt-5-nano` as primary: add `fallback_models: ['opencode/big-pickle']`

## Known caveats

- `opencode/minimax-m2.5-free` (coder) remains "sometimes available" — it now has 2-level fallback (`gpt-5-nano`, then `big-pickle`)
- Curator agents (`curator_init`, `curator_phase`) now have explicit model assignments with 1-level fallback for better reliability
- Council agents (`council_member`, `council_moderator`) now have explicit model assignments with 1-level fallback
