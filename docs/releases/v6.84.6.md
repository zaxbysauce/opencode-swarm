# v6.84.6 (2026-04-25)

## What changed

- **Fixed ProviderModelNotFound for default agent models** — Replaced `opencode/trinity-large-preview-free` (no longer consistently available in free tier) with `opencode/big-pickle` across all agent defaults.
- **Added fallback protection** — Added `opencode/gpt-5-nano` as `fallback_models` for all agents with potentially-unavailable primary models. When a primary model is unavailable, the guardrails fallback chain has a recovery path.

## Why

Users would encounter `ProviderModelNotFound` when default agent models tried to initialize with `opencode/trinity-large-preview-free`, which was removed from the OpenCode Zen free model roster.

## Migration steps

- **New installations:** No action required; fresh installs use the corrected defaults automatically.
- **Existing installations with an `opencode-swarm.json`:** The installer does not overwrite existing config files. If you explicitly pinned `opencode/trinity-large-preview-free` (or another removed/unavailable model), update that setting to a currently supported model such as `opencode/big-pickle`.
- **To add fallback protection to existing configs:** Add `fallback_models: ['opencode/gpt-5-nano']` entries to agents using potentially-unavailable primary models.

## Known caveats

- `opencode/minimax-m2.5-free` (coder) remains "sometimes available" — it now has `gpt-5-nano` as a fallback
- Curator agents (`curator_init`, `curator_phase`) inherit their *model* from explorer but do not inherit `fallback_models` unless explicitly configured. The default installer now includes `fallback_models: ['opencode/gpt-5-nano']` for curator agents.
