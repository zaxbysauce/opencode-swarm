# Release 7.20.0

## What changed

- **test_runner: three-layer defense against session-blocking fan-out** (`src/tools/test-runner.ts`)
  - Layer 1: `MAX_SAFE_SOURCE_FILES = 1` — synchronous pre-resolution guard fires when `sourceFiles.length > 1` (from PR #867)
  - Layer 2: `estimateFanOut()` — pre-resolution fan-out estimation via cached impact map read (~100ms) fires when `sourceFiles.length === 1` but fan-out would exceed 50
  - Layer 3: `budgetExceeded` returned from `analyzeImpact()` — runtime guard during graph traversal

- **test_runner: new `estimateFanOut` function** (`src/tools/test-runner.ts`)
  - Reads cached impact map without spawning subprocesses
  - Completes in <100ms by design
  - Used by Layer 2 guards for `graph` and `impact` scopes

- **analyzer: `skipRebuild` option for `loadImpactMap`** (`src/test-impact/analyzer.ts`)
  - When `skipRebuild: true`, returns stale cache instead of rebuilding
  - Enables `estimateFanOut` pre-resolution reads without triggering unbounded impact map rebuilds

- **analyzer: `budget` parameter for `analyzeImpact`** (`src/test-impact/analyzer.ts`)
  - Third parameter to `analyzeImpact()` limits traversal to a maximum test file count
  - Sets `budgetExceeded: true` on the return value when budget is exhausted

- **Skill docs: running-tests updated** (`.opencode/skills/running-tests/SKILL.md`)
  - Merged PR #867 scope safety table with three-layer defense documentation
  - Added "Treat as SKIP without retry" guidance to error messages

## Why

Fixes the root cause of Issue #864: LLMs calling `test_runner` with multiple source files fan out through the import graph, exceed `MAX_SAFE_TEST_FILES = 50`, get `scope_exceeded`, then retry with `scope: 'all'` which kills the OpenCode SSE session.

The three-layer defense catches this at progressively earlier points:
1. Before any I/O when too many source files are passed
2. Before graph traversal when a single file would fan out to too many tests
3. During impact analysis if the budget is exhausted

## Migration steps

None — this is a purely additive change. Existing single-file `test_runner` calls are unaffected.

## Breaking changes

None.

## Known caveats

- `estimateFanOut` requires a cached impact map (`.swarm/cache/impact-map.json`). On first run or cold cache, it returns `0` and the Layer 2 guard is bypassed — this is safe because the Layer 1 guard (`MAX_SAFE_SOURCE_FILES = 1`) still catches the common case.
- Bun test runner (`mock.module`) has known cross-file isolation issues — affected tests pass individually but may fail in combined runs. This is a pre-existing Bun infrastructure limitation, not a code bug.
