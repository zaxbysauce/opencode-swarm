# v6.40.2

## Bug Fixes

- **Session state loss on fast exit**: `createSnapshotWriterHook` used a 2-second debounce that silently dropped writes when the process exited before the timer fired. Replaced with an immediate promise-chain write; concurrent calls are serialized so the last writer wins. Also fixes 10 previously-failing session unit tests caused by the debounce.

- **Stale snapshot re-read loop**: `readSnapshot` returned `null` for incompatible state versions but left the file in place, causing it to be silently rejected on every subsequent restart. It now renames `state.json → state.json.quarantine` before returning `null`, clearing the read path for future startups.

## Internal Quality Improvements

Remediates 8 findings from the AI slop review report, improving code maintainability, observability, and consistency. No API or behavioral changes.

### What Changed

- **God function decomposition**: The 462-line `toolBefore` handler in `guardrails.ts` has been decomposed into 6 named sub-handlers (`handleDelegatedWriteTracking`, `handleLoopDetection`, `handleTestSuiteBlocking`, `handlePlanAndScopeProtection`, `resolveSessionAndWindow`, `trackToolCall`). The orchestrator is now 44 lines.

- **Duplicate function elimination**: `estimateComplexity` in `complexity-hotspots.ts` was a character-for-character copy of `estimateCyclomaticComplexity` in `quality/metrics.ts`. The duplicate is removed; the canonical version is now exported and shared.

- **Silent catch block logging**: Added debug-level logging (`log()`, only active when `OPENCODE_SWARM_DEBUG=1`) to 12 previously silent catch blocks across `guardrails.ts`, `decision-drift-analyzer.ts`, `config-doctor.ts`, `handoff-service.ts`, and `role-filter.ts`.

- **Dead code removal**: Deleted unused constants `_N_GRAM_SIZE` (metrics.ts) and `_TEST_CONTENT_PATTERNS` (placeholder-scan.ts).

- **Dead variable fix**: The `oldState` variable in `circuit-breaker.ts` `transitionTo()` is now passed to `onStateChange` callbacks, adding transition observability and removing the `biome-ignore` suppression.

- **Function consolidation**: Replaced local `containsControlCharacters` in `symbols.ts` with the canonical `containsControlChars` from `utils/path-security.ts`.

- **Test reclassification**: Added documentation header to `architect-v6-prompt.test.ts` clarifying that its 400 assertions are prompt regression tests, not behavioral or security tests.

- **Contributing skill**: New `.opencode/contributing/SKILL.md` skill automates the full PR workflow from `contributing.md`.

### Why

These are all internal quality improvements identified by an AI slop review. The goal is to reduce maintenance burden, improve debuggability, and eliminate copy-paste divergence risks. The session snapshot fixes resolve a silent data-loss bug introduced in v6.33.1.
