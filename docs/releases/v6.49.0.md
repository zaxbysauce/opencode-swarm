# v6.49.0 — README Comprehensive Documentation Update

## What changed

### README accuracy and completeness pass

This release brings the README into full alignment with the codebase as of v6.48. Every
section was audited against the release notes for v6.41 through v6.48. No code changes.

### Critic drift detection — documented

The `critic_drift_verifier` agent entry now explains what drift detection actually means:
it compares the completed implementation against the original plan spec at phase close,
returns APPROVED or NEEDS_REVISION, and is a **blocking gate** (not advisory) when
NEEDS_REVISION is returned. The distinction between the three critic roles is now
explicit in the agents table. The `PHASE-WRAP` mode entry now lists the full set of
steps executed at phase close, including the critic delegation, `write_drift_evidence`
call, and the mandatory dual-gate verification (`completion-verify.json` +
`drift-verifier.json`) required before `phase_complete` is accepted.

### Curator default corrected (`true`, not `false`)

The Curator section described `curator.enabled` as `false` by default in three places.
The default changed to `true` in v6.42. All three locations are corrected:

- Section description updated to "enabled by default as of v6.42"
- Configuration JSON example changed from `"enabled": false` to `"enabled": true`
- `enabled` field default in the config table changed from `false` to `true`
- v6.22 recent-changes blurb amended to note the v6.42 default change

The Curator section also now mentions that since v6.42, the Curator performs real LLM
analysis by delegating to the Explorer agent. Before v6.42 the delegation was scaffolded
but never wired.

### `/swarm close` added to command tables

`/swarm close` (introduced v6.41, expanded v6.47) was absent from both the quick-reference
commands table and the full "All Slash Commands" table. Both tables now include it with a
description covering: retrospectives, lesson curation, evidence archiving, `context.md`
reset, config-backup cleanup, and the `--prune-branches` flag. The `.swarm/close-lessons.md`
lesson injection mechanism is also documented.

### Plan ledger / durability documented

The `.swarm/` directory tree now reflects the v6.44 plan durability model:

- `plan-ledger.jsonl` added as the authoritative source of truth
- `plan.md` and `plan.json` annotated as projections derived from the ledger
- `SWARM_PLAN.md` and `SWARM_PLAN.json` checkpoint artifacts added
- A callout explains crash recovery and the import/restore path
- `docs/plan-durability.md` added to the Documentation section

The Persistent Memory section previously implied `plan.json` was authoritative. This
no longer holds since v6.44.

### New tools added to Available Tools Reference

Three tools introduced in v6.45 were absent from the tools table and agent allocations:

- `search` — workspace-scoped ripgrep-style structured text search; registered for
  architect, coder, reviewer, explorer, test_engineer
- `suggest_patch` — reviewer-safe context-anchored patch suggestion (read-only, no file
  writes); registered for reviewer and architect
- `batch_symbols` — batched symbol extraction from multiple files in one call;
  75–98% call reduction vs sequential; registered for architect, explorer, reviewer

Two additional tools missing from the table were also added:

- `save_plan` — plan persistence with ledger writes and checkpoint artifact generation;
  requires explicit `working_directory` parameter
- `co_change_analyzer` — dark matter detection; scans git history for co-changing files
  during DISCOVER mode; architect-only

The architect's tool count was changed from the stale "23 tools" to "All registered
tools" to avoid future staleness as the tool set grows.

### Step 5l-ter added to EXECUTE pipeline

The test drift detection step (added v6.45) was absent from the Full Execution Pipeline
diagram. It now appears between step 5l (regression sweep) and 5m (pre-commit checklist),
with a description of the six conditional triggers.

### Dark matter detection documented in DISCOVER mode

DISCOVER mode description now notes that co-change analysis runs automatically during
every DISCOVER pass to detect hidden file couplings. Git-less repos and repos with
fewer than 20 commits skip silently.

### Missing slash commands added to full command table

The following commands were absent from the "All Slash Commands" table:

`/swarm close`, `/swarm write-retro`, `/swarm reset-session`, `/swarm simulate`,
`/swarm promote`, `/swarm evidence-summary`, `/swarm knowledge-add`,
`/swarm knowledge-remove`, `/swarm handoff`, `/swarm turbo`, `/swarm checkpoint`

### Evidence bundle types table completed

Two mandatory gate evidence types were missing from the Evidence Bundles table:

- `completion-verify` — deterministic gate written automatically before `phase_complete`
- `drift-verifier` — written by architect via `write_drift_evidence` after critic verdict

### `.swarm/` directory tree updated

Five entries were missing: `plan-ledger.jsonl`, `SWARM_PLAN.md`, `SWARM_PLAN.json`,
`close-summary.md`, `close-lessons.md`, `doc-manifest.json`, `spec.md`,
and `curator-briefing.md`. All are now present with descriptions.

### Recent Changes section updated

Added summaries for v6.42.0, v6.44.0, v6.45.0, and v6.47.0 — none were reflected in
the recent changes section despite shipping significant user-facing features.

## Why

A gap analysis against all release documentation (v6.34–v6.48) revealed 14 categories
of missing or incorrect information. The most impactful were: the `curator.enabled`
default being wrong (causing confusion for users upgrading from v6.41), the Critic's
drift detection role being invisible in the docs despite being a blocking production
gate, and `/swarm close` being entirely absent from both command reference tables.

## Migration steps

None — documentation-only change.

None.
