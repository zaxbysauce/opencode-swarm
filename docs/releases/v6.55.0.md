# v6.55.0

## What changed

### Conflict resolution mechanisms (Issue #414)

Four targeted additive additions from the Conflict Resolution Mechanisms Audit Report. No existing logic was modified.

#### New event types (`src/types/events.ts`)

- **`AgentConflictDetectedEvent`** — telemetry event emitted when two agents enter a reject/resubmit conflict. Carries `sourceAgent`, `targetAgent`, `conflictType` (feedback_rejection | authority_collision | retry_spiral | scope_disagreement | quality_gate_dispute), and `resolutionPath` (self_resolve | soundingboard | simplification | sme_consult | user_escalation).
- **`AuthorityHandoffResolvedEvent`** — telemetry event type defined for when a stale or conflicted delegation authority is resolved. Carries `previousAgent`, `newAgent`, and `reason` (task_complete | stale_delegation | conflict_escalation | manual_reset). **Not yet emitted** — type is defined and included in `V619Event` for forward compatibility; emit wiring is deferred to a follow-up PR.
- Both types are included in the `V619Event` union.

#### New hook (`src/hooks/conflict-resolution.ts`)

- **`resolveAgentConflict(input)`** — centralized conflict resolution policy. Call this when a Reviewer rejects Coder output repeatedly, when the Critic returns REPHRASE/RESOLVE and the Architect loops, or when the Test Engineer blocks previously reviewed work.
  - `rejectionCount < 3` → pushes a `CONFLICT DETECTED` advisory into `pendingAdvisoryMessages` (self-resolve path).
  - `rejectionCount >= 3` → pushes a `CONFLICT ESCALATION` advisory (route to Critic in SOUNDING_BOARD mode).
  - No-ops if no session exists for the given `sessionID`.
  - Emits `agent_conflict_detected` telemetry (fire-and-forget cast; does not widen `TelemetryEvent` union — wiring deferred to a follow-up PR).

#### New adversarial pattern (`src/hooks/adversarial-detector.ts`)

- **`REJECTION_SPIRAL`** — new pattern group added to `detectAdversarialPatterns()`. Triggers at `severity: HIGH` / `confidence: HIGH` when an agent text signals a stuck reject/resubmit cycle (e.g. "rejected again for the 3rd time", "same feedback again", "stuck in a loop"). Pattern 3 uses `stuck|trapped|endless|repeating` + `loop|cycle` to avoid false positives on neutral process language such as "our feedback loop".
- `AdversarialPatternMatch.pattern` union extended with `'REJECTION_SPIRAL'`.

#### New delegation metadata (`src/state.ts`, `src/hooks/delegation-tracker.ts`)

- **`DelegationReason` type** exported from `src/state.ts`: `normal_delegation | review_rejected | critic_consultation | retry_circuit_breaker | conflict_escalation | stale_recovery`.
- **`lastDelegationReason?: DelegationReason`** added to `AgentSessionState` (after `delegationActive`). Set to `'normal_delegation'` whenever a non-architect agent is activated via `createDelegationTrackerHook`. Not cleared on architect reset — the field preserves the reason from the most recent non-architect activation.
- `DelegationReason` re-exported from `src/hooks/delegation-tracker.ts` for consumer convenience.

#### Sounding board parser (`src/agents/critic.ts`)

- **`SoundingBoardVerdict`** type: `'UNNECESSARY' | 'REPHRASE' | 'APPROVED' | 'RESOLVE'`.
- **`SoundingBoardResponse`** interface: `verdict`, `reasoning`, optional `improvedQuestion` (REPHRASE), `answer` (RESOLVE), `warning` (MANIPULATION DETECTED).
- **`parseSoundingBoardResponse(raw: string): SoundingBoardResponse | null`** — regex-based parser for raw Critic sounding board output. Lenient on whitespace and casing. Returns `null` for empty input or missing verdict line. Populates `warning` from explicit `Warning:` lines or `[MANIPULATION DETECTED]` markers. Optional-field regexes use explicit section-header lookaheads (`Answer|Warning|Verdict`) to correctly capture multiline values.

## Why

The Conflict Resolution Mechanisms Audit (Issue #414) identified four gaps in the pipeline's ability to detect, record, and resolve agent-vs-agent conflicts:

1. No typed event for recording when two agents enter a conflict, making post-mortem analysis difficult.
2. No centralized policy for escalating repeated reviewer rejections to the Critic's sounding board.
3. No adversarial pattern for detecting when the pipeline stalls in a reject/resubmit spiral.
4. No typed parser for Critic sounding board output, forcing all consumers to parse raw strings.

These additions provide the missing infrastructure without modifying any existing logic.

## Migration steps

No migration required. All additions are purely additive:
- New event types in `V619Event` union — existing type guards on specific event types are unaffected.
- `lastDelegationReason?` is optional — existing `AgentSessionState` consumers do not break.
- `resolveAgentConflict` is not exported from `src/index.ts` — no accidental surface area added.

## Breaking changes

None.

## Known caveats

- `agent_conflict_detected` telemetry is emitted via a type cast (not in the `TelemetryEvent` union). It will appear in `telemetry.jsonl` but is not yet handled by any listener. Union wiring is deferred to a follow-up PR.
- `lastDelegationReason` is only set to `'normal_delegation'` by `createDelegationTrackerHook`. Callers that want to record other reasons (`review_rejected`, `conflict_escalation`, etc.) must set the field directly on the session.
