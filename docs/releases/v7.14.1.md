# v7.14.1

## What changed

### Full-auto deadlock fix
- Fixed a deadlock where BLOCKED oversight verdicts returned `'pause'` (system-wide halt) instead of `'deny'` (local tool rejection). This caused full-auto sessions to freeze irrecoverably when a single tool was blocked.
- Added `architectOutput` serialization to the permission hook so blocked-tool diagnostics include the architect's reasoning instead of `[object Object]`.

### Knowledge system hardening (14 findings from deep-dive audit)
- **Atomicity**: `/swarm acknowledge-spec-drift` now updates the plan before deleting the staleness file — a crash between the two operations previously left the plan updated but the staleness flag intact, causing spurious drift warnings.
- **Idempotency**: `knowledge_ack` falls back to a random UUID when `ctx.sessionID` is missing, preventing dedup-key collisions across sessions.
- **Type safety**: Fixed a TypeScript generic parameter narrowing issue in `bumpCountersBatch` by using a standard `unknown` intermediate cast.
- **Sentinel delimiter**: Knowledge injection idempotency now uses a zero-width-non-joiner + ASCII sentinel instead of an emoji, preventing false-positive duplicate detections when the BOOK emoji appears naturally in user prompts.
- **Quarantine/restore**: `restoreEntry` now re-validates lessons via `validateLesson()` before restoring quarantined entries, and `quarantineEntry` stores the original status for proper restoration.
- **Observability**: Added defensive warning log for unreachable `recordLessonsShown` catch path. Documented `sweepStaleTodos` free-lifetime behavior with JSDoc.
- **Test coverage**: Added ~130 new tests covering knowledge-recall (40 tests), knowledge-remove (20 tests), full-auto deadlock regression (6 tests), malformed JSONL entries (5 tests), and dangerous command validation (6 tests). Unskipped a previously-broken test in knowledge-reader and fixed its assertion.

## Why
A deep-dive audit of the knowledge system identified 14 findings (5 HIGH, 6 MEDIUM, 3 LOW) across atomicity, idempotency, observability, type safety, and test coverage. The full-auto deadlock was discovered during Phase 5 investigation as a two-root-cause issue — fixing only the verdict mapping or only the serialization would not have resolved it.

## Breaking changes
None. All changes are backward-compatible.

## Known caveats
- Three pre-existing test files (`knowledge-reader.test.ts`, `knowledge-store.test.ts`, `knowledge-validator.test.ts`) use `vitest` imports instead of `bun:test`. This is a pre-existing issue unrelated to this PR.
- The `chmod` Layer 2 dangerous-command pattern (`/\bchmod\s+-R\s+777\b/`) is case-sensitive but `validateLesson` lowercases input before matching — the pattern never matches `chmod -r 777`. Logged for a future fix.
