# OpenCode Swarm v6.33.6 Release Notes

## Summary

Version **6.33.6** fixes mid-session hangs caused by the stale delegation check forcefully reverting active sub-agent identity to architect after 10 seconds, and stale per-session state surviving process restarts.

---

## Bug Fixes

### 1. Stale delegation check hijacked active sub-agents

The stale delegation reversion in `tool.execute.before` used `||` (OR) logic:
```
!session.delegationActive || Date.now() - lastAgentEventTime > 10s
```

Since `lastAgentEventTime` is only updated on `chat.message` events (not tool calls), the second condition became true after 10 seconds of coder tool activity. This forcefully reverted the active agent identity to ORCHESTRATOR_NAME while the coder was still running, causing:

- Guardrails to exempt the coder (thinking it was the architect)
- False architect-only warnings injected into coder's messages (self-coding, gate violations, advisories)
- System prompt corruption via architect-specific context injection
- The LLM becoming confused and unresponsive

**Fix:** Changed `||` to `&&` so the reversion only fires when delegation has **explicitly ended** (delegationActive=false, set by Task toolAfter or chat.message) AND the chat.message callback is delayed (>10s).

### 2. Transient per-session state survived process restart

While v6.33.5 reset InvocationWindow counters and timestamps on rehydration, many other transient fields were preserved from the previous process lifetime:

| Field | Effect when stale |
|-------|-------------------|
| `revisionLimitHit` | Permanently caps coder revisions |
| `coderRevisions` | Premature revision limit |
| `selfFixAttempted` + `lastGateFailure` | False self-fix detection warnings |
| `architectWriteCount` | False self-coding warnings |
| `pendingAdvisoryMessages` | Stale advisories injected into prompts |
| `model_fallback_index` / `modelFallbackExhausted` | Stuck fallback state |
| `scopeViolationDetected` | False scope violation warnings |
| `delegationActive` | Prevents clean delegation lifecycle |

**Fix:** All transient fields are now reset to clean defaults during rehydration, alongside the existing timestamp and window resets.

---

## Migration Notes

- No config changes required
- Existing `.swarm/session/state.json` snapshots are fully compatible
- Plan/evidence data (taskWorkflowStates, gateLog, reviewerCallCount) is preserved through restart as before — only transient counters and flags are reset
