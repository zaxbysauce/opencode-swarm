# v7.0.1

## What changed

### Issue #583: Housekeeping

- **SWARM_PLAN relocation**: Moved `SWARM_PLAN.{json,md}` from project root to `.swarm/` directory for better artifact containment and cleaner project structure
- **Backward compatibility**: `importCheckpoint()` gracefully falls back to root-level `SWARM_PLAN.json` for pre-v7.x sessions with deprecation warning
- **Cleanup during transition**: `/swarm close` deletes SWARM_PLAN artifacts from both `.swarm/` and root locations during the migration window
- **Spiral detection fix**: New `saveCheckpointRecord()` function records checkpoint state without staging or committing user work, eliminating unwanted mid-flight git commits
- **Documentation update**: Updated plan-durability.md and recovery-guide.md to reflect the new `.swarm/` location; updated log messages in src/plan/manager.ts

### Issue #698: Update/install cache fixes

- **Platform-specific cache paths**: Extended `getPluginCachePaths()` to include:
  - macOS: `~/Library/Caches/opencode/` paths in addition to XDG standard
  - Windows: `%LOCALAPPDATA%\opencode\` and `%APPDATA%\opencode\` paths with env-var fallbacks
  - Fixes detection of plugin caches across all platform layout variations
- **Lock file eviction**: New `getPluginLockFilePaths()` and `evictLockFiles()` functions clear bun.lock/bun.lockb/package-lock.json candidates so bun re-resolves `@latest` on next start
- **Safety guards**: Multi-layer validation for lock file paths:
  - ≥4 segment depth check (prevents root/home directory accidents)
  - Basename whitelist (bun.lock, bun.lockb, package-lock.json only)
  - Parent directory must be named `opencode`
  - EISDIR-safe unlink handling
- **Integrated eviction**: Both `update()` and `install()` commands now evict lock files in addition to plugin cache
- **User feedback**: Improved output lists lock file locations checked in the no-cache-found diagnostic branch

## Migration steps

No user action required. The migration from root-level to `.swarm/` SWARM_PLAN is automatic and backward compatible:
- New sessions create `.swarm/SWARM_PLAN.json`
- Sessions resuming from pre-v7.x checkpoints load from root with a deprecation warning (visible with `OPENCODE_SWARM_DEBUG=1`)
- Running `/swarm close` migrates and cleans up both locations
