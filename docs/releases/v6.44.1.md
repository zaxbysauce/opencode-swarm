# v6.44.1

## What changed

### CI: per-file subprocess isolation for integration tests

The integration job in `ci.yml` now runs each test file in its own `bun --smol` subprocess. Previously, all integration and `./test` files ran in a single `bun test` invocation, which caused ~48 ordering-dependent failures due to shared module cache and `process.chdir()` contamination between tests.

### Bug fixes across 7 source/test areas

**`src/tools/update-task-status.ts`** — `checkReviewerGate` used `workingDirectory!` (non-null assertion) when the parameter was `undefined`. Calling `path.join(undefined, ...)` threw a `TypeError` that was silently caught and reported as "evidence file corrupt or unreadable", blocking 12 workflow-state-machine tests.

**`src/tools/lint.ts`** — `detectAvailableLinter` now resolves binaries via ancestor directory walk-up and a manual `process.env.PATH` scan. `Bun.which` does not pick up runtime `PATH` mutations (set in test `beforeEach`), so a manual scan is required. A new `resolveLinterBinPath()` export ensures detection and execution use the same binary path.

**`src/tools/pre-check-batch.ts`** — `runLintOnFiles` now calls `resolveLinterBinPath()` instead of hardcoding `workspaceDir/node_modules/.bin/<linter>`, keeping detection and execution consistent.

**`src/hooks/guardrails.ts`** — `handleDelegatedWriteTracking` passed the raw `directory` parameter (which is the config object in legacy calls) to `checkFileAuthority`, causing `path.relative(config_object, ...)` to throw `TypeError: The "from" property must be of type string, got object`. Fixed to use the already-normalized `effectiveDirectory`.

**`src/background/plan-sync-worker.ts`** — `start()` now always runs polling alongside the native `fs.watch` watcher (not as a fallback). `fs.watch` is unreliable in Bun's test environment; polling must fire concurrently.

## Why

CI was non-green. The `continue-on-error: true` flag on the integration job masked 84 failures. This PR removes the mask by fixing the root causes so the job passes without the bypass.

## Migration steps

None. All changes are internal to test infrastructure and tool internals.

## Breaking changes

None.

## Known caveats

- ~120 unit tests in `tests/unit/tools/checkpoint*.test.ts` and `tests/unit/commands/checkpoint.test.ts` remain failing. These are pre-existing failures unrelated to this PR (verified by checking out the base commit).
- 1 security test (`Command Injection Patterns`) remains failing — also pre-existing.
- The integration test suite must be run with per-file subprocess isolation (`bun --smol test <file>`) to avoid ordering contamination; the batch command `bun test tests/integration ./test` still shows ~57 failures due to shared module state.
