# v6.53.3

## What changed

### TUI shortcut routing fixed for multi-word compound commands

`/swarm config-doctor` and `/swarm evidence-summary` TUI shortcuts now route correctly to their handlers. Previously, selecting these from the OpenCode command picker caused them to fall through to generic help text.

**Root cause**: When a user selects a `swarm-config-doctor` shortcut, OpenCode sends `command: 'swarm-config-doctor'` to the handler. The handler strips the `swarm-` prefix to get `config-doctor` (dash-joined), then looks it up in `COMMAND_REGISTRY`. The registry only had `'config doctor'` (space-joined) keys for these compound commands, so the lookup failed silently.

**Fix**: Added `'config-doctor'` and `'evidence-summary'` dash-joined aliases in `COMMAND_REGISTRY` pointing to the existing handlers. Space-based paths (`/swarm config doctor`, `/swarm evidence summary`) are unaffected — `resolveCommand` tries the space-joined compound key first and only falls through to the dash alias when called via the TUI shortcut path.

### Catch-all `/swarm` command description updated

The description shown in the command picker for the catch-all `/swarm` command now lists all 30 top-level subcommands. Seven previously missing commands (`turbo`, `full-auto`, `write-retro`, `reset-session`, `simulate`, `promote`, `checkpoint`) were added to the description.

### Biome lint error resolved

Fixed an unused caught variable `error` → `_error` in `src/tools/phase-complete.ts` that was causing `biome ci` to report 2 errors (format + lint). Test files `full-auto-registration.test.ts` and `shortcut-routing.test.ts` were reformatted to satisfy biome format requirements.

### Test improvements

- **`full-auto-registration.test.ts`**: Fixed a trivially-passing assertion that tested a hard-coded string literal instead of retrieving the actual key from `COMMAND_REGISTRY`. The test now proves the registry key itself contains no spaces.
- **`shortcut-routing.test.ts`**: Added regression tests verifying `swarm-config-doctor` and `swarm-evidence-summary` shortcuts route to their handlers (not help text), plus backward-compatibility tests confirming space-based compound commands still work.
- **`tests/unit/index-commands.test.ts`**: Updated command count (32→33 total including catch-all), description assertion, and `expectedSubcommands` array to include `swarm-full-auto`.

## Why

The `swarm-config-doctor` and `swarm-evidence-summary` shortcuts were silently broken — users selecting them from the picker would see the generic help text instead of the command output. The test suite had also drifted from the actual registered command set after `swarm-full-auto` was added in v6.52.0.

## Migration steps

None. All changes are backward-compatible. Existing `/swarm config doctor` and `/swarm evidence summary` (space-based) invocations continue to work identically.

## Breaking changes

None.

## Known caveats

Four additional compound registry keys (`'doctor tools'`, `'knowledge migrate'`, `'knowledge quarantine'`, `'knowledge restore'`) do not have dash aliases, but none are exposed as TUI shortcuts so there is no user-facing impact. This pre-existing inconsistency is documented for future reference.
