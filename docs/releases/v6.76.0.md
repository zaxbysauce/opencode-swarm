# v6.76.0

## What Changed

**Convention test discovery now resolves test files from any directory layout, not just colocated siblings**

- Convention scope split into two paths: direct test-file input passes through unchanged; source-file input maps to matching tests by convention
- Convention lookup now searches repo-root `tests/`, `test/`, `spec/` directories in addition to colocated files
- Language-specific test file mappings added for PowerShell (`.Tests.ps1`), Java (`src/main/java` → `src/test/java`), and Go (`_test.go`)
- Discovery normalized against `working_directory`, not ambient `process.cwd()`
- All discovery uses `resolveWorkspacePath` for consistent path handling

**Targeted execution safety for unsupported frameworks**

- Frameworks that cannot execute individual test files (`go-test`, `cargo`, `maven`, `gradle`, `dotnet-test`, `ctest`, `swift-test`) now fail explicitly with a clear error message instead of silently widening into suite/package-level runs
- `getTargetedExecutionUnsupportedReason()` returns a human-readable reason for each unsupported framework

**Test engineer prompt alignment**

- Go entry in RULES section now includes a prominent `⚠️ CANNOT TARGET` warning that go test runs packages, not individual files
- EXECUTION BOUNDARY section clarified that `_test.go` files are discovered by convention but cannot be individually executed
- Prompt guidance updated to match runner's actual contract: unsupported targeted execution should be reported as SKIPPED, not retried with broader scope

**Bug fixes from CI and code review**

- Java test file detection now uses case-sensitive suffix matching (`*Test.java`/`*Tests.java`) to prevent false positives like `Contest.java` and `Latest.java`
- Minitest targeted execution now uses `ruby -e 'require_relative ...'` pattern instead of positional file arguments, which silently dropped all files after the first
- Windows test cleanup now gracefully handles EBUSY errors when subprocess hasn't released temp directory handle

## Why

The convention scope previously treated every input as a source file and only searched for colocated test siblings. This missed common out-of-source layouts (repo-root test directories) and language-specific patterns. Additionally, frameworks like Go that target packages rather than individual files would silently execute the entire package when given a single file, producing misleading results.

## Migration

No breaking changes. Convention discovery now finds more test files than before (strict superset of previous behavior). The `getTargetedExecutionUnsupportedReason` check means that Go, Cargo, Maven, Gradle, dotnet, CTest, and Swift test files discovered by convention will be explicitly rejected for targeted execution rather than silently broadened.
