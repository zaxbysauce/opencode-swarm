# v6.63.0 — Repo Map / Code Graph

## What changed

### New tool: `repo_map` (src/tools/repo-map.ts)

- Builds a structural repo map by walking the source tree and extracting exports, imports, and call edges per file
- Uses Tree-sitter AST parsing (21 bundled WASM grammars) with regex fallback for unsupported languages
- Computes an `importanceScore` based on in-degree (how many files import each file)
- Persists the map to `.swarm/repo-map.json` with 60-second staleness caching
- Three operation modes:
  - **build** — generates or updates the full structural map
  - **localize** — returns a `LocalizationContext` for a target file (importedBy, exportsUsedExternally, blastRadius, parallelPatterns)
  - **blast-radius** — BFS traversal through the import graph to find all affected files

### Architect prompt: REPO MAP LOCALIZATION step

- New step `5c-ante` in the architect's task delegation workflow (after dark matter detection, before coder delegation)
- Instructs the architect to call `repo_map` with build/localize/blast-radius modes
- Includes a structured `LOCALIZATION CONTEXT` block in coder task delegations

### Coder prompt: LOCALIZATION AWARENESS section

- New section explaining how to interpret the LOCALIZATION CONTEXT block
- Instructions to verify imports, exports, and signature compatibility before finalizing changes

### Reviewer prompt: BLAST RADIUS VERIFICATION section

- New section requiring the reviewer to verify blast radius files for broken imports
- Checks backward compatibility of changed exported symbols

### New hook: repo-map-injector (src/hooks/repo-map-injector.ts)

- Optional auto-injection hook following the knowledge-injector pattern
- Injects compact `[REPO-MAP CONTEXT]` summaries into architect messages when task delegation is detected
- Budget-aware (skips if context >80% full), idempotent, and error-safe

## Why

The most expensive failure mode in Swarm is edits made with incomplete awareness of call sites, dependents, and blast radius — leading to reviewer churn, repeated coder loops, and brittle fixes that break unrelated code. This provides structural awareness before any edit is made.

Addresses the architect-coder looping pattern observed in v6.x and complements the v7.0 roadmap priority for Intelligent Context Engineering.

Closes #460

## Migration steps

No migration required. The `repo_map` tool is automatically available to the architect agent. The `.swarm/repo-map.json` file is gitignored and generated on demand.

## Breaking changes

None.

## Known caveats

- Tree-sitter call edge extraction is best-effort — dynamic imports, reflection-based calls, and computed member access are not tracked
- The regex fallback path supports TypeScript/JavaScript/Python import extraction only; other languages get symbol extraction without call edges
- Large repos (>50K files) will have the map truncated to top-K files by importance score
- The repo-map-injector hook has a 5-minute staleness window; rapid file changes may produce slightly outdated structural summaries
