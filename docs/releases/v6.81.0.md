# v6.81.0 — Documentation Refresh

## What changed

Comprehensive refresh of all user-facing documentation to match the current v6.81.0 feature set. No code changes; the docs tree was restructured, rewritten, and expanded with four new user guides.

### Quick-start and README

- `docs/getting-started.md` rewritten from an 82-line stub into a 293-line first-run walkthrough. Covers macOS, Linux, and Windows across both the OpenCode desktop app and TUI. Target: zero to first successful swarm task in under 15 minutes.
- `README.md` restructured from 1720 lines to 367. PRM, guardrails, quality gates, context budget, and file-locking internals moved into a collapsed "Advanced Topics" section. New "Execution Modes" table early in the doc. First 30 lines now communicate what Swarm does without the reader scrolling.

### Installation routing

- `docs/installation.md` gets a "Choose Your Path" header directing readers to the Linux/Docker guide, LLM-operator runbook, or the full reference.
- Back-links added from `docs/installation-linux-docker.md` and `docs/installation-llm-operator.md` to the canonical guide.
- Contributor-focused "Development" section removed from installation.md (belongs in CONTRIBUTING.md, not an end-user doc).

### New subsystem guides

Four docs created by extracting subsystem details from source code and the architecture deep-dive:

- `docs/commands.md` — reference for all 41 `/swarm` subcommands grouped by function, derived from `src/commands/registry.ts`.
- `docs/modes.md` — session modes (Balanced, Turbo, Full-Auto) and project modes (`execution_mode`: strict/balanced/fast), with their Tier-3 exemption list and safety counters.
- `docs/knowledge.md` — hive vs swarm knowledge, storage paths per platform, full lifecycle (creation, promotion, quarantine, TTL decay), validation pipeline, and complete `knowledge.*` config reference.
- `docs/evidence-and-telemetry.md` — evidence bundle schema (13 types), retention policy, telemetry JSONL event catalog (40 events), curator summary, drift reports, and query patterns using `jq`.

### Index and routing

- `docs/index.md` rewritten with an "I am a…" decision tree (new user / operator / contributor) and an accurate release history derived from `ls docs/releases/`.

### Cleanup and archive

- `/docs/dev/` (18 files of v6.9.0 planning artifacts) archived to `/docs/archive/dev/`. Cross-links in `docs/architecture.md` and `docs/releases/v6.41.0.md` repaired.
- Two point-in-time reports (`tech-debt-report.md`, `knowledge-system-verification-report.md`) moved from the repo root to `/docs/archive/reports/` with explicit "archived YYYY-MM-DD" headers.
- `CI_TEST_TRIGGER.md` (a CI-pipeline test artifact) deleted.
- `docs/archive/ARCHIVE.md` created as an explainer for the archive layout.

### Accuracy fixes

- Stale model names (`claude-opus-4-6`, `claude-sonnet-4-5`) replaced with canonical `anthropic/claude-sonnet-4-20250514` in README, `docs/installation.md`, and `docs/design-rationale.md`.
- "44 commands" claim (stale) corrected to "41 commands" in five locations across README, `docs/index.md`, and `docs/getting-started.md`. Count matches `src/commands/registry.ts`.
- Mode-naming drift: getting-started and README had tables listing "Strict" as a session mode alongside Turbo and Balanced. The code actually exposes Turbo and Full-Auto as session modes and `execution_mode: strict/balanced/fast` as a project mode. Tables reorganized to match.

## Why

User-facing docs had drifted substantially from the code between v6.9 and v6.81. Specifically: the quick-start guide didn't explain how to run a first task, the README buried the pitch under 1700+ lines of internal detail, installation guidance was duplicated across three files with no routing, and four subsystems (modes, knowledge, evidence, telemetry) were either undocumented or buried in architecture.md. This refresh brings docs back to parity and adds verification gates (anti-AI-slop adversarial review, completeness checks) so future drift is caught earlier.

## Migration

No migration needed. Documentation-only change; no API, config schema, or behavior changes.

## Quality gates applied

- Anti-AI-slop adversarial review (sub-agent pass) run on all newly-written and restructured docs. Seventeen flags found across the combined review passes; all resolved before commit.
- Completeness verification (independent sub-agent) confirmed: no broken cross-links in user docs, 100% command coverage in `docs/commands.md` against `src/commands/registry.ts`, and `getting-started.md` step-by-step traces against real file paths and registry entries.

## Known caveats

- Internal cross-references within `/docs/archive/dev/*.md` that use `docs/dev/...` relative paths are now broken within the archive directory. Archive docs are historical; not worth rewriting.
- `docs/releases/v6.81.0.md` (this file) is written at docs-refresh time; if release-please regenerates release notes for this version from commit messages, this file may be superseded.
