# Release v6.53.4

## What Changed

- Fixed the savePlan ledger reinitialization flow when a plan migrates to a new identity.
- Added regression coverage for the failure, success, and concurrent reinitialization paths.
- Tightened the migration regression suite so it uses the real ledger initialization path.

## Why

Issue 392 showed that a plan identity change could leave the workspace in an unsafe ledger state if reinitialization failed or raced with another writer.

## Migration Steps

No user action is required.

## Breaking Changes

None.

## Known Caveats

- The fix relies on filesystem rename semantics; on some platforms, a transient lock may still cause a warning before the normal save path continues.
