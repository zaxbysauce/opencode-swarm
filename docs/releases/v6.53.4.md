# Release v6.53.4

## What Changed

- Fixed `applyCuratorKnowledgeUpdates` in `src/hooks/curator.ts` to handle null/undefined `knowledgeConfig`, null/undefined `recommendations`, and null recommendation items without throwing.
- Expanded test coverage to include guards against these adversarial edge cases.

## Why

When `knowledgeConfig` was null or undefined, or when `recommendations` contained null items, `applyCuratorKnowledgeUpdates` would throw, breaking the curator phase. These null cases can occur in edge conditions such as incomplete knowledge base entries or corrupted state.

No user action is required.
