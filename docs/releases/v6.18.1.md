# OpenCode Swarm v6.18.1 Release Notes

## Summary

Version **6.18.1** delivers a critical fix for retrospective evidence handling. The release introduces a new `write_retro` plugin tool and accompanying slash command to correctly format retro evidence for `phase_complete`, resolving the `RETROSPECTIVE_MISSING` barrier that previously blocked all phases after retrospection. A version bump to 6.18.1 accompanies these changes.

---

## Detailed Fixes

### 1. Correct Retro Evidence Serialization

The Architect's system prompt wrote flat JSON retrospectives directly to disk, which conflicted with `EvidenceBundleSchema.parse()` expectations. This mismatch caused `phase_complete` to mistakenly flag missing retrospectives. The new `write_retro` tool now:
- Accepts flat retro fields (as written by the Architect's prompt).
- Internally constructs a `RetrospectiveEvidence` entry.
- Utilizes `saveEvidence()` to wrap the data into a proper `EvidenceBundle` with atomic write semantics.
- Returns a structured JSON response (`{ success, phase, task_id, message }`).

### 2. Introduced `/swarm write-retro` Slash Command

A user-facing slash command mirrors the plugin tool:
- Parses a JSON string from the command argument.
- Delegates to `executeWriteRetro()`.
- Provides helpful markdown feedback when no arguments are supplied or errors occur.

### 3. Fixed Task Complexity Enum

The system prompt previously used invalid `task_complexity` enum values (`"low"`, `"medium"`, `"high"`) that do not exist in `RetrospectiveEvidenceSchema`. The correct values are `"trivial"`, `"simple"`, `"moderate"`, `"complex"`.

### 4. Comprehensive Tests

Added 96 new unit and adversarial tests covering both the tool and command layers, ensuring robust functionality and error handling.

---

## Upgrade Notes

- **Backward compatible**: No breaking changes; all existing workflows continue to function unchanged.
- **Retro evidence migration**: Retro bundles written with the legacy flat JSON format will still fail `phase_complete`. Re-write them using `/swarm write-retro <json>` or programmatically via the `write_retro` tool.
- **Tool availability**: The `write_retro` plugin tool is now exposed to the Architect as a first-class plugin tool in addition to the slash command.
