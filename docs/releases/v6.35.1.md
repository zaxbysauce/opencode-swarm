# Release Notes — v6.35.1

**Patch Release | 2026-03-27**

---

## Overview

This patch release addresses four targeted fixes: evidence gate reliability after session restarts, runaway model output detection, and delegation mechanism clarity. No breaking changes.

- **Evidence gate fallback**: Gates now pass when evidence directory exists even without evidence.json, preventing false blocks after restarts
- **Runaway output detector**: New configurable guard catches models (e.g., GLM-5) generating thousands of lines without tool use
- **Delegation clarity**: Architect delegation headers now explicitly reference the Task tool
- **Critic model inheritance**: Documented for completeness — already applied in v6.35.0

---

## Bug Fixes

- **Evidence gate fallback** (`update-task-status.ts`): When no `evidence.json` exists for a task but the `.swarm/evidence/{taskId}/` directory contains other files, the gate now returns unblocked instead of falling through to unreliable in-memory session state. This prevents false blocks after session restarts.
- **Runaway output detector** (`guardrails.ts`, `schema.ts`): New configurable detector catches models generating thousands of lines of text without using tools (e.g., GLM-5 fake-thinking patterns). Advisory warning at 3 consecutive high-output (>4000 chars) turns without tool calls. Hard STOP injection at configurable max (default 5) turns. Configured via `guardrails.runaway_output_max_turns` (zod: int, min 1, max 20, default 5).
- **Critic model inheritance** (`agents/index.ts`): Critic sub-roles (sounding_board, drift_verifier) now check for explicit user config before falling back to the `critic` model. Already applied in v6.35.0; noted here for completeness.
- **Delegation mechanism clarity** (`architect.ts`): Updated delegation format header to explicitly state that delegations are performed via the Task tool, not by writing text into chat.

---

## Migration Notes

None. All changes are additive or internal. The new `guardrails.runaway_output_max_turns` config field has a default value of 5 and is optional — existing configurations continue to work without modification.
